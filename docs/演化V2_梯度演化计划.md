## 物理与抽样定义

- `flow-density-sigma` 继续作为 participant 沉积宽度 `sigma_dep`，定义初态：
  `s0 = sum_i w_i G_sigma_dep(r - r_i)`，本轮 `w_i = 1`。
- V2 新增两个额外平滑宽度：
  - `gradient-sigma-em`：发射 marker 密度的额外卷积宽度
  - `gradient-sigma-dyn`：动力学梯度密度的额外卷积宽度
- 实现中用 Gaussian 方差相加：
  - `sigma_em_eff = sqrt(sigma_dep^2 + gradient_sigma_em^2)`
  - `sigma_dyn_eff = sqrt(sigma_dep^2 + gradient_sigma_dyn^2)`
- 强制校验：`gradient-sigma-dyn > gradient-sigma-em`，因此 `s_dyn` 必然比 `s_em` 更平滑。
- 每个 participant 仍用现有 Gamma-Poisson/NBD-like 模型抽 multiplicity。
- 对每个候选粒子：
  1. 从该 participant 对应的 `s_em` Gaussian component 抽 entropy marker 初态位置 `r0`。
  2. 若 `s_em(r0) < gradient-density-cutoff-fraction * max_s_em`，最多重抽 64 次；仍失败则跳过该候选。
  3. 在 `r0` 查询 `s_dyn`：
     `G = -grad(s_dyn) / (s_dyn + floor)`，
     `floor = gradient-density-floor-fraction * max_s_dyn`。
  4. 位移：
     `dr = Lmax * tanh(kappa_r * |G|) * Ghat + xi`。
  5. `xi` 是二维 Gaussian diffusion，宽度 `gradient-diffusion-sigma`。
  6. 末态发射点：
     `rf = r0 + dr`。
  7. 横向速度：
     `vT = vmax * tanh(kappa_v * |G|) * Ghat`。
- `source_x/source_y` 始终保留原 participant 坐标。
- 新增 `x0/y0` 表示 V2 entropy marker 初态位置 `r0`；非 V2 模式下设为 participant anchor，避免 schema 语义空洞。
- `particles.x/y` 仍表示最终横向发射点 `rf` 或现有模式下的实际发射点。

## Public API 与配置变更

- 扩展 enum：
  - `DensityEvolutionMode::GradientResponse`
  - `FlowVelocitySamplerMode::GradientResponse`
  - 新增 `CooperFryeWeightMode::{None, MtCosh}`
- `BlastWaveConfig` 新增字段与默认值：
  - `gradientSigmaEm = 0.0`
  - `gradientSigmaDyn = 1.0`
  - `gradientDensityFloorFraction = 1.0e-4`
  - `gradientDensityCutoffFraction = 1.0e-6`
  - `gradientDisplacementMax = 1.5`
  - `gradientDisplacementKappa = 1.0`
  - `gradientDiffusionSigma = 0.0`
  - `gradientVMax = 0.75`
  - `gradientVelocityKappa = 1.0`
  - `debugGradientResponse = false`
  - `cooperFryeWeightMode = None`
- CLI/config 新增 key/flag：
  - `gradient-sigma-em`
  - `gradient-sigma-dyn`
  - `gradient-density-floor-fraction`
  - `gradient-density-cutoff-fraction`
  - `gradient-displacement-max`
  - `gradient-displacement-kappa`
  - `gradient-diffusion-sigma`
  - `gradient-vmax`
  - `gradient-velocity-kappa`
  - `debug-gradient-response` / `--no-debug-gradient-response`
  - `cooper-frye-weight <none|mt-cosh>`
- 配置校验：
  - 所有 sigma、fraction、kappa、Lmax 必须 finite 且非负。
  - `0 <= gradient-vmax < 1`。
  - `density-evolution = gradient-response` 必须配 `flow-velocity-sampler = gradient-response`。
  - `flow-velocity-sampler = gradient-response` 不能用于非 V2 density evolution。
  - 保留旧 `none` 和 `affine-gaussian` 行为不变。

- 修改 public headers：

  - `gradientMagnitude`

- `EmissionSampler.cpp`：
  - 保留现有 participant hotspot 与 density-field backend。
  - 新增 gradient-response backend，按上述 marker -> gradient -> `rf/vT` 流程生成 sites。
  - 继续使用现有 multiplicity 抽样函数。

- `FlowFieldModel`：

  - 增加 `evaluateFlowField(medium, site, params)` overload。
  - `GradientResponse` sampler 返回 site 中保存的横向速度，并设置 `betaT`、`phiB`、`rhoRaw=atanh(betaT)`。
- `RunOptions.cpp`：
  - 解析新 enum、新 flag、新 config key。
  - 更新 `--help` 和 config key 列表。
- ROOT schema：
  - `ParticleBranches` 新增 `x0`、`y0`、`emission_weight`。
