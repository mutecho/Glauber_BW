# Blast-Wave 数学物理公式流程说明

## 说明范围

本文按当前代码主线说明一次 `generate_blastwave_events` 运行中每一步的数学/物理定义，以及它在程序中的抽象操作。它描述的是当前实现，而不是完整流体动力学模型：

- 当前生成单一直接 `pi+`。
- 当前不是完整 Cooper-Frye freeze-out 超曲面采样；`cooper-frye-weight = mt-cosh` 只是一个可选权重代理。
- 默认链路是 `density-evolution = affine-gaussian`、`flow-velocity-sampler = covariance-ellipse`、`thermal-sampler = maxwell-juttner`。
- `density-evolution = none`、`flow-velocity-sampler = density-normal`、`flow-velocity-sampler = affine-effective`、`density/flow = gradient-response`、`v2{2}(pT)` 都是当前可选分支。

## 符号约定

- 横向坐标写作
  \[
  \mathbf r=(x,y)^T,\qquad \Delta \mathbf r=\mathbf r-\bar{\mathbf r}.
  \]
- 自然单位约定 \(c=1\)。长度单位是 `fm`，动量和质量单位是 `GeV`。
- participant 点云记为
  \[
  \{(\mathbf r_i,w_i)\}_{i=1}^{N_{\rm part}},
  \]
  当前 Glauber 生成器中 \(w_i=1\)。
- participant-plane 的短轴方向记为 \(\mathbf e_{\rm in}\)，长轴方向记为 \(\mathbf e_{\rm out}\)。代码里 `in-plane` 对应 covariance minor axis，`out-of-plane` 对应 major axis。
- 初态 participant 几何量使用下标 `0` 或 `participant`；freeze-out/发射阶段几何量使用下标 `f`。
- 横向流快度记为 \(\rho_t\)，横向速度大小记为 \(\beta_T=\tanh\rho_t\)。

## 总体抽象流程

当前 `generateEvent()` 的总流程可以抽象成：

```text
配置参数
  -> sampleParticipants()
  -> buildEventMedium()
  -> sampleEmissionSites()
  -> sample eta_s
  -> sample local thermal momentum p*
  -> evaluate flow velocity beta
  -> Lorentz boost p* -> p_lab
  -> accumulate EventInfo / ParticleRecord / ROOT payload
```

对应代码落点：

- 事件编排：`src/BlastWaveGenerator.cpp`
- participant 几何：`src/BlastWaveGeneratorGeometry.cpp`
- 介质构造：`src/EventMedium.cpp`
- 发射点抽样：`src/EmissionSampler.cpp`
- 流场：`src/FlowFieldModel.cpp`
- 热动量：`src/MaxwellJuttnerMomentumSampler.cpp`
- 共享观测量：`src/PhysicsUtils.cpp`
- ROOT 写出：`apps/generate_blastwave/RootEventFileWriter.cpp`

### 代码位置

- `apps/generate_blastwave_events.cpp`：程序入口，完成参数解析、事件循环和 writer 调用。
- `include/blastwave/BlastWaveGenerator.h`：`BlastWaveConfig`、`GeneratedEvent`、`EventInfo`、`ParticleRecord` 等主数据契约。
- `src/BlastWaveGenerator.cpp`：`BlastWaveGenerator::generateEvent()`，串起几何、介质、发射、动量、flow 和事件摘要。

## 1. Woods-Saxon 核子抽样与 participant 判定

### 数学/物理公式

单个核子的径向 Woods-Saxon 密度取

\[
\rho_{\rm WS}(r)\propto
\frac{1}{1+\exp((r-R)/a)},
\]

其中 \(R=\texttt{woodsSaxonRadius}\)，\(a=\texttt{woodsSaxonDiffuseness}\)。球坐标抽样的径向概率包含体积因子：

\[
P(r)\,dr \propto r^2 \rho_{\rm WS}(r)\,dr.
\]

两束核分别在横向 \(x\) 方向平移

\[
x_A=x-\frac{b}{2},\qquad x_B=x+\frac{b}{2},
\]

其中 \(b=\texttt{impactParameter}\)。两个核子发生一次几何碰撞的判据为

\[
d_T^2=(x_A-x_B)^2+(y_A-y_B)^2
\le \frac{\sigma_{NN}}{\pi}.
\]

至少发生一次碰撞的核子被标记为 participant。

当前中心度只是固定 \(b\) 的显示映射：

\[
{\rm centrality}
=\operatorname{clip}\left(
\frac{100\,b}{2R},0,100
\right).
\]

### 抽象操作

这一层把“两个带 Woods-Saxon 形状的核在固定冲量参数下几何相交”抽象成一个逐事件 participant 点云：

\[
\{(x_i,y_i,z_i,\text{nucleus\_id}_i)\}.
\]

后续所有初态几何、密度和发射锚点都从这个点云派生。

### 代码位置

- `src/BlastWaveGeneratorGeometry.cpp`：`sampleParticipants()` 实现两核抽样和 \(NN\) black-disk participant 判定。
- `src/BlastWaveGeneratorGeometry.cpp`：`sampleSingleNucleon()` 实现 Woods-Saxon 径向拒绝抽样和核子坐标平移。
- `src/PhysicsUtils.cpp`：`computeCentralityPercent()` 实现当前固定 \(b\) 的中心度显示映射。
- `include/blastwave/BlastWaveGenerator.h`：`BlastWaveConfig` 保存 `impactParameter`、`sigmaNN`、`woodsSaxonRadius`、`woodsSaxonDiffuseness` 等参数。

## 2. Participant covariance 椭圆与初态 \(\epsilon_2,\Psi_2\)

### 数学/物理公式

对横向点云计算加权中心：

\[
W=\sum_i w_i,\qquad
\bar x=\frac{1}{W}\sum_i w_i x_i,\qquad
\bar y=\frac{1}{W}\sum_i w_i y_i.
\]

二阶矩/covariance 为

\[
\Sigma=
\begin{pmatrix}
\sigma_x^2 & \sigma_{xy}\\
\sigma_{xy} & \sigma_y^2
\end{pmatrix},
\]

其中

\[
\sigma_x^2=\frac{1}{W}\sum_i w_i(x_i-\bar x)^2,
\]

\[
\sigma_y^2=\frac{1}{W}\sum_i w_i(y_i-\bar y)^2,
\]

\[
\sigma_{xy}=\frac{1}{W}\sum_i w_i(x_i-\bar x)(y_i-\bar y).
\]

代码使用的二阶 eccentricity 向量为

\[
E_x=\sigma_y^2-\sigma_x^2,\qquad
E_y=2\sigma_{xy}.
\]

participant eccentricity 和 participant-plane 角为

\[
\epsilon_2=
\frac{\sqrt{E_x^2+E_y^2}}{\sigma_x^2+\sigma_y^2},
\]

\[
\Psi_2=\frac{1}{2}\operatorname{atan2}(E_y,E_x).
\]

协方差矩阵本征值为

\[
\lambda_{\rm major/minor}
=\frac{1}{2}\left[
\operatorname{tr}\Sigma
\pm
\sqrt{(\sigma_x^2-\sigma_y^2)^2+4\sigma_{xy}^2}
\right],
\]

半轴长度为

\[
R_{\rm major/minor}=\sqrt{\lambda_{\rm major/minor}}.
\]

给定一个查询点 \(\mathbf r\)，协方差归一化法向和广义半径定义为

\[
\mathbf q=\Sigma^{-1}(\mathbf r-\bar{\mathbf r}),
\]

\[
\hat{\mathbf n}_{\Sigma}=\frac{\mathbf q}{|\mathbf q|},
\]

\[
\tilde r^2=(\mathbf r-\bar{\mathbf r})^T\Sigma^{-1}(\mathbf r-\bar{\mathbf r}).
\]

### 抽象操作

这一层把离散 participant 点云压缩成一个事件级椭圆几何代理：

- `centerX/centerY`：事件几何中心
- `eps2/psi2`：初态二阶各向异性
- `major/minor axis`：稳定的主轴基
- `inverse covariance`：供流场方向、归一化半径和 fallback 使用

重要语义：`events.eps2` 和 `events.psi2` 始终是初态 participant 几何，不会被 freeze-out 几何覆盖。

### 代码位置

- `include/blastwave/FlowFieldModel.h`：`FlowEllipseInfo` 定义 covariance 椭圆、主轴、逆协方差、`eps2` 和 `psi2`。
- `src/FlowFieldGeometry.cpp`：`computeFlowEllipseInfo()` 从加权横向点云恢复事件中心、covariance、主轴、半轴、\(\epsilon_2\)、\(\Psi_2\)。
- `src/EventMedium.cpp`：`finalizeEllipseFromCovariance()` 在 affine kernel covariance 加入后重新计算 freeze-out 椭圆派生量。
- `src/BlastWaveGenerator.cpp`：`generateEvent()` 把 `medium.participantGeometry.eps2/psi2` 写入 `EventInfo`。

## 3. Gaussian point-cloud 密度场

### 数学/物理公式

当前横向密度统一用 Gaussian point-cloud 表示。对支持点 \(\mathbf r_i\) 和二维核协方差 \(K\)，密度为

\[
s(\mathbf r)
=\sum_i w_i\,G_K(\mathbf r-\mathbf r_i),
\]

\[
G_K(\Delta\mathbf r)
=
\frac{1}{2\pi\sqrt{\det K}}
\exp\left[
-\frac{1}{2}
\Delta\mathbf r^T K^{-1}\Delta\mathbf r
\right].
\]

密度梯度解析计算：

\[
\nabla s(\mathbf r)
=
-\sum_i
w_i\,G_K(\mathbf r-\mathbf r_i)
K^{-1}(\mathbf r-\mathbf r_i).
\]

等方核时

\[
K=\sigma^2 I.
\]

带仿射演化的发射阶段允许使用完整二维核：

\[
K=
\begin{pmatrix}
K_{xx} & K_{xy}\\
K_{xy} & K_{yy}
\end{pmatrix}.
\]

### 抽象操作

这一层把离散点云变成可查询的连续场：

```text
support points + kernel covariance
  -> density s(x,y)
  -> gradient grad s(x,y)
```

这个接口同时服务于：

- 初态密度 `initialDensity`
- 发射阶段密度 `emissionDensity`
- gradient-response 的 marker 密度 `s_em`
- gradient-response 的动力学密度 `s_dyn`
- density-normal 和 affine-effective 中的密度法向
- ROOT debug density map

### 代码位置

- `include/blastwave/DensityFieldModel.h`：`WeightedTransversePoint`、`DensityField`、`DensityFieldSample` 以及 density evaluator 接口。
- `src/FlowFieldDensity.cpp`：`buildGaussianPointCloudDensityField()` 构造等方或完整 covariance kernel 的 Gaussian point-cloud。
- `src/FlowFieldDensity.cpp`：`evaluateDensityField()` 计算 \(s(x,y)\) 和解析 \(\nabla s(x,y)\)。
- `apps/generate_blastwave/RootEventFileWriter.cpp`：`fillDensityHistogramFromField()` 把解析密度场栅格化成 ROOT `TH2F` debug map。

## 4. EventMedium：从初态点云到发射阶段介质

### 代码位置

- `include/blastwave/EventMedium.h`：`EventMediumParameters`、`EventMedium`、`AffineEffectiveClosure` 和 `DensityEvolutionMode`。
- `src/BlastWaveGeneratorGeometry.cpp`：`buildMedium()` 把 generator config 和 participant 点云转成 `EventMediumParameters`。
- `src/EventMedium.cpp`：`buildEventMedium()` 是 `none`、`affine-gaussian`、`gradient-response` 三条介质路径的调度入口。
- `src/EventMedium.cpp`：`computeSupportDensityScale()` 为 gradient-response 的 density floor/cutoff 提供事件级 density scale。

### 4.1 `density-evolution = none`

#### 数学/物理公式

无演化路径直接令

\[
s_0=s_{\rm marker}=s_{\rm dyn}=s_f,
\]

\[
\Sigma_f=\Sigma_{\rm participant}.
\]

#### 抽象操作

这是旧路径/对照路径：participant 几何、发射几何和密度场保持同一语义。

#### 代码位置

- `src/EventMedium.cpp`：`buildEventMedium()` 的 `DensityEvolutionMode::None` 分支把 `initialDensity` 复制到 marker、dynamics 和 emission density，并保留 participant geometry。
- `src/BlastWaveGeneratorSampling.cpp`：`sampleEventEmissionSites()` 在该模式下选择 `EmissionSamplerMode::ParticipantHotspot`。

### 4.2 `density-evolution = affine-gaussian`

#### 数学/物理公式

先在 participant-plane 主轴基中分解点坐标：

\[
\Delta\mathbf r_i=\mathbf r_i-\bar{\mathbf r},
\]

\[
x_{{\rm in},i}=\Delta\mathbf r_i\cdot\mathbf e_{\rm in},
\qquad
x_{{\rm out},i}=\Delta\mathbf r_i\cdot\mathbf e_{\rm out}.
\]

对短轴/长轴做不同仿射膨胀：

\[
\mathbf r_i'
=\bar{\mathbf r}
+\lambda_{\rm in}\,x_{{\rm in},i}\mathbf e_{\rm in}
+\lambda_{\rm out}\,x_{{\rm out},i}\mathbf e_{\rm out},
\]

其中

\[
\lambda_{\rm in}=\texttt{affine-lambda-in},
\qquad
\lambda_{\rm out}=\texttt{affine-lambda-out}.
\]

Gaussian kernel 也被同一个线性映射拉伸，并额外加入演化平滑：

\[
A=\lambda_{\rm in}\,\mathbf e_{\rm in}\mathbf e_{\rm in}^T
+\lambda_{\rm out}\,\mathbf e_{\rm out}\mathbf e_{\rm out}^T,
\]

\[
K_f=A(\sigma_{\rm dep}^2 I)A^T+\sigma_{\rm evo}^2 I.
\]

其中

\[
\sigma_{\rm dep}=\texttt{flow-density-sigma},
\qquad
\sigma_{\rm evo}=\texttt{affine-sigma-evo}.
\]

发射密度为

\[
s_f(\mathbf r)=\sum_i w_i\,G_{K_f}(\mathbf r-\mathbf r_i').
\]

发射几何代理把 transformed support cloud 的 covariance 和 kernel covariance 加在一起：

\[
\Sigma_f={\rm Cov}(\{\mathbf r_i'\})+K_f.
\]

若启用 affine-effective，几何闭合量从初末态半轴恢复：

\[
G_{\rm in}=\frac{\sigma_{{\rm in},f}}{\sigma_{{\rm in},0}},
\qquad
G_{\rm out}=\frac{\sigma_{{\rm out},f}}{\sigma_{{\rm out},0}},
\]

\[
\Lambda_{\rm in}=\ln G_{\rm in},
\qquad
\Lambda_{\rm out}=\ln G_{\rm out},
\]

\[
\bar\Lambda=\frac{1}{2}(\Lambda_{\rm in}+\Lambda_{\rm out}),
\qquad
\Delta\Lambda=\frac{1}{2}(\Lambda_{\rm in}-\Lambda_{\rm out}),
\]

\[
H_{\rm in}^{\rm eff}=\frac{\Lambda_{\rm in}}{\Delta\tau_{\rm ref}},
\qquad
H_{\rm out}^{\rm eff}=\frac{\Lambda_{\rm out}}{\Delta\tau_{\rm ref}}.
\]

#### 抽象操作

这一层把初态 participant cloud 变成一个更宽、更椭圆响应后的 freeze-out density：

```text
participant points
  -> affine axis transform
  -> anisotropic Gaussian kernel
  -> emissionDensity s_f
  -> emissionGeometry Sigma_f
  -> optional affine-effective closure
```

注意：`participantGeometry` 仍保留初态语义；`emissionGeometry` 是 freeze-out 诊断和流场查询用的几何代理。

#### 代码位置

- `src/EventMedium.cpp`：`applyAffineResponse()` 实现 participant-plane 中的 \(\lambda_{\rm in/out}\) 仿射变换。
- `src/EventMedium.cpp`：`computeAffineKernelCovariance()` 实现 \(K_f=A\sigma_{\rm dep}^2IA^T+\sigma_{\rm evo}^2I\)。
- `src/EventMedium.cpp`：`buildEventMedium()` 的 `DensityEvolutionMode::AffineGaussianResponse` 分支构造 `emissionDensity`、`emissionGeometry`。
- `src/EventMedium.cpp`：`buildAffineEffectiveClosure()` 从初末态半轴恢复 \(G_{\rm in/out}\)、\(\Lambda_{\rm in/out}\)、\(\bar\Lambda\)、\(\Delta\Lambda\)。
- `include/blastwave/BlastWaveGenerator.h`：`BlastWaveConfig` 保存 `affineLambdaIn`、`affineLambdaOut`、`affineSigmaEvo` 和 affine-effective 相关参数。

### 4.3 `density-evolution = gradient-response`

#### 数学/物理公式

该路径构造三套密度：

\[
s_0:\quad \sigma_0=\texttt{flow-density-sigma},
\]

\[
s_{\rm em}:\quad
\sigma_{\rm em}^{\rm eff}
=\sqrt{\sigma_0^2+\texttt{gradient-sigma-em}^2},
\]

\[
s_{\rm dyn}:\quad
\sigma_{\rm dyn}^{\rm eff}
=\sqrt{\sigma_0^2+\texttt{gradient-sigma-dyn}^2}.
\]

`markerDensity = s_em` 用于抽 marker 初态位置，`dynamicsDensity = s_dyn` 用于给出位移和横向速度。生成时会在接受的最终发射点上重新计算 freeze-out 几何：
\[
\Sigma_f={\rm Cov}(\{\mathbf r_{f,j}\}).
\]

#### 抽象操作

该路径把“介质演化”推迟到逐粒子的 gradient response：

```text
s_em: where marker r0 is sampled
s_dyn: which local gradient pushes r0 to rf and sets beta_T
accepted rf cloud: later determines eps2_f / psi2_f
```

该路径必须和 `flow-velocity-sampler = gradient-response` 成对使用。

#### 代码位置

- `src/EventMedium.cpp`：`buildEventMedium()` 的 `DensityEvolutionMode::GradientResponse` 分支构造 `initialDensity`、`markerDensity`、`dynamicsDensity`。
- `src/BlastWaveGeneratorSampling.cpp`：`sampleEventEmissionSites()` 在该模式下选择 `EmissionSamplerMode::GradientResponse`。
- `src/BlastWaveGeneratorValidation.cpp`：`validateConfig()` 校验 gradient medium 和 gradient flow 必须成对使用，并校验相关参数范围。
- `src/BlastWaveGenerator.cpp`：`generateEvent()` 对 gradient-response 的 accepted final emission points 重新计算 `emissionGeometry`。

## 5. 每个 participant 的 multiplicity

### 数学/物理公式

每个 participant 的候选粒子数使用 Gamma-Poisson 形式的负二项分布。先抽

\[
\lambda_i\sim \Gamma(k,\theta=\mu/k),
\]

再抽

\[
n_i\sim {\rm Poisson}(\lambda_i).
\]

于是

\[
\mathbb E[n_i]=\mu,
\qquad
{\rm Var}(n_i)=\mu+\frac{\mu^2}{k}.
\]

其中 \(\mu=\texttt{nbd-mu}\)，\(k=\texttt{nbd-k}\)。

### 抽象操作

这一层把每个 participant source anchor 转换成若干个候选 emission site。当前粒子总数 \(N_{\rm ch}\) 就是所有被接受 emission site 的数量。

### 代码位置

- `include/blastwave/EmissionSampler.h`：`EmissionParameters` 保存 `nbdMu`、`nbdK`。
- `src/EmissionSampler.cpp`：`sampleHotspotMultiplicity()` 实现 Gamma-Poisson 负二项 multiplicity 抽样。
- `src/EmissionSampler.cpp`：三个 emission backend 都调用 `sampleHotspotMultiplicity()` 为每个 participant 产生候选粒子数。

## 6. 横向 emission site 抽样

### 代码位置

- `include/blastwave/EmissionSampler.h`：`EmissionSite` 定义 `initialPosition`、`position`、`sourceAnchor`、gradient-response velocity 和 `emissionWeight`。
- `include/blastwave/EmissionSampler.h`：`EmissionSamplerMode` 与 `EmissionParameters` 定义三种 backend 的公共配置。
- `src/BlastWaveGeneratorSampling.cpp`：`sampleEventEmissionSites()` 根据 `densityEvolutionMode` 选择 emission backend。
- `src/EmissionSampler.cpp`：`sampleEmissionSites()` 是 backend dispatch 入口。

### 6.1 `ParticipantHotspot`

#### 数学/物理公式

无演化旧路径下，marker 初态位置就是 participant anchor：

\[
\mathbf r_0=\mathbf r_i.
\]

最终发射位置加上等方 Gaussian smear：

\[
\mathbf r_f=\mathbf r_i+\boldsymbol\xi,
\qquad
\boldsymbol\xi\sim\mathcal N(0,\sigma_{\rm smear}^2 I).
\]

#### 抽象操作

```text
participant anchor -> local hotspot smear -> final emission position
```

写出语义：

- `source_x/source_y = participant anchor`
- `x0/y0 = participant anchor`
- `x/y = smeared final emission point`

#### 代码位置

- `src/EmissionSampler.cpp`：`sampleParticipantHotspotEmissionSites()` 实现旧 hotspot 发射路径。
- `src/EmissionSampler.cpp`：`sampleSmearedPosition()` 实现 \(\mathbf r_i+\mathcal N(0,\sigma_{\rm smear}^2I)\)。
- `src/BlastWaveGeneratorSampling.cpp`：`sampleEventEmissionSites()` 在 `density-evolution = none` 下选择该 backend。

### 6.2 `DensityField`

#### 数学/物理公式

`affine-gaussian` 默认使用该 backend。对第 \(i\) 个 participant，使用第 \(i\) 个 freeze-out support center \(\mathbf r'_i\) 和共享 kernel \(K_f\) 抽样：
\[
\mathbf r_f\sim\mathcal N(\mathbf r'_i,K_f).
\]

marker 初态仍记录原 participant：

\[
\mathbf r_0=\mathbf r_i.
\]

#### 抽象操作

```text
participant anchor
  -> corresponding freeze-out support center
  -> sample from emission density kernel
  -> final emission position
```

写出语义：

- `source_x/source_y = participant anchor`
- `x0/y0 = participant anchor`
- `x/y = sampled freeze-out position`

#### 代码位置

- `src/EmissionSampler.cpp`：`sampleDensityFieldEmissionSites()` 从 `medium.emissionDensity.supportPoints` 和共享 kernel 抽 freeze-out emission point。
- `src/EmissionSampler.cpp`：`buildKernelSampler()` 对二维 kernel covariance 做 Cholesky 形式采样准备。
- `src/EmissionSampler.cpp`：`sampleKernelPosition()` 执行二维 Gaussian kernel 抽样。
- `src/BlastWaveGeneratorSampling.cpp`：`sampleEventEmissionSites()` 在 `density-evolution = affine-gaussian` 下选择该 backend。

### 6.3 `GradientResponse`

#### 数学/物理公式

先从 marker density 对应 Gaussian component 抽 marker：

\[
\mathbf r_0\sim \mathcal N(\mathbf r_i,K_{\rm em}).
\]

若

\[
s_{\rm em}(\mathbf r_0)
< f_{\rm cutoff}\,s_{\rm em}^{\rm scale},
\]

则重新抽样，直到达到重试上限。这里

\[
f_{\rm cutoff}=\texttt{gradient-density-cutoff-fraction}.
\]

在 \(\mathbf r_0\) 查询动力学密度梯度，并构造向外响应向量：

\[
\mathbf g
=
\frac{-\nabla s_{\rm dyn}(\mathbf r_0)}
{\max\left(\epsilon,\ s_{\rm dyn}(\mathbf r_0)+f_{\rm floor}s_{\rm dyn}^{\rm scale}\right)}.
\]

其中

\[
f_{\rm floor}=\texttt{gradient-density-floor-fraction}.
\]

方向为

\[
\hat{\mathbf g}=\frac{\mathbf g}{|\mathbf g|}.
\]

确定性位移长度为

\[
\ell_{\rm disp}
=d_{\max}\tanh(\kappa_{\rm disp}|\mathbf g|),
\]

其中

\[
d_{\max}=\texttt{gradient-displacement-max},
\qquad
\kappa_{\rm disp}=\texttt{gradient-displacement-kappa}.
\]

最终位移为

\[
\Delta\mathbf r
=\ell_{\rm disp}\hat{\mathbf g}
+\boldsymbol\xi_{\rm diff},
\]

\[
\boldsymbol\xi_{\rm diff}\sim
\mathcal N(0,\sigma_{\rm diff}^2 I).
\]

最终发射位置为

\[
\mathbf r_f=\mathbf r_0+\Delta\mathbf r.
\]

该 site 的横向速度大小为

\[
\beta_T
=v_{\max}\tanh(\kappa_v|\mathbf g|),
\]

\[
\boldsymbol\beta_T=\beta_T\hat{\mathbf g}.
\]

其中

\[
v_{\max}=\texttt{gradient-vmax},
\qquad
\kappa_v=\texttt{gradient-velocity-kappa}.
\]

#### 抽象操作

```text
participant anchor
  -> sample marker r0 from s_em
  -> query -grad(s_dyn)/(s_dyn + floor)
  -> bounded displacement r0 -> rf
  -> bounded transverse beta_T saved on EmissionSite
```

写出语义：

- `source_x/source_y = participant anchor`
- `x0/y0 = marker initial position r0`
- `x/y = final emission position rf`
- `emission_weight = site weight before optional particle-level mt-cosh multiplier`

#### 代码位置

## 7. 纵向空间快度与 freeze-out 坐标

### 数学/物理公式

若 `eta-plateau = L` 且 `sigma-eta = \sigma_\eta`，当前空间快度分布可以理解为中心平台加 Gaussian 尾部：

\[
P(\eta_s)\propto
\begin{cases}
1, & |\eta_s|\le L,\\
\exp\left[-\frac{(|\eta_s|-L)^2}{2\sigma_\eta^2}\right], & |\eta_s|>L.
\end{cases}
\]

实现中先用面积比选择平台或尾部：

\[
P_{\rm plateau}
=
\frac{L}{L+\sigma_\eta\sqrt{\pi/2}}.
\]

固定 proper time 为

\[
\tau=\tau_0.
\]

发射点四坐标写成

\[
t=\tau_0\cosh\eta_s,
\qquad
z=\tau_0\sinh\eta_s,
\]

\[
x=x_f,\qquad y=y_f.
\]

### 抽象操作

横向 emission site 只提供 \(x_f,y_f\)。这一层把它补成一个 boost-invariant 风格的 freeze-out spacetime point：

```text
(x_f, y_f) + eta_s + tau0 -> (t, x, y, z)
```

## 8. 局域静止系热动量

### 数学/物理公式

默认 `maxwell-juttner` 模式抽动量大小 \(p^*=|\mathbf p^*|\)，概率密度取截断形式

\[
f(p^*)\propto
(p^*)^2
\exp\left[-\frac{\sqrt{(p^*)^2+m^2}}{T}\right],
\qquad
0\le p^*\le p_{\max}.
\]

代码用数值表构造 CDF：

\[
F(p)=
\frac{\int_0^p f(p')\,dp'}
{\int_0^{p_{\max}} f(p')\,dp'}.
\]

然后用 inverse-CDF 插值抽样 \(p^*\)。

动量方向各向同性：

\[
\cos\theta\sim U(-1,1),
\qquad
\phi\sim U(0,2\pi).
\]

局域静止系四动量：

\[
p_x^*=p^*\sin\theta\cos\phi,
\]

\[
p_y^*=p^*\sin\theta\sin\phi,
\]

\[
p_z^*=p^*\cos\theta,
\]

\[
E^*=\sqrt{m^2+(p^*)^2}.
\]

兼容模式 `gamma` 使用

\[
p^*\sim \Gamma(k=3,\theta=T).
\]

### 抽象操作

这一层只负责“局域静止系热动量”。它不知道事件几何、流场和 ROOT 输出：

```text
mass + temperature + sampler mode -> local four momentum p*
```

### 代码位置

- `include/blastwave/MaxwellJuttnerMomentumSampler.h`：Maxwell-Juttner inverse-CDF sampler 对外接口。
- `src/MaxwellJuttnerMomentumSampler.cpp`：`thermalWeight()`、构造函数和 `sample()` 实现查表积分、归一化 CDF 和插值抽样。
- `src/BlastWaveGenerator.cpp`：`BlastWaveGenerator` 构造函数在默认 `maxwell-juttner` 模式下初始化 sampler table。
- `src/BlastWaveGeneratorSampling.cpp`：`sampleThermalMomentum()` 抽局域静止系四动量，并保留 `gamma` 兼容模式。

### 9.1 通用速度转换

给定横向流快度 \(\rho_t\)，横向速度为

\[
\beta_T=\tanh\rho_t.
\]

默认数值保护对 covariance/density-normal 路径使用

\[
\beta_T\le 0.95.
\]

横向方向角为

\[
\phi_B=\operatorname{atan2}(n_y,n_x),
\]

于是

\[
\beta_x=\beta_T\cos\phi_B,
\qquad
\beta_y=\beta_T\sin\phi_B.
\]

随后叠加 Bjorken 纵向速度。若横向 sampler 给出 \((\beta_x^T,\beta_y^T)\)，生成器实际 boost 使用

\[
\beta_x=\frac{\beta_x^T}{\cosh\eta_s},
\qquad
\beta_y=\frac{\beta_y^T}{\cosh\eta_s},
\qquad
\beta_z=\tanh\eta_s.
\]

这样总速度满足

\[
\beta^2
=
\frac{(\beta_T)^2}{\cosh^2\eta_s}
+\tanh^2\eta_s
<1
\]

只要 \(\beta_T<1\)。

#### 代码位置

### 9.2 `flow-velocity-sampler = covariance-ellipse`

#### 数学/物理公式

先用发射阶段几何 \(\Sigma_f\) 计算

\[
\mathbf q=\Sigma_f^{-1}(\mathbf r_f-\bar{\mathbf r}_f),
\]

\[
\hat{\mathbf n}_{\Sigma}=\frac{\mathbf q}{|\mathbf q|},
\]

\[
\tilde r^2=(\mathbf r_f-\bar{\mathbf r}_f)^T
\Sigma_f^{-1}
(\mathbf r_f-\bar{\mathbf r}_f).
\]

若当前是 `affine-gaussian`，二阶响应振幅来自初态 participant 几何：

\[
a_2=\kappa_2\,\epsilon_{2,0}.
\]

流快度为

\[
\rho_t
=
\rho_0\,\tilde r^{\alpha}
\exp\left[
2a_2\cos 2(\phi_B-\Psi_{2,0})
\right],
\]

其中

\[
\alpha=\texttt{flow-power}.
\]

这里 \(\phi_B\) 是 lab frame 中的 covariance-normal 方向角。

若是 `density-evolution = none` 的旧比较路径，则代码使用

\[
x\rho_t=\tilde r^\alpha \left[\rho_0+a_2\cos(2\phi_B')\right]
\]

其中 \(\phi_B'\) 是在主轴投影空间中恢复的角。

#### 抽象操作

```text
emissionGeometry covariance
  -> covariance normal direction
  -> generalized radius rTilde
  -> rho_t radial profile with optional second-order modulation
  -> beta_T and transverse beta vector
```

#### 代码位置

- `src/FlowFieldModel.cpp`：`sampleEllipseMetric()` 计算 \(\mathbf q\)、\(\tilde r\)、covariance normal 和主轴投影。
- `src/FlowFieldModel.cpp`：`computeAffineKappaModulation()` 实现 \(a_2=\kappa_2\epsilon_{2,0}\) 的 V1a 二阶响应因子。

### 9.3 `flow-velocity-sampler = density-normal`

#### 数学/物理公式

先在发射密度上取向外 density normal：

\[
\hat{\mathbf n}_{s}
=
-\frac{\nabla s_f(\mathbf r_f)}
{|\nabla s_f(\mathbf r_f)|}.
\]

若局部密度太平或梯度数值退化，则 fallback 到 covariance normal：

\[
\hat{\mathbf n}_{s}\rightarrow \hat{\mathbf n}_{\Sigma}.
\]

流强度默认只依赖广义半径：

\[
\rho_t=\rho_0\tilde r^\alpha.
\]

若显式启用 `density-normal-kappa-compensation`，并且处于 `affine-gaussian + density-normal`，则额外乘上 V1a 二阶响应：

\[
\rho_t
=
\rho_0\tilde r^\alpha
\exp\left[
2\kappa_2\epsilon_{2,0}
\cos 2(\phi_B-\Psi_{2,0})
\right].
\]

### 抽象操作

```text
emissionDensity gradient
  -> outward density normal
  -> covariance normal fallback if flat
  -> rho0 * rTilde^flowPower
  -> optional kappa2 compensation
  -> beta vector
```

这个 sampler 的物理意图是：方向各向异性主要来自 freeze-out density 的几何梯度，而不是显式强行叠加的解析二阶项。

### 9.4 `flow-velocity-sampler = affine-effective`

该 sampler 只允许配合 `density-evolution = affine-gaussian`。它先用第 4.2 节的闭合量恢复

\[
H_{\rm in}^{\rm eff},\qquad H_{\rm out}^{\rm eff}.
\]

令

\[
\bar H=\frac{1}{2}
(H_{\rm in}^{\rm eff}+H_{\rm out}^{\rm eff}),
\]

\[
\bar R=\sqrt{\sigma_{{\rm in},f}\sigma_{{\rm out},f}}.
\]

并在 freeze-out 主轴基中写

\[
x'=(\mathbf r_f-\bar{\mathbf r}_f)\cdot\mathbf e_{\rm in},
\]

\[
y'=(\mathbf r_f-\bar{\mathbf r}_f)\cdot\mathbf e_{\rm out}.
\]

#### `affine-effective-mode = additive-rho`

该模式保留 density-normal 方向，并把 affine 几何修正加到 \(\rho_0\) baseline 上。

局域 affine 几何速度代理：

\[
u_{\rm geom}
=
\kappa_{\rm flow}\,\tilde r^\alpha
\sqrt{
(H_{\rm in}^{\rm eff}x')^2
+
(H_{\rm out}^{\rm eff}y')^2
}.
\]

各向同性参考项：

\[
u_{\rm iso}
=
\kappa_{\rm flow}\,\tilde r^\alpha
\bar H\,\tilde r\,\bar R.
\]

速度代理先被 `affine-u-max` 截断：

\[
u_{\rm geom}^{\rm clip}=\min(u_{\rm geom},u_{\max}),
\]

\[
u_{\rm iso}^{\rm clip}=\min(u_{\rm iso},u_{\max}).
\]

几何修正写成流快度差：

\[
\Delta\rho_{\rm geom}
=
\operatorname{atanh}(u_{\rm geom}^{\rm clip})
-
\operatorname{atanh}(u_{\rm iso}^{\rm clip}).
\]

最终

\[
\rho_t
=
\max\left(
0,\,
\rho_0\tilde r^\alpha+\Delta\rho_{\rm geom}
\right),
\]

\[
\beta_T
=
\min\left(u_{\max},\tanh\rho_t\right).
\]

方向仍是 density-normal 方向。

#### `affine-effective-mode = full-tensor`

该模式直接把主轴膨胀率映射成横向速度张量：

\[
u_{\rm in}'
=
\kappa_{\rm flow}\tilde r^\alpha
H_{\rm in}^{\rm eff}x',
\]

\[
u_{\rm out}'
=
\kappa_{\rm flow}\tilde r^\alpha
H_{\rm out}^{\rm eff}y'.
\]

旋回 lab frame：

\[
\boldsymbol\beta_T
=
u_{\rm in}'\mathbf e_{\rm in}
+
u_{\rm out}'\mathbf e_{\rm out}.
\]

若

\[
|\boldsymbol\beta_T|>u_{\max},
\]

则整体缩放到 \(u_{\max}\)。该模式中 `rho0` 不参与速度公式。

#### 抽象操作

```text
affine-gaussian initial/final semi-axes
  -> effective expansion rates H_in/out
  -> additive-rho: rho0 baseline + geometry rapidity correction
  -> full-tensor: principal-axis velocity tensor
  -> clipped beta vector
```

`affine-kappa-aniso` 当前只是 legacy/no-op 兼容参数。

#### 代码位置

- `src/EventMedium.cpp`：`buildAffineEffectiveClosure()` 生成 affine-effective 所需的初末态半轴闭合量。
- `src/FlowFieldModel.cpp`：`computeAffineEffectiveSharedTerms()` 计算 \(H_{\rm in/out}^{\rm eff}\)、\(\bar H\)、\(\bar R\)。
- `src/FlowFieldModel.cpp`：`evaluateAffineEffectiveAdditiveRho()` 实现 `additive-rho` 公式。
- `src/FlowFieldModel.cpp`：`evaluateAffineEffectiveFullTensor()` 实现 `full-tensor` 公式。

## 13. 差分 `v2{2}(pT)` 分析

### 数学/物理公式

该分析是可选 payload，配置 `v2pt-bins` 后在生成时做，也可用 `analyze_blastwave_v2pt` 后处理。

每个事件中，参考粒子是该事件所有粒子。单位 track weight 固定为 1，当前故意忽略 `particles.emission_weight`。令

\[
Q_2=\sum_{\rm ref} e^{2i\phi}.
\]

对一个 \(p_T\) bin，POI Q-vector 为

\[
P_2(p_T{\rm\ bin})
=
\sum_{\rm POI\ in\ bin}e^{2i\phi}.
\]

积分参考 cumulant：

\[
c_2\{2\}
=
\frac{
\sum_{\rm events}\left(|Q_2|^2-M\right)
}{
\sum_{\rm events}M(M-1)
},
\]

其中 \(M\) 是事件粒子数。

差分 cumulant：

\[
d_2\{2\}(p_T)
=
\frac{
\sum_{\rm events}
\left(
{\rm Re}\left[P_2(p_T)Q_2^*\right]-m_{p_T}
\right)
}{
\sum_{\rm events}m_{p_T}(M-1)
},
\]

其中 \(m_{p_T}\) 是该事件中落入该 \(p_T\) bin 的 POI 数。

最终

\[
v_2\{2\}(p_T)
=
\frac{d_2\{2\}(p_T)}
{\sqrt{c_2\{2\}}}.
\]

误差使用 leave-one-event-out jackknife。若共有 \(N\) 个 jackknife 样本，记第 \(a\) 个 leave-one-out 结果为 \(v_a\)，均值为 \(\bar v\)，则

\[
\sigma_v
=
\sqrt{
\frac{N-1}{N}
\sum_{a=1}^{N}(v_a-\bar v)^2
}.
\]

### 抽象操作

```text
particles tree or live particles
  -> unit-weight Q2 and P2 accumulators
  -> integrated c2{2}
  -> differential d2{2}(pT)
  -> v2{2}(pT) histogram + jackknife errors
```

它和 `events.v2` 是不同对象：

- `events.v2` 是逐事件 Q-vector 模长摘要，默认可被 `emission_weight` 影响。
- `v2{2}(pT)` 是跨事件二粒子 cumulant 差分分析，当前固定单位 track weight。

## 15. 端到端物理抽象总结

从物理抽象上，当前生成器做的是：

1. 用 Woods-Saxon + black-disk \(NN\) 碰撞判据得到逐事件 participant 初态。
2. 用 participant covariance 得到 \(\epsilon_{2,0}\)、\(\Psi_{2,0}\)、主轴和广义半径度量。
3. 用 Gaussian point-cloud 构造连续横向密度。
4. 根据 `density-evolution` 把初态密度变成发射阶段密度，或在 gradient-response 中把演化推到逐粒子响应。
5. 根据 participant multiplicity 产生 emission sites。
6. 对每个 site 抽 \(\eta_s\)、局域热动量和横向流速度。
7. 用 Lorentz boost 得到 lab-frame 粒子四动量。
8. 从粒子和发射点云回填事件级几何、源尺寸和末态 flow 摘要。
9. 把事件、participant、粒子和可选 QA/analysis 对象写入 ROOT。

因此代码里的核心抽象不是“完整 hydrodynamic evolution”，而是：

\[
\text{participant geometry}
\rightarrow
\text{event medium}
\rightarrow
\text{emission sites}
\rightarrow
\text{local thermal momentum + flow boost}
\rightarrow
\text{ROOT observables}.
\]

### 代码位置

- `apps/generate_blastwave_events.cpp`：端到端运行的最外层事件循环。
- `src/BlastWaveGenerator.cpp`：单事件物理链路的核心编排。
- `apps/generate_blastwave/RootEventFileWriter.cpp`：端到端输出契约落盘。
- `apps/qa_blastwave_output.cpp`：端到端输出契约的独立读取和一致性校验。
