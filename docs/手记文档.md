## 简介
这是一个用 C++17 和 ROOT 实现的小型 Blast-Wave 事例生成器。它的目标不是做完整重离子末态模拟，而是在一个足够小、足够清晰、可继续扩展的代码骨架里，完成以下事情：

- 用 Monte Carlo Glauber 产生逐事例的 participant 几何涨落
- 在 participant hotspot 或演化后的 freeze-out density 上做简化的 blast-wave 发射
- 通过 ROOT 输出事件树、participant 树、粒子树和基础 QA 直方图
- 提供一个独立 QA 程序检查输出是否合法、接口是否稳定

目前末态粒子只有 `pi+`。

## 项目主要思路：

1. Glauber Model 采样核内核子分布
2. 给定 impact parameter，对两个核做横向偏移，确定 overlap
3. overlap 内，如果来自两核的核子横向距离满足 $d_T^2 \leq \sigma_{NN}/\pi$，则认为发生碰撞并成为 participant
4. participant 相当于受损核子，目前权重为 `1`，有几个分支：
   1. `density-evolution = none`：participant 直接作为发射源锚点
   2. `density-evolution = affine-gaussian`：默认路径，participant 先形成初态 density，再经过固定参数仿射膨胀和平滑，得到 freeze-out density
   3. `density-evolution = gradient-response`：V2 opt-in 路径，从 marker density 抽初态 marker，再用 dynamics density 的梯度生成末态位移和横向速度
5. 在做好的源中进行末态粒子生成，其中：
   1. 发射粒子数由 NBD（负二项分布）确定，代码中用 Gamma-Poisson 等价抽样，参数为 `nbd-mu` 和 `nbd-k`
   2. 粒子的横向发射位置由当前发射 backend 决定：
      1. `none`：participant 位置 + Gaussian smear
      2. 默认 `affine-gaussian`：从 freeze-out density 的 Gaussian component 抽样
      3. V2 `gradient-response`：先抽 marker 初态位置 `r0`，再由密度梯度得到末态位置 `rf`
   3. 粒子的纵向位置由 $\eta_s$ 分布和固定 $\tau_0$ 给出
   4. 粒子速度（动量）由两部分确定：
      1. 流体元速度
      2. 流体元静止系中粒子热动量
      3. 后者 boost 到实验室系中得到粒子四动量

## 具体实现

### 流体元

#### 位置

- 横向位置由发射源 + Gaussian 展宽决定，但“发射源”取决于介质演化模式
- `density-evolution = none` 时，发射源就是原始 participant，展宽宽度由 `smear` 控制
- 默认 `density-evolution = affine-gaussian` 时，先由 participant 构造初态 Gaussian point-cloud density $s_0$，再在 participant plane 中做默认仿射响应：
  - 短轴 / in-plane 方向默认膨胀系数为 `1.20`
  - 长轴 / out-of-plane 方向默认膨胀系数为 `1.05`
  - 再加默认 `0.5 fm` 的 evolution smoothing
  - 得到的 `emissionDensity` 是实际抽横向发射点的 freeze-out density
- V2 `gradient-response` 时，代码区分三套 density：
  - `s0`：初始 participant density
  - `s_em`：抽 marker 初态位置 `r0`
  - `s_dyn`：计算 $-\nabla s/(s+\mathrm{floor})$，给出位移和横向速度
- 纵向位置由给定的 $\eta_s$ 分布决定：
  - 中间是宽度 `eta-plateau` 的平台
  - 两侧是宽度 `sigma-eta` 的 Gaussian tail
  - 代码用固定 proper time `tau0` 写出
    - $t = \tau_0 \cosh \eta_s$
    - $z = \tau_0 \sinh \eta_s$

#### 速度

- 横向速度场的实现：
  1. 默认 `flow-velocity-sampler = covariance-ellipse`
     - 先从当前 `emissionGeometry` 恢复事件级协方差椭圆
     - 对每个发射点计算相对事件中心的 $\Delta r$
     - 用 $\Sigma^{-1}\Delta r$ 得到椭圆法向方向
     - 归一化半径为 $\tilde r = \sqrt{\Delta r^T\Sigma^{-1}\Delta r}$
     - 默认 V1a 下横向流快度为
       $$
       \rho_T = \rho_0 \tilde r^m
       \exp\left(2 a_2 \cos 2(\phi_B-\psi_{2,\mathrm{initial}})\right)
       $$
       其中 $a_2 = \kappa_2 \epsilon_{2,\mathrm{initial}}$
     - 最后取 $\beta_T = \tanh(\max(0,\rho_T))$，并限制在 `0.95` 以下
  2. 可选 `flow-velocity-sampler = density-normal`
     - 方向优先取 `emissionDensity` 的外法向 $-\nabla s/|\nabla s|$
     - 如果梯度太平或数值退化，则回退到协方差椭圆法向
     - 默认速度大小只由 `rho0` 和 `flow-power` 给出，即 `rhoRaw = rho0 * \tilde r^m`
     - 若显式开启 `density-normal-kappa-compensation`，才恢复现有的 `kappa2` 指数补偿公式
  3. V2 `flow-velocity-sampler = gradient-response`
     - 不再重新查询椭圆或 density-normal
     - 直接使用发射点生成阶段保存的 `betaTX/betaTY`
     - 必须和 `density-evolution = gradient-response` 同时启用，否则配置校验会报错
- 纵向速度场的实现：
  - 使用 Bjorken-like 纵向流
  - $\beta_z = \tanh\eta_s$
  - 横向速度在三维组合时除以 $\cosh\eta_s$
  - 因此代码中实际 boost 用的是
    - $\beta_x = \beta_{T,x}/\cosh\eta_s$
    - $\beta_y = \beta_{T,y}/\cosh\eta_s$
    - $\beta_z = \tanh\eta_s$

### 粒子

#### 位置

- `particles.x/y` 是最终横向发射位置
- `particles.z/t` 由 $\tau_0$ 和 $\eta_s$ 给出
- `particles.source_x/source_y` 保留原始 participant 坐标
- `particles.x0/y0` 记录初态 marker 位置：
  - 非 V2 模式下是 participant anchor
  - V2 模式下是抽到的 `r0`
- `events.r2_0`、`events.r2_f`、`events.r2_ratio` 由粒子级 `x0/y0` 和最终 `x/y` 重新计算得到，用来检查横向源是否真的发生膨胀

#### 速度

- 局域静止系中的热动量方向是各向同性的
- 热动量大小默认由 Maxwell-Juttner 分布给出，也就是相对论版本的 Maxwell-Boltzmann 热分布：
  $$
  w(p) \propto p^2 \exp\left(-\frac{\sqrt{p^2+m^2}}{T}\right)
  $$
- 当前实现用 inverse-CDF 查表抽样，公开参数为：
  - `thermal-sampler = maxwell-juttner`
  - `mj-pmax`
  - `mj-grid-points`
- 旧的 `thermal-sampler = gamma` 仍保留为兼容模式
- 抽到局域静止系四动量后，用流体元速度做 Lorentz boost，得到实验室系 `px/py/pz/E`
- 事件级 `v2` 由末态粒子的二阶 Q-vector 计算，若启用 `cooper-frye-weight = mt-cosh`，则使用粒子权重参与 Q-vector

## 输出和 QA

- ROOT 输出里固定有：
  - `events`
  - `participants`
  - `particles`
- 基础 QA 直方图包括：
  - `Npart`
  - `eps2`
  - `psi2`
  - `eps2_f`
  - `psi2_f`
  - `chi2`
  - `r2_0`
  - `r2_f`
  - `r2_ratio`
  - `v2`
  - `cent`
  - `participant_x-y`
  - `x-y`
  - `px-py`
  - `pT`
  - `eta`
  - `phi`
- `qa_blastwave_output` 会重新读 ROOT 文件并检查：
  - tree 和 branch 是否存在
  - participant 数是否和 event summary 一致
  - 粒子是否在质量壳上
  - `centrality` 是否和固定 `b` 映射一致
  - `events.v2` 是否能由粒子树重算
  - `r2_0/r2_f/r2_ratio` 是否能由粒子树重算
  - 可选 debug payload 如果存在，则也必须自洽

## 注意事项

- 现在默认路径已经不是旧的 participant hotspot 直接发射，而是 `density-evolution = affine-gaussian`
- 如果想回到旧对照路径，需要显式设置 `density-evolution = none`
- `eps2/psi2` 始终来自初态 participant geometry
- `eps2_f/psi2_f/chi2` 是 freeze-out geometry 的诊断量，不改变初态几何语义
- V1a 中二阶流响应使用初态 eccentricity vector，公开参数是 `kappa2`，不是旧的 `rho2`
- `density-normal-kappa-compensation` 默认关闭，且只允许与 `density-evolution = affine-gaussian` 和 `flow-velocity-sampler = density-normal` 联合使用
- `flow-density-sigma` 和 `smear` 语义不同：
  - `flow-density-sigma` 是 density 沉积宽度
  - `smear` 是旧 participant hotspot 发射路径的横向 smear
- V2 `gradient-response` 仍然是 opt-in，不替换默认 V1a
- V2 必须同时设置：
  - `density-evolution = gradient-response`
  - `flow-velocity-sampler = gradient-response`
- `qa/` 目录中可能有旧 schema 的历史 ROOT 文件，不要默认当作当前输出契约样本
