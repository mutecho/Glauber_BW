# Blast_Wave density-normal 累计梯度修正流强实现计划

## Required Public Contract

新增配置键：

```text
flow-trans-magnitude-mode = radius-profile | shell-gradient-corrected
flow-trans-gradient-strength = 0.0
flow-trans-gradient-density-floor-fraction = 1.0e-4
flow-trans-gradient-max-factor-delta = 0.2
```

决策已锁定：

- 默认 `flow-trans-magnitude-mode = radius-profile`，现有行为不变。
- `shell-gradient-corrected` 只允许：
  - `flow-velocity-sampler = density-normal`
  - `flow-trans-radius = density-percentile:<p>` 或 `density-level:<fraction>`
- `flow-trans-rho0` 就是 \(\rho_{\max}\)，不新增 `flow-trans-rho-max`。
- \(s(\mathbf{x})\) 复用现有 density-defined `xiUsed = r / R(phi)`，保持 `[0,1]` clamp。
- `flow-trans-gradient-strength = 0` 合法，corrected mode 退化为主径向项。
- 显式 `flow-trans-gradient-*` 参数若没有选择 `shell-gradient-corrected`，validation 报错。
- `covariance` 半径下选择 `shell-gradient-corrected` 报错。
- 修正上限限制乘法偏离量 `delta`，不是限制快度增量。

核心公式：

\[
\rho_T(\mathbf{x})=
\rho_{\max}s(\mathbf{x})^\alpha
\left[1+\lambda_A\left(\tilde A(\mathbf{x})-\langle\tilde A\rangle_s\right)\right]
\]

其中：

\[
A(\phi,s)=
\int_0^{sR(\phi)}
\max\left(
0,
\frac{-\nabla s_f(\mathbf c+t\hat e_\phi)\cdot \hat e_\phi}
{\max(s_f(\mathbf c+t\hat e_\phi), f_{\rm floor}s_{\rm scale})}
\right)dt
\]

归一化策略：

\[
\tilde A = \frac{A-\langle A\rangle_{\rm event}}{\mathrm{RMS}_{\rm event}}
\]

然后按相近 `s` 壳层扣除 \(\langle\tilde A\rangle_s\)。若 RMS 退化，修正表置零。

## Implementation Changes

- 在 `include/blastwave/FlowFieldModel.h` 增加 `FlowTransMagnitudeMode { RadiusProfile, ShellGradientCorrected }`，并扩展 `FlowFieldParameters`：
  - `flowTransMagnitudeMode`
  - `flowTransGradientStrength`
  - `flowTransGradientDensityFloorFraction`
  - `flowTransGradientMaxFactorDelta`
  - 对应 `has*` 显式配置标记按 validation 需要增加到 generator config 层。
- 在 `include/blastwave/BlastWaveGenerator.h` 和 `src/BlastWaveGeneratorSampling.cpp` 增加同名 config 字段并传入 runtime parameters。
- 在 `apps/generate_blastwave/RunOptions.cpp`：
  - 解析 `flow-trans-magnitude-mode`
  - 解析三个 `flow-trans-gradient-*`
  - 更新 help 的 key list 和 QA tuning knobs
  - 保持 CLI 覆盖 config 的现有规则。
- 在 `src/BlastWaveGeneratorValidation.cpp`：
  - mode enum 合法性由解析层保证；
  - strength 必须 finite；
  - density floor fraction 必须 finite 且 `>0`；
  - max factor delta 必须 finite 且 `0 <= value < 1`，保证 `1 + delta` 不会变负；
  - `shell-gradient-corrected` 只能在 density-normal + density-defined radius 下使用；
  - 显式 gradient 参数但 mode 不是 `shell-gradient-corrected` 时拒绝，避免静默无效参数。
- 在 `include/blastwave/EventMedium.h` 扩展事件级 mutable cache：
  - 现有 `FlowTransRadiusProfile` 保留；
  - 新增 `FlowTransGradientCorrectionProfile`，保存 mode/fraction/resolution/floor、center、angular/radial samples、event mean/RMS、`tildeA` 网格、shell mean 网格、valid 标记。
- 在 `src/FlowFieldModel.cpp`：
  - 复用 `getFlowTransRadiusProfile()` 的 boundary radii；
  - 新增构建 gradient correction profile 的 helper；
  - 使用 `evaluateDensityField()` 计算 density 和 gradient；
  - 每条角向射线沿半径做梯形累计，径向坐标映射到 `s = r / R(phi)`；
  - 先填 `A(angle,sIndex)`，再做全事件 mean/RMS，最后对每个 `sIndex` 求角向 shell mean；
  - 采样点通过现有 `computeDensityDefinedXiUsed()` 得到 `s`，再用角向+径向插值得到 `tildeA` 和 shell mean；
  - 计算 `delta = strength * (tildeA - shellMean)`，截断到 `[-maxFactorDelta,+maxFactorDelta]`；
  - `rhoRaw = flowTransRho0 * pow(s, flowTransProfilePower) * (1 + delta)`；
  - 非 finite、无效 profile、无效 boundary 时 fallback 到当前 `radius-profile` 行为；
  - 最终仍走现有 `computeBetaT()`，保证 `0 <= betaT <= kBetaTMax`。
- 新增完整中文配置 `config/test_b8_density_normal_flow_trans_gradient.cfg`：
  - 复制现有 `config/test_b8_density_normal_flow_trans.cfg` 风格；
  - 每个配置解释和配置分行；
  - 演示 `shell-gradient-corrected`、`density-percentile:0.95`、`balanced`、弱修正 `flow-trans-gradient-strength`。
