## 核心实现
- 新增核心模块 [include/blastwave/FlowFieldModel.h](/Users/allenzhou/Research_software/Blast_wave/include/blastwave/FlowFieldModel.h) 和 [src/FlowFieldModel.cpp](/Users/allenzhou/Research_software/Blast_wave/src/FlowFieldModel.cpp)，不引入新依赖，用解析式 2x2 对角化实现事件椭圆求解。
- 在该模块中定义 `WeightedTransversePoint`，字段固定为 `x`、`y`、`weight`，当前生成器全部用 `weight = 1.0`，为以后扩展保留接口。
- 在该模块中定义 `FlowEllipseInfo`，字段固定为 `valid`、`centerX`、`centerY`、`sigmaX2`、`sigmaY2`、`sigmaXY`、`lambdaMajor`、`lambdaMinor`、`radiusMajor`、`radiusMinor`、`majorAxisX`、`majorAxisY`、`minorAxisX`、`minorAxisY`、`eps2`、`psi2`。
- `FlowEllipseInfo` 的计算规则固定为：先求带权质心，再求中心化二阶矩，再求 `trace = sigmaX2 + sigmaY2` 与 `delta = sqrt((sigmaX2 - sigmaY2)^2 + 4*sigmaXY^2)`，再取 `lambdaMajor = 0.5*(trace + delta)`、`lambdaMinor = 0.5*(trace - delta)`，半轴定义为对应平方根。
- `eps2` 和 `psi2` 继续沿用现有事件摘要定义：`eccX = sigmaY2 - sigmaX2`、`eccY = 2*sigmaXY`、`eps2 = sqrt(eccX^2 + eccY^2)/trace`、`psi2 = 0.5*atan2(eccY, eccX)`；这样 `events.eps2/psi2` 与现有 QA 和下游语义兼容。
- 椭圆有效性 `valid` 的判据固定为：`Npart >= 2`、`trace > 1e-12`、`lambdaMajor > 1e-12`、`lambdaMinor > 1e-12`。无效时仍保留已能计算的质心、协方差和 `eps2/psi2`，但流场评估不使用归一化椭圆半径。
- 主轴定向要做确定性约束：若 `eps2 > 1e-12`，让短轴方向与 `(cos(psi2), sin(psi2))` 的点积非负；若事件近圆，则用长轴向量的字典序正向作为 tie-breaker；随后用固定右手系构造另一根轴，避免 debug 输出和测试出现纯符号翻转抖动。
- 新流场公式固定为：对发射点先减事件质心，再投影到长短轴基底得到 `x'`、`y'`；定义 `rTilde = sqrt(x'^2/radiusMajor^2 + y'^2/radiusMinor^2)`；定义椭圆法向角 `phiB = atan2(y'/radiusMinor^2, x'/radiusMajor^2)`；定义 `rhoRaw = pow(rTilde, flowPower) * (rho0 + rho2 * eps2 * cos(2*phiB))`。
- `rho2` 明确采用 `rho2 * eps2`，不是文档原稿里与事件几何脱钩的固定振幅；目的就是让接近圆对称的事件自动退回近各向同性流场。
- 横向流大小固定按 `betaT = tanh(max(0, rhoRaw))` 求值，再裁到 `0.95`；若椭圆无效则直接令 `betaT = 0`。这样保留当前“永不超光速”的安全边界，也避免负 `rho` 产生向内流。
- 实验室系速度构造继续沿用当前 Bjorken 风格组合：横向部分沿椭圆法向，纵向部分仍来自 `etaS`；若椭圆无效则只保留 `bz = tanh(etaS)`，令 `bx = by = 0`。
- 发射点仍然使用 smeared hotspot 坐标求流场，事件椭圆仍只由未 smear 的 participants 构造；`sourceX/sourceY`、`x/y` 的现有记录语义不变。
