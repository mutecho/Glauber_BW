## Locked Decisions

- `flow-trans-rho0` 与 `flow-trans-profile-power` 是旧 `rho0` / `flow-power` 的直接替代公开名，所有继续使用 transverse rapidity baseline/profile 的 sampler 都应读取新字段。
- 旧 `rho0` 与 `flow-power` 不做兼容解析；config 与 CLI 都直接报错，并在错误文本里给出迁移提示。
- 本轮新增的行为配置只允许 `flow-velocity-sampler = density-normal`：
  - `flow-trans-direction-gradient-fraction`
  - `flow-trans-radius = covariance | density-percentile:<p> | density-level:<fraction>`
- 暂不重命名或替换 `kappa2`；它继续是二阶 response 系数。
- 新增 `EventMedium::emissionDensityScale`，用于 `emissionDensity` 的 density-level cutoff/floor，不复用 `markerDensityScale` 或 `dynamicsDensityScale`。
- `affine-effective`、`covariance-ellipse` 和 `gradient-response` 本轮不接入新的方向/半径行为；如果在这些 sampler 下显式配置 `flow-trans-direction-gradient-fraction` 或 `flow-trans-radius`，validation 直接报错。
- 累计梯度修正在本历史第一阶段不实现；2026-05-07 第二阶段已新增 `flow-trans-magnitude-mode = shell-gradient-corrected` 与 `flow-trans-gradient-*` 行为。

## Implementation Changes

- 配置层：更新 `BlastWaveConfig`、`FlowFieldParameters`、`RunOptions` 和 validation。
  - 新键支持 config 与 CLI。
  - `rho0` 与 `flow-power` 作为旧公开名直接拒绝。
  - `flow-trans-direction-gradient-fraction` 要求 `[0,1]`。
  - `density-percentile:<p>` 要求 `0<p<1`。
- `density-level:<fraction>` 当前要求 `0 < fraction < 1`；早期只要求正值的描述已被 sigma-equivalent 标定修复取代。
  - `flow-trans-rho0` / `flow-trans-profile-power` 作为横向流强 baseline/profile 的唯一公开名称。
- 介质层：在 [include/blastwave/EventMedium.h](/Users/allenzhou/Research_software/Blast_wave/include/blastwave/EventMedium.h:55) 增加 `emissionDensityScale`，在 [src/EventMedium.cpp](/Users/allenzhou/Research_software/Blast_wave/src/EventMedium.cpp:224) 与现有 scale 一起计算。
- 流场层：重构 [src/FlowFieldModel.cpp](/Users/allenzhou/Research_software/Blast_wave/src/FlowFieldModel.cpp:220) 的 density-normal 路径。
  - 方向：几何径向、density-gradient 外向、按 `flow-trans-direction-gradient-fraction` 混合并归一化。
  - 半径：`covariance` 复现旧 `rTilde`；`density-percentile` 与 `density-level` 使用事件级发射密度角向边界 profile cache。
  - 流强：`rhoRaw = flowTransRho0 * pow(xiFlow, flowTransProfilePower)`；density-defined 半径下 `xiShell = r / R(phi)` 只作为 shell 坐标，主流强使用 `xiFlow = q * xiShell`，`flowTransRho0` 是 covariance-equivalent `xiFlow = 1` 处的参考快度尺度。
  - fallback：混合方向 -> gradient 方向 -> 几何径向 -> covariance normal -> 零流。
- 不改 ROOT 主 schema；只有确实需要 debug 图时才添加可选 payload，并同步 QA all-or-none 检查。
