# 方案 2：Density-Defined 半径强度尺度改为无界 `xi = r / R(phi)`

## Summary

- 目标：让 `flow-trans-radius = density-percentile:<p>` / `density-level:<fraction>` 只定义局域密度几何尺度 `R(phi)`，不再把 `R(phi)` 当最大流强边界。
- 核心行为：基础流强坐标从当前 `xiUsed = clamp(r / R(phi), 0, 1)` 改为 `xiUsed = r / R(phi)`，只保留非有限值 fallback 和物理 `betaT` cap。
- 不新增配置项，不改 ROOT schema，不改 CLI 名称；这是现有 density-defined radius 语义修正。
- 影响范围：只影响 `flow-velocity-sampler = density-normal` 且使用 density-defined radius 的路径；`covariance` 半径、默认 `covariance-ellipse` sampler、`affine-effective`、`gradient-response` 不变。

## Key Changes

- 在 `src/FlowFieldModel.cpp` 修改 `computeDensityDefinedXiUsed()`：
  - 保留 `boundaryRadius` 插值、`radialDistance` 计算和非有限 fallback。
  - 将返回值从上限 clamp 改为无上限非负值：
    - 若 `xi = radialDistance / boundaryRadius` 有限且 `xi >= 0`，返回 `xi`。
    - 若非有限，返回 `NaN`，让调用方继续 fallback 到 covariance `metric.rTilde`。
  - 不再返回 `std::min(1.0, ...)`。
- 保持 `sampleFlowTransGradientCorrectionValue()` 的表域 lookup clamp：
  - `shell-gradient-corrected` 的修正表仍只定义在已有 `0 <= xi <= 1` 边界 profile 上。
  - 对 `xi > 1` 的点，基础 `rhoRaw` 使用无界 `xi^power`；修正因子 lookup 使用外层表格值。
  - 加短注释说明这是 correction-profile 插值域限制，不是基础流强尺度 clamp。
- 不改 public config：
  - `flow-trans-radius = covariance | density-percentile:<p> | density-level:<fraction>` 保持原语法。
  - `flow-trans-rho0` 语义更新为 “`xi=1` 参考半径处的横向流快度尺度”，不是 density boundary 之外的最大值。
- 不改输出：
  - ROOT trees/histograms、metadata、QA schema 全部保持不变。
