# `response-test-023` 逐事件宽分布随机化实施计划

## 目标与现状

在 `/Users/allenzhou/Research_software/Blast_wave` 中，为现有 `initial-geometry = response-test-023` 增加一个 opt-in 的逐事件模板参数随机化功能，使人工 023 的 measured `events.eps2/eps3` 分布更接近当前 Glauber 宽分布。

当前事实：

- Glauber 当前约为：`eps2 max ~0.625, p95 ~0.437`；`eps3 max ~0.595, p95 ~0.375`。
- 固定 023 当前明显更窄：`eps2 max ~0.30-0.40`；`eps3 max ~0.22-0.32`。
- notebook 拟合不改变 epsilon 区间；`eps2/eps3` 来自生成器写入的 ROOT `events` tree。
- 现有固定 023 要保留，继续作为 closure baseline。
- 新功能只对 `response-test-023` 生效，不兼容、不泛化到其他初态模式。

## 公共接口与配置

新增配置项，默认全部保持关闭或无效，不改变现有行为：

- `initial-geometry-fluctuate = false | true`
- `initial-geometry-source-count-min`
- `initial-geometry-source-count-max`
- `initial-geometry-a2-min`
- `initial-geometry-a2-max`
- `initial-geometry-a3-min`
- `initial-geometry-a3-max`
- `initial-geometry-r2x-min`
- `initial-geometry-r2x-max`
- `initial-geometry-r2y-min`
- `initial-geometry-r2y-max`
- `initial-geometry-r3-min`
- `initial-geometry-r3-max`
- `initial-geometry-sigma3-min`
- `initial-geometry-sigma3-max`

CLI 同步支持：

- `--initial-geometry-fluctuate`
- `--no-initial-geometry-fluctuate`
- 其余范围项用 `--key value` 形式。

校验规则：

- `initial-geometry-fluctuate=true` 只能与 `initial-geometry=response-test-023` 联合使用；Glauber 下直接报错。
- fluctuate=true 时，上述范围必须完整配置。
- source-count 范围为正整数，且 `min <= max`。
- double 范围必须 finite，非负或正值按原字段语义校验，且 `min <= max`。
- fluctuate=false 时，所有新增范围项即使配置也不参与生成；建议允许解析但不使用，避免误伤旧配置调试。

新增完整中文 cfg：

- 建议文件名：`config/test_023_fluctuating.cfg`
- 输出：`qa/test_023_fluctuating.root`
- 每个配置项前一行中文解释。
- 推荐中等宽度范围：
  - `source-count = 180..700`
  - `a2 = 0.0..1.0`
  - `a3 = 0.0..0.7`
  - `r2x = 1.8..3.2 fm`
  - `r2y = 0.9..1.8 fm`
  - `r3 = 1.4..3.0 fm`
  - `sigma3 = 0.45..1.3 fm`
- `r0`、`phi2`、`phi3` 第一版不随机化，继续使用单值配置。

## 实现要点

核心代码位置：

- `include/blastwave/BlastWaveGenerator.h`
- `src/BlastWaveGeneratorGeometry.cpp`
- `src/BlastWaveGenerator.cpp`
- `src/BlastWaveGeneratorValidation.cpp`
- `apps/generate_blastwave/RunOptions.cpp`
- `apps/generate_blastwave/RunOptions.h`
- 如新增 ROOT branches，还需改 `include/blastwave/io/RootOutputSchema.h`、`src/RootOutputSchema.cpp`、`apps/qa_blastwave_output.cpp`。

实现方式：

- 在 `BlastWaveConfig` 中加入 fluctuate 开关、范围字段、范围是否配置过的标志。
- 定义一个 event-local 023 参数结构，例如 `ResponseTest023EventParameters`，包含 source-count、A2、A3、r2x、r2y、r3、sigma3、r0、phi2、phi3。
- `sampleResponseTest023Participants()` 内部先构造 event-local 参数：
  - fluctuate=false：从原固定配置复制。
  - fluctuate=true：用当前 `rng_` 按 uniform 分布抽范围值。
- 原 `1:A2:A3` 分量分配、Gaussian 点云生成、三角 hotspot、recenter 逻辑使用 event-local 参数，不再直接读全局固定字段。
- `GeneratedEvent.info.geoA2/geoA3/geoR3/geoSigma3` 必须写入 event-local 实际抽样值。
- 因为随机化 `r2x/r2y`，建议新增 event branches：
  - `geo_r2x`
  - `geo_r2y`
- `Npart` 自然等于 event-local source-count，不需要额外 branch。
- 固定模式必须保持现有行为：`initial-geometry-fluctuate=false` 时，现有 `test_023_*.cfg` 输出逻辑不变。

## 测试与验收

单元测试：

- `RunOptionsTest`
  - 默认 `initialGeometryFluctuate=false`。
  - cfg 和 CLI 能解析所有新增项。
  - CLI override 生效。
  - 缺范围、`min > max`、非法负值、Glauber 下 fluctuate=true 都触发 validation failure。
- `BlastWaveGeneratorResponseTest`
  - fixed 023 仍生成指定 source-count。
  - fluctuating 023 多事件生成中，`Npart` 落在 source-count range。
  - `geo_a2/geo_a3/geo_r3/geo_sigma3/geo_r2x/geo_r2y` 落在配置范围。
  - 固定 seed 下结果可复现，且多个事件出现不同模板参数。

运行验证：

- build：`cmake --build /Users/allenzhou/Research_software/Blast_wave/build -j4`
- tests：`ctest --test-dir /Users/allenzhou/Research_software/Blast_wave/build --output-on-failure`
- ROOT 生成与 QA：
  - 用 O2Physics ROOT executor 运行 `config/test_023_fluctuating.cfg --nevents 5000`
  - 对输出跑 `qa_blastwave_output --expect-nevents 5000`
- 分布验收：
  - 用 `uproot` 或现有 notebook 对比 `qa/test_023_fluctuating.root` 与当前 `qa/*_glauber.root` 的 `eps2/eps3` 分位数。
  - 成功标准：fluctuating 023 的 `eps2` p95 进入约 `0.35..0.50`，`eps3` p95 进入约 `0.28..0.43`，并明显宽于当前固定 023。
  - 若第一版范围没达到目标，只调整 `config/test_023_fluctuating.cfg` 范围，不改算法。
