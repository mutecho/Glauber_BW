# 历史计划归档

本目录保存已经执行、替代或只作为设计参考的计划文档。它们是 `historical_reference`，默认不参与当前文档写集。

## 使用方式

- 查设计动机、旧方案取舍或实现背景时可以读这里。
- 判断当前行为时，优先看 `../项目说明.md`、`../数学物理公式流程说明.md`、`../手记文档.md`、`../../project-state/` 和代码。
- 若某个历史计划的结论重新成为当前契约，应把结论整合进当前说明或 `project-state/`，不要直接把历史计划改成新的源文档。

## 特别说明

- `agent_guide.md` 和 `blastwave_generator_agent_handoff.md` 在这里属于历史归档。
- 当前 agent 工作规则以仓库根目录 `AGENTS.md` 和 `../../project-state/doc-sync-map.yml` 为准。
