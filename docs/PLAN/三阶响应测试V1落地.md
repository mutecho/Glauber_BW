## Implementation Changes

- Add initial-geometry config surface:
  - `enum class InitialGeometryMode { Glauber, ResponseTest023 }`
  - `BlastWaveConfig.initialGeometryMode = Glauber`
  - Template knobs: `initial-geometry-source-count`, `initial-geometry-r0`, `initial-geometry-a2`, `initial-geometry-r2x`, `initial-geometry-r2y`, `initial-geometry-phi2`, `initial-geometry-a3`, `initial-geometry-r3`, `initial-geometry-sigma3`, `initial-geometry-phi3`, `debug-initial-geometry`.
  - CLI/config keys mirror field names; default behavior remains Glauber.

- Add template source generation:
  - Keep existing `sampleParticipants()` for `glauber`.
  - In `response-test-023`, generate a synthetic point cloud with `nucleusId = -1`.
  - Use point-cloud approximation instead of a density grid: allocate source counts in proportion to `1 : A2 : A3`.
  - `n=0`: Gaussian background centered at zero with width `R0`.
  - `n=2`: implement one baseline only, preferably elliptical Gaussian using `R2x/R2y/Phi2`.
  - `n=3`: three Gaussian hotspots centered at `R3 * (cos(phi3 + 2*pi*k/3), sin(...))`, width `sigma3`.
  - Recenter the synthetic cloud after generation to suppress artificial dipole contamination.
  - Pass the resulting points through existing `buildEventMedium()` so all current `density-evolution` and `EmissionSampler` modes continue to work.

- Add initial harmonic observables:
  - Add reusable ROOT-free helper for weighted recentered harmonic geometry:
    - inputs: `vector<WeightedTransversePoint>`, harmonic `n`
    - outputs: `epsilon`, `psi`, `centerX`, `centerY`, `rRms`
    - formula: `E_n = - sum(w r^n exp(i n phi)) / sum(w r^n)` or use the sign convention explicitly documented in code; keep producer and QA consistent.
  - Preserve current `eps2/psi2` covariance semantics for compatibility.
  - Add `eps3/psi3` from the new helper on `medium.participantPoints`.

- Add final-state harmonic observables:
  - During particle generation, accumulate weighted `Q2` and `Q3`.
  - Keep existing `events.v2 = |Q2| / sum(w)`.
  - Add:
    - `v3 = |Q3| / sum(w)`
    - `v2_wrt_psi2 = Re(Q2 * exp(-2i psi2)) / sum(w)`
    - `v3_wrt_psi3 = Re(Q3 * exp(-3i psi3)) / sum(w)`

- Extend ROOT contract:
  - `events` branches: `initial_geometry_mode`, `eps3`, `psi3`, `v3`, `v2_wrt_psi2`, `v3_wrt_psi3`, `x_center_init`, `y_center_init`, `R_rms_init`, `geo_a2`, `geo_a3`, `geo_r3`, `geo_sigma3`.
  - Histograms: `eps3`, `psi3`, `v3`, `v2_wrt_psi2`, `v3_wrt_psi3`, plus response/cross-talk TH2: `v3_wrt_psi3_vs_eps3`, `v2_wrt_psi2_vs_eps2`, `v3_wrt_psi3_vs_eps2`, `v2_wrt_psi2_vs_eps3`.
  - Optional debug TH2: `initial_geometry_density_x-y`, first valid event only, enabled by `debug-initial-geometry`.

- Extend QA:
  - Bind/read new event branches.
  - Recompute `v3`, `v2_wrt_psi2`, `v3_wrt_psi3` from `particles`.
  - Validate finite ranges: `eps3 in [0,1]`, finite `psi3`, `v3 in [0,1]`, projected values finite and within `[-1,1]`.
  - Validate new histogram entry counts and optional initial-geometry density map if present.

- Add config/docs:
  - Add `config/test_b8_response_023.cfg` with full Chinese explanations and complete config entries.
  - Recommended initial sample config: `density-evolution = none`, `flow-velocity-sampler = density-normal`, fixed `rho0`, fixed multiplicity and longitudinal settings, scan only `initial-geometry-a3` first.
  - Update docs and `project-state/` because this changes algorithm semantics, config contract, ROOT schema, QA, and user run flow.

## Assumptions

- V1 does not add `kappa3` or explicit third-harmonic flow modulation; it tests whether current density/flow/emission chain carries triangular geometry.
- `A2/A3` are template weights only; all analysis should use measured `eps2/eps3`.
- Synthetic template sources may be stored in the existing `participants` tree with `nucleus_id = -1`.
- The implementation should prefer existing point-cloud density interfaces over introducing a new grid density backend.
