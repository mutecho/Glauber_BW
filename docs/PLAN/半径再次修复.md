# 方案 3：density-defined 半径改为 covariance/sigma-equivalent 强度尺度

## Summary

把 `flow-trans-radius = density-percentile:<p>` / `density-level:<f>` 从当前的直接 `xi = r / R_density(phi)` 改成“密度边界只决定角向几何形状，流强半径用 covariance-equivalent 尺度”。目标是保留 density-defined 半径对局域几何和角向结构的响应，同时让 `flow-trans-rho0` 仍表示与 `flow-trans-radius = covariance` 可比的 `xi_flow = 1` 强度尺度，避免 `p=0.95` 这类外边界继续压低 bulk 流强。

## Key Changes

- 不新增公开配置项；继续使用现有 `flow-trans-radius` 语法。
- 在 `src/FlowFieldModel.cpp` 内部拆分两个半径变量：
  - `xi_shell = r / R_density(phi)`：density shell 坐标，用于 shell-gradient correction 表 lookup。
  - `xi_flow = q * xi_shell`：进入主流强公式的 covariance-equivalent 半径。
- 等效尺度固定为：
  - `density-percentile:p`: `q = sqrt(-2 * log(1 - p))`，实现用 `std::log1p(-p)`。
  - `density-level:f`: `q = sqrt(-2 * log(f))`。
- 主流强改为：
  - `rhoRaw = flowTransRho0 * pow(xi_flow, flowTransProfilePower) * correctionFactor`
  - `correctionFactor` 仍只来自 `shell-gradient-corrected`。
- `shell-gradient-corrected` 的 correction table lookup 必须继续使用 `xi_shell`，不是 `xi_flow`；否则 `p=0.95` 时边界处 `xi_flow≈2.45` 会错误地把大部分点夹到最外层 correction shell。
- `flow-trans-radius = covariance` 保持现有 `metric.rTilde` 行为不变。
- density-defined profile cache 仍只缓存 `R_density(phi)`；不需要把 `q` 写进 `FlowTransRadiusProfile`，因为它只由 mode/fraction 即时派生。
- 收紧 `density-level` validation：要求 `0 < f < 1`。同步错误信息、RunOptions/validation 测试和文档；`f >= 1` 在方案 3 下没有实数 sigma-equivalent 半径。
- 不改 ROOT 输出 schema、metadata branch、配置文件语法或 `flow-trans-radius-resolution` 的采样含义。
