Blast-Wave 数学物理公式流程说明

说明范围

本文按当前代码主线说明一次 generate_blastwave_events 运行中每一步的数学/物理定义，以及它在程序中的抽象操作。它描述的是当前实现，而不是完整流体动力学模型：

  • 当前生成单一直接 pi+。
  • 当前不是完整 Cooper-Frye freeze-out 超曲面采样；cooper-frye-weight = mt-cosh 只是一个可选权重代理。
  • 默认链路是 density-evolution = affine-gaussian、flow-velocity-sampler = covariance-ellipse、thermal-sampler = maxwell-juttner。
  • density-evolution = none、flow-velocity-sampler = density-normal、flow-velocity-sampler = affine-effective、density/flow = gradient-response、v2{2}(pT) 都是当前可选分支。

符号约定

  • 横向坐标写作 \[ \mathbf r=(x,y)^T,\qquad \Delta \mathbf r=\mathbf r-\bar{\mathbf r}. \]
  • 自然单位约定 \(c=1\)。长度单位是 fm，动量和质量单位是 GeV。
  • participant 点云记为 \[ \{(\mathbf r_i,w_i)\}_{i=1}^{N_{\rm part}}, \] 当前 Glauber 生成器中 \(w_i=1\)。
  • participant-plane 的短轴方向记为 \(\mathbf e_{\rm in}\)，长轴方向记为 \(\mathbf e_{\rm out}\)。代码里 in-plane 对应 covariance minor axis，out-of-plane 对应 major axis。
  • 初态 participant 几何量使用下标 0 或 participant；freeze-out/发射阶段几何量使用下标 f。
  • 横向流快度记为 \(\rho_t\)，横向速度大小记为 \(\beta_T=\tanh\rho_t\)。

总体抽象流程

当前 generateEvent() 的总流程可以抽象成：

配置参数
  -> sampleParticipants()
  -> buildEventMedium()
  -> compute initial geometry observables (eps2/psi2, eps3/psi3)
  -> sampleEmissionSites()
  -> sample eta_s
  -> sample local thermal momentum p*
  -> evaluate flow velocity beta
  -> Lorentz boost p* -> p_lab
  -> accumulate EventInfo / ParticleRecord / ROOT payload

对应代码落点：

  • 事件编排：src/BlastWaveGenerator.cpp
  • participant 几何：src/BlastWaveGeneratorGeometry.cpp
  • 介质构造：src/EventMedium.cpp
  • 发射点抽样：src/EmissionSampler.cpp
  • 流场：src/FlowFieldModel.cpp
  • 热动量：src/MaxwellJuttnerMomentumSampler.cpp
  • 共享观测量：src/PhysicsUtils.cpp
  • ROOT 写出：apps/generate_blastwave/RootEventFileWriter.cpp

代码位置

  • apps/generate_blastwave_events.cpp：程序入口，完成参数解析、事件循环和 writer 调用。
  • include/blastwave/BlastWaveGenerator.h：BlastWaveConfig、GeneratedEvent、EventInfo、ParticleRecord 等主数据契约。
  • src/BlastWaveGenerator.cpp：BlastWaveGenerator::generateEvent()，串起几何、介质、发射、动量、flow 和事件摘要。
  • apps/generate_blastwave/RunOptions.cpp：CLI/config 参数解析到 BlastWaveConfig 与 app 层选项。

1. Woods-Saxon 核子抽样与 participant 判定

数学/物理公式

单个核子的径向 Woods-Saxon 密度取

\[ \rho_{\rm WS}(r)\propto \frac{1}{1+\exp((r-R)/a)}, \]

其中 \(R=\texttt{woodsSaxonRadius}\)，\(a=\texttt{woodsSaxonDiffuseness}\)。球坐标抽样的径向概率包含体积因子：

\[ P(r)\,dr \propto r^2 \rho_{\rm WS}(r)\,dr. \]

两束核分别在横向 \(x\) 方向平移

\[ x_A=x-\frac{b}{2},\qquad x_B=x+\frac{b}{2}, \]

其中 \(b=\texttt{impactParameter}\)。两个核子发生一次几何碰撞的判据为

\[ d_T^2=(x_A-x_B)^2+(y_A-y_B)^2 \le \frac{\sigma_{NN}}{\pi}. \]

至少发生一次碰撞的核子被标记为 participant。

当前中心度只是固定 \(b\) 的显示映射：

\[ {\rm centrality} =\operatorname{clip}\left( \frac{100\,b}{2R},0,100 \right). \]

抽象操作

这一层把“两个带 Woods-Saxon 形状的核在固定冲量参数下几何相交”抽象成一个逐事件 participant 点云：

\[ \{(x_i,y_i,z_i,\text{nucleus\_id}_i)\}. \]

后续所有初态几何、密度和发射锚点都从这个点云派生。

代码位置

  • src/BlastWaveGeneratorGeometry.cpp：sampleParticipants() 实现两核抽样和 \(NN\) black-disk participant 判定。
  • src/BlastWaveGeneratorGeometry.cpp：sampleSingleNucleon() 实现 Woods-Saxon 径向拒绝抽样和核子坐标平移。
  • src/PhysicsUtils.cpp：computeCentralityPercent() 实现当前固定 \(b\) 的中心度显示映射。
  • include/blastwave/BlastWaveGenerator.h：BlastWaveConfig 保存 impactParameter、sigmaNN、woodsSaxonRadius、woodsSaxonDiffuseness 等参数。

1.1 response-test-023 人工初态模板

当配置

initial-geometry = response-test-023

时，sampleParticipants() 不再抽 Woods-Saxon 核子，而是生成一个横向 synthetic point cloud。该模板用于 response/closure test，不用于替代真实初态模型。

点云密度的概念形式为

\[ \rho(x,y)=\rho_0(x,y)+A_2\rho_2(x,y)+A_3\rho_3(x,y). \]

这里 \(A_2,A_3\) 只是模板混合权重，不等同于实际 \(\epsilon_2,\epsilon_3\)。默认 initial-geometry-source-allocation = ratio-total 保持原固定总池契约：按 1:A2:A3 把 initial-geometry-source-count 个离散源点分配到三类分量：

  • \(n=0\)：宽度 initial-geometry-r0 的圆形 Gaussian 背景；
  • \(n=2\)：局域主轴宽度 initial-geometry-r2x/r2y、方向 initial-geometry-phi2 的椭圆 Gaussian；
  • \(n=3\)：三个中心位于 \[ R_3(\cos(\Phi_3+2\pi k/3),\sin(\Phi_3+2\pi k/3)),\quad k=0,1,2 \] 的 Gaussian hotspot，参数为 initial-geometry-r3、initial-geometry-sigma3、initial-geometry-phi3。

生成后点云会整体 recenter，使

\[ \bar x\simeq 0,\qquad \bar y\simeq 0, \]

以压低人工 dipole contamination。模板源在 participants tree 中使用 nucleus_id=-1，随后仍走同一个 buildEventMedium()、EmissionSampler、flow 和 ROOT 输出链。

当 initial-geometry-fluctuate = true 时，response-test-023 在每个事件开始时抽样 event-local 模板参数：

\[ N_{\rm src}\sim U_{\mathbb{Z}}(N_{\min},N_{\max}),\quad A_2\sim U(A_{2,\min},A_{2,\max}),\quad A_3\sim U(A_{3,\min},A_{3,\max}), \]

\[ R_{2x}\sim U(R_{2x,\min},R_{2x,\max}),\quad R_{2y}\sim U(R_{2y,\min},R_{2y,\max}),\quad R_3\sim U(R_{3,\min},R_{3,\max}),\quad \sigma_3\sim U(\sigma_{3,\min},\sigma_{3,\max}). \]

r0、phi2、phi3 第一版不随机化。随机化只作用于 response-test-023，用于构造比固定 closure baseline 更宽的人工 events.eps2/eps3 分布；Glauber 不读取这些范围。ROOT events.geo_* 字段保存每个事件实际抽到的模板参数，因此分析时应使用 events.eps2/eps3 判断几何响应，而不是把配置范围或 \(A_2,A_3\) 解释成物理 eccentricity。

默认模板用同一个固定总源数池近似实现

\[ N_0:N_2:N_3\simeq 1:A_2:A_3 . \]

因此 \(A_2\) 与 \(A_3\) 的抽样独立只说明模板权重随机数独立，不保证最终点云中测得的 \(\epsilon_2\) 与 \(\epsilon_3\) 独立。提高 \(A_3\) 会在同一个总源数预算里改变背景和二阶椭圆分量的相对占比，同时改变二阶矩分母；提高 \(A_2\) 也会以同样方式改变三角 hotspot 的相对统计权重。

可选 initial-geometry-source-allocation = independent-pools 把 initial-geometry-source-count 解释为圆形背景池 \(N_0\)，并使用

\[ N_2=\mathrm{round}(N_0 A_2),\qquad N_3=\mathrm{round}(N_0 A_3), \]

其中 \(A_3>0\) 且取整后少于 3 时，\(N_3\) 会补到 3 个源点，使三个 hotspot 至少各有一个样本。该模式去掉了固定总源数下的组分竞争，但总点数和共同径向矩仍随 \(A_2,A_3\) 改变。2026-06-03 的 5000-event 验证显示它没有达到 |corr(eps2,eps3)| < 0.08 的 measured-eccentricity 解耦验收：full fluctuating 样本 corr(eps2,eps3)=-0.331139，eps3_only 控制样本 corr(eps2,eps3)=-0.316660。因此它目前是诊断/对照模式，不是最终 decorrelation 方案；若需要严格隔离二阶和三阶初态自由度，应继续做分层匹配后的事件抽样、目标 eccentricity 反解，或其它显式 decorrelation 方案。

  • include/blastwave/BlastWaveGenerator.h：InitialGeometryMode 与 initialGeometry* 配置字段。
  • src/BlastWaveGeneratorGeometry.cpp：sampleParticipants() 在 Glauber 与 sampleResponseTest023Participants() 之间分派。
  • apps/generate_blastwave/RunOptions.cpp：解析 initial-geometry* 和 debug-initial-geometry。
  • config/test_023_dense.cfg：完整中文说明的三阶响应测试配置。
  • config/test_023_fluctuating.cfg：完整中文说明的逐事件宽分布 response-test 配置。
  • config/test_023_dense_eps2_only_fluct.cfg 与 config/test_023_dense_eps3_only_fluct.cfg：只打开一个模板权重区间的交叉性控制配置。
  • config/test_023_dense_fluct_independent_pools.cfg：使用独立源池分配的宽分布诊断配置；它不应被解释为已经通过 eps2/eps3 解耦验收。

2. Participant 初态几何：\(\epsilon_2,\Psi_2\) 与 \(\epsilon_3,\Psi_3\)

数学/物理公式

对横向点云计算加权中心：

\[ W=\sum_i w_i,\qquad \bar x=\frac{1}{W}\sum_i w_i x_i,\qquad \bar y=\frac{1}{W}\sum_i w_i y_i. \]

二阶矩/covariance 为

\[ \Sigma= \begin{pmatrix} \sigma_x^2 & \sigma_{xy}\\ \sigma_{xy} & \sigma_y^2 \end{pmatrix}, \]

其中

\[ \sigma_x^2=\frac{1}{W}\sum_i w_i(x_i-\bar x)^2, \]

\[ \sigma_y^2=\frac{1}{W}\sum_i w_i(y_i-\bar y)^2, \]

\[ \sigma_{xy}=\frac{1}{W}\sum_i w_i(x_i-\bar x)(y_i-\bar y). \]

代码使用的二阶 eccentricity 向量为

\[ E_x=\sigma_y^2-\sigma_x^2,\qquad E_y=2\sigma_{xy}. \]

participant eccentricity 和 participant-plane 角为

\[ \epsilon_2= \frac{\sqrt{E_x^2+E_y^2}}{\sigma_x^2+\sigma_y^2}, \]

\[ \Psi_2=\frac{1}{2}\operatorname{atan2}(E_y,E_x). \]

协方差矩阵本征值为

\[ \lambda_{\rm major/minor} =\frac{1}{2}\left[ \operatorname{tr}\Sigma \pm \sqrt{(\sigma_x^2-\sigma_y^2)^2+4\sigma_{xy}^2} \right], \]

半轴长度为

\[ R_{\rm major/minor}=\sqrt{\lambda_{\rm major/minor}}. \]

给定一个查询点 \(\mathbf r\)，协方差归一化法向和广义半径定义为

\[ \mathbf q=\Sigma^{-1}(\mathbf r-\bar{\mathbf r}), \]

\[ \hat{\mathbf n}_{\Sigma}=\frac{\mathbf q}{|\mathbf q|}, \]

\[ \tilde r^2=(\mathbf r-\bar{\mathbf r})^T\Sigma^{-1}(\mathbf r-\bar{\mathbf r}). \]

三阶几何不从 covariance 主轴推出，而是对同一个加权初态点云使用 recentered harmonic moment。以上面的 \(\bar x,\bar y\) 为中心，定义

\[ r_i=\sqrt{(x_i-\bar x)^2+(y_i-\bar y)^2}, \qquad \phi_i=\operatorname{atan2}(y_i-\bar y,x_i-\bar x). \]

对 \(n\) 阶 harmonic，代码采用

\[ M_n=\sum_i w_i r_i^n, \qquad C_n=-\sum_i w_i r_i^n\cos(n\phi_i), \qquad S_n=-\sum_i w_i r_i^n\sin(n\phi_i), \]

\[ E_n=\frac{C_n+iS_n}{M_n} = -\frac{\sum_i w_i r_i^n e^{in\phi_i}}{\sum_i w_i r_i^n}. \]

因此当前写入 ROOT 的初态三阶量为

\[ \epsilon_3=\frac{\sqrt{C_3^2+S_3^2}}{M_3}, \qquad \Psi_3=\frac{1}{3}\operatorname{atan2}(S_3,C_3). \]

同一个 helper 还给出 recentered 初态中心和 RMS 半径：

\[ R_{\rm rms,0} = \sqrt{\frac{1}{W}\sum_i w_i r_i^2}. \]

若有效点云为空或 \(M_3\le 0\)，helper 返回 invalid，事件级 eps3/psi3/x_center_init/y_center_init/R_rms_init 保持默认值。对 response-test-023，initial-geometry-a3 只是三角 hotspot 模板的混合权重；分析和响应图应使用这里从实际点云计算出的 events.eps3，不能把模板权重当作 \(\epsilon_3\)。

抽象操作

这一层把离散 participant 点云压缩成事件级初态几何代理：

  • centerX/centerY：事件几何中心
  • eps2/psi2：初态二阶 covariance eccentricity 与 participant-plane
  • eps3/psi3：初态三阶 recentered harmonic eccentricity 与三阶平面
  • x_center_init/y_center_init/R_rms_init：三阶 helper 同步使用的 recentered 初态中心和 RMS 半径
  • major/minor axis：稳定的主轴基
  • inverse covariance：供流场方向、归一化半径和 fallback 使用

重要语义：events.eps2/psi2 始终保留 covariance 椭圆语义；events.eps3/psi3 使用 recentered harmonic 语义。两者都来自初态 participant 点云，不会被 freeze-out 几何覆盖。

3. Gaussian point-cloud 密度场

数学/物理公式

当前横向密度统一用 Gaussian point-cloud 表示。对支持点 \(\mathbf r_i\) 和二维核协方差 \(K\)，密度为

\[ s(\mathbf r) =\sum_i w_i\,G_K(\mathbf r-\mathbf r_i), \]

\[ G_K(\Delta\mathbf r) = \frac{1}{2\pi\sqrt{\det K}} \exp\left[ -\frac{1}{2} \Delta\mathbf r^T K^{-1}\Delta\mathbf r \right]. \]

密度梯度解析计算：

\[ \nabla s(\mathbf r) = -\sum_i w_i\,G_K(\mathbf r-\mathbf r_i) K^{-1}(\mathbf r-\mathbf r_i). \]

等方核时

\[ K=\sigma^2 I. \]

带仿射演化的发射阶段允许使用完整二维核：

\[ K= \begin{pmatrix} K_{xx} & K_{xy}\\ K_{xy} & K_{yy} \end{pmatrix}. \]

抽象操作

这一层把离散点云变成可查询的连续场：

support points + kernel covariance
  -> density s(x,y)
  -> gradient grad s(x,y)

这个接口同时服务于：

  • 初态密度 initialDensity
  • 发射阶段密度 emissionDensity
  • gradient-response 的 marker 密度 s_em
  • gradient-response 的动力学密度 s_dyn
  • density-normal 和 affine-effective 中的密度法向
  • ROOT debug density map

代码位置

  • include/blastwave/DensityFieldModel.h：WeightedTransversePoint、DensityField、DensityFieldSample 以及 density evaluator 接口。
  • src/FlowFieldDensity.cpp：buildGaussianPointCloudDensityField() 构造等方或完整 covariance kernel 的 Gaussian point-cloud。
  • src/FlowFieldDensity.cpp：evaluateDensityField() 计算 \(s(x,y)\) 和解析 \(\nabla s(x,y)\)。
  • apps/generate_blastwave/RootEventFileWriter.cpp：fillDensityHistogramFromField() 把解析密度场栅格化成 ROOT TH2F debug map。

4.2 density-evolution = affine-gaussian

数学/物理公式

先在 participant-plane 主轴基中分解点坐标：

\[ \Delta\mathbf r_i=\mathbf r_i-\bar{\mathbf r}, \]

\[ x_{{\rm in},i}=\Delta\mathbf r_i\cdot\mathbf e_{\rm in}, \qquad x_{{\rm out},i}=\Delta\mathbf r_i\cdot\mathbf e_{\rm out}. \]

对短轴/长轴做不同仿射膨胀：

\[ \mathbf r_i' =\bar{\mathbf r} +\lambda_{\rm in}\,x_{{\rm in},i}\mathbf e_{\rm in} +\lambda_{\rm out}\,x_{{\rm out},i}\mathbf e_{\rm out}, \]

其中

\[ \lambda_{\rm in}=\texttt{affine-lambda-in}, \qquad \lambda_{\rm out}=\texttt{affine-lambda-out}. \]

Gaussian kernel 也被同一个线性映射拉伸，并额外加入演化平滑：

\[ A=\lambda_{\rm in}\,\mathbf e_{\rm in}\mathbf e_{\rm in}^T +\lambda_{\rm out}\,\mathbf e_{\rm out}\mathbf e_{\rm out}^T, \]

\[ K_f=A(\sigma_{\rm dep}^2 I)A^T+\sigma_{\rm evo}^2 I. \]

其中

\[ \sigma_{\rm dep}=\texttt{flow-density-sigma}, \qquad \sigma_{\rm evo}=\texttt{affine-sigma-evo}. \]

发射密度为

\[ s_f(\mathbf r)=\sum_i w_i\,G_{K_f}(\mathbf r-\mathbf r_i'). \]

发射几何代理把 transformed support cloud 的 covariance 和 kernel covariance 加在一起：

\[ \Sigma_f={\rm Cov}(\{\mathbf r_i'\})+K_f. \]

\[ G_{\rm in}=\frac{\sigma_{{\rm in},f}}{\sigma_{{\rm in},0}}, \qquad G_{\rm out}=\frac{\sigma_{{\rm out},f}}{\sigma_{{\rm out},0}}, \]

\[ \Lambda_{\rm in}=\ln G_{\rm in}, \qquad \Lambda_{\rm out}=\ln G_{\rm out}, \]

\[ \bar\Lambda=\frac{1}{2}(\Lambda_{\rm in}+\Lambda_{\rm out}), \qquad \Delta\Lambda=\frac{1}{2}(\Lambda_{\rm in}-\Lambda_{\rm out}), \]

\[ H_{\rm in}^{\rm eff}=\frac{\Lambda_{\rm in}}{\Delta\tau_{\rm ref}}, \qquad H_{\rm out}^{\rm eff}=\frac{\Lambda_{\rm out}}{\Delta\tau_{\rm ref}}. \]

抽象操作

  -> affine axis transform
  -> anisotropic Gaussian kernel
  -> emissionDensity s_f
  -> emissionGeometry Sigma_f
  -> optional affine-effective closure

注意：participantGeometry 仍保留初态语义；emissionGeometry 是 freeze-out 诊断和流场查询用的几何代理。

6.3 GradientResponse

数学/物理公式

先从 marker density 对应 Gaussian component 抽 marker：

\[ \mathbf r_0\sim \mathcal N(\mathbf r_i,K_{\rm em}). \]

若

\[ s_{\rm em}(\mathbf r_0) < f_{\rm cutoff}\,s_{\rm em}^{\rm scale}, \]

则重新抽样，直到达到重试上限。这里

\[ f_{\rm cutoff}=\texttt{gradient-density-cutoff-fraction}. \]

在 \(\mathbf r_0\) 查询动力学密度梯度，并构造向外响应向量：

\[ \mathbf g = \frac{-\nabla s_{\rm dyn}(\mathbf r_0)} {\max\left(\epsilon,\ s_{\rm dyn}(\mathbf r_0)+f_{\rm floor}s_{\rm dyn}^{\rm scale}\right)}. \]

其中

\[ f_{\rm floor}=\texttt{gradient-density-floor-fraction}. \]

方向为

\[ \hat{\mathbf g}=\frac{\mathbf g}{|\mathbf g|}. \]

确定性位移长度为

\[ \ell_{\rm disp} =d_{\max}\tanh(\kappa_{\rm disp}|\mathbf g|), \]

其中

\[ d_{\max}=\texttt{gradient-displacement-max}, \qquad \kappa_{\rm disp}=\texttt{gradient-displacement-kappa}. \]

最终位移为

\[ \Delta\mathbf r =\ell_{\rm disp}\hat{\mathbf g} +\boldsymbol\xi_{\rm diff}, \]

\[ \boldsymbol\xi_{\rm diff}\sim \mathcal N(0,\sigma_{\rm diff}^2 I). \]

最终发射位置为

\[ \mathbf r_f=\mathbf r_0+\Delta\mathbf r. \]

该 site 的横向速度大小为

\[ \beta_T =v_{\max}\tanh(\kappa_v|\mathbf g|), \]

\[ \boldsymbol\beta_T=\beta_T\hat{\mathbf g}. \]

其中

\[ v_{\max}=\texttt{gradient-vmax}, \qquad \kappa_v=\texttt{gradient-velocity-kappa}. \]

抽象操作

participant anchor
  -> sample marker r0 from s_em
  -> query -grad(s_dyn)/(s_dyn + floor)
  -> bounded displacement r0 -> rf
  -> bounded transverse beta_T saved on EmissionSite

写出语义：

  • source_x/source_y = participant anchor
  • x0/y0 = marker initial position r0
  • x/y = final emission position rf
  • emission_weight = site weight before optional particle-level mt-cosh multiplier

9.3 flow-velocity-sampler = density-normal

数学/物理公式

先在发射密度上取向外 density normal：

\[ \hat{\mathbf n}_{s} = -\frac{\nabla s_f(\mathbf r_f)} {|\nabla s_f(\mathbf r_f)|}. \]

若局部密度太平或梯度数值退化，则 fallback 到 covariance normal：

\[ \hat{\mathbf n}_{s}\rightarrow \hat{\mathbf n}_{\Sigma}. \]

默认未显式配置新方向/半径键时，流强度只依赖协方差广义半径：

\[ \rho_t=\rho_{\rm trans}\tilde r^\alpha. \]

若显式启用 density-normal-kappa-compensation，并且处于 affine-gaussian + density-normal，则额外乘上 V1a 二阶响应：

\[ \rho_t = \rho_{\rm trans}\tilde r^\alpha \exp\left[ 2\kappa_2\epsilon_{2,0} \cos 2(\phi_B-\Psi_{2,0}) \right]. \]

若显式配置 flow-trans-direction-gradient-fraction = f_g 或 flow-trans-radius，则 \(f_g\) 表示局域梯度响应相对全局膨胀约束的份额。椭圆、三角和 hotspot 形状仍由 density-gradient 外向方向 \(\hat{\mathbf n}_{s}\) 给出；全局膨胀只作为外向投影下限，抑制热点造成的反向流。

先定义几何径向单位矢量

\[ \hat{\mathbf n}_{r} = \frac{\mathbf r-\mathbf c}{|\mathbf r-\mathbf c|}, \]

以及梯度相对几何径向的投影

\[ c=\hat{\mathbf n}_{s}\cdot\hat{\mathbf n}_{r},\qquad c_{\min}=1-f_g. \]

显式 flow-trans 方向为

\[ \hat{\mathbf n}_{\rm flow} = \begin{cases} \hat{\mathbf n}_{r}, & f_g=0 \ \text{且几何径向有效},\\ \hat{\mathbf n}_{s}, & f_g=1 \ \text{且 density gradient 有效},\\ \hat{\mathbf n}_{s}, & 0<f_g<1,\ c\ge c_{\min},\\ c_{\min}\hat{\mathbf n}_{r} +\sqrt{1-c_{\min}^2}\,\hat{\mathbf t}, & 0<f_g<1,\ c<c_{\min},\\ \hat{\mathbf n}_{r}, & \text{density gradient 或切向方向退化但几何径向有效},\\ \hat{\mathbf n}_{\Sigma}, & \text{上述方向均退化时的 covariance normal fallback}. \end{cases} \]

其中

\[ \hat{\mathbf t} = \frac{\hat{\mathbf n}_{s}-c\hat{\mathbf n}_{r}} {|\hat{\mathbf n}_{s}-c\hat{\mathbf n}_{r}|}. \]

因此 \(1-f_g\) 不是一个会均匀稀释各向异性流的方向混合权重，而是一个最小外向投影；只有当局域梯度方向太向内或太横向时才把它投影到全局膨胀 cone 的边界。默认 radius-profile 模式下总 baseline \(\rho_{\rm trans}\xi_{\rm flow}^{\alpha}\) 保持不变。

半径变量拆成 density shell 坐标和主流强坐标。对 covariance 半径，

\[ \xi_{\rm flow}=\tilde r. \]

对 density-defined 半径，先用发射密度边界定义 shell 坐标

\[ \xi_{\rm shell} = \begin{cases} r/r_p(\phi), & \texttt{flow-trans-radius = density-percentile:p},\\ r/r_{\rm level}(\phi), & \texttt{flow-trans-radius = density-level:fraction}, \end{cases} \]

其中 \(r_p(\phi)\) 是沿角向射线累计发射密度达到分位数 \(p\) 的边界，\(r_{\rm level}(\phi)\) 是发射密度降到 \(\texttt{fraction}\times\texttt{emissionDensityScale}\) 的边界，并要求 \(0<\texttt{fraction}<1\)。主流强再换算到 covariance/sigma-equivalent 尺度：

\[ \xi_{\rm flow}=q\,\xi_{\rm shell}, \]

\[ q= \begin{cases} \sqrt{-2\log(1-p)}, & \texttt{density-percentile:p},\\ \sqrt{-2\log(f)}, & \texttt{density-level:f}. \end{cases} \]

也就是说，density-defined \(R(\phi)\) 只决定角向几何形状；flow-trans-rho0 对应 covariance-equivalent 的 \(\xi_{\rm flow}=1\) 强度尺度。

flow-trans-radius-resolution 只控制这个事件级角向边界表的数值网格：

\[ \texttt{balanced}=240\times256,\qquad \texttt{precise}=360\times512,\qquad \texttt{fast}=120\times128. \]

这里第一维是角向采样点数，第二维是每条射线的径向采样点数。该配置不改变 \(R(\phi)\)、\(\xi_{\rm shell}=r/R(\phi)\)、\(q\) 或横向流强公式本身；precise 保留旧的 \(360\times512\) 数值路径，balanced 是新的默认折中。density-defined 半径中的 \(R(\phi)\) 是局域密度几何参考尺度，不是最大流强边界；代码对有限且非负的 \(\xi_{\rm shell}\) 或 \(\xi_{\rm flow}\) 不做上限截断，并使用

\[ \rho_t=\rho_{\rm trans}\xi_{\rm flow}^{\alpha}. \]

这里 \(\rho_{\rm trans}=\texttt{flow-trans-rho0}\) 表示 \(\xi_{\rm flow}=1\) 的快度尺度。最终速度仍走通用保护 \(\beta_T=\min(0.95,\tanh\max(0,\rho_t))\)。

若进一步选择

flow-trans-magnitude-mode = shell-gradient-corrected

则流强保持同一个主径向项，但乘上壳层累计梯度修正。该模式只允许 density-normal + density-percentile/level，因为它需要上面定义的 \(R(\phi)\)。修正表只在边界 profile 内定义，因此 lookup 使用

\[ s_{\rm lookup}=\min(\max(\xi_{\rm shell},0),1), \]

这只限制 correction-profile 的插值域，不限制基础 \(\rho_{\rm trans}\xi_{\rm flow}^{\alpha}\)。

先沿每条角向射线构造累计量：

\[ A(\phi,s_{\rm lookup})= \int_0^{s_{\rm lookup}R(\phi)} \max\left( 0, \frac{-\nabla s_f(\mathbf c+t\hat{\mathbf e}_\phi)\cdot\hat{\mathbf e}_\phi} {\max(s_f(\mathbf c+t\hat{\mathbf e}_\phi), f_{\rm floor}s_{\rm scale})} \right)dt. \]

事件级归一化为

\[ \tilde A= \frac{A-\langle A\rangle_{\rm event}} {\mathrm{RMS}_{\rm event}}, \]

然后在相同 \(s\) 壳层扣除角向平均，得到乘法修正：

\[ \delta= f_g\lambda_A \left[ \tilde A(\phi,s_{\rm lookup})-\langle\tilde A\rangle_{s_{\rm lookup}} \right], \qquad \delta\in[-\delta_{\max},+\delta_{\max}]. \]

最终快度为

\[ \rho_t= \rho_{\rm trans}\xi_{\rm flow}^{\alpha}(1+\delta). \]

其中 \(f_g=\texttt{flow-trans-direction-gradient-fraction}\)， \(\lambda_A=\texttt{flow-trans-gradient-strength}\)， \(f_{\rm floor}=\texttt{flow-trans-gradient-density-floor-fraction}\)， \(\delta_{\max}=\texttt{flow-trans-gradient-max-factor-delta}\)。当 \(\lambda_A=0\)、event RMS 退化或 correction cache 无效时，该模式退回主径向项。

抽象操作

emissionDensity gradient
  -> pure geometric-expansion direction at gradient share = 0
  -> pure density-gradient direction at gradient share = 1
  -> density-gradient direction projected into the global-expansion outward cone for 0 < share < 1
  -> covariance normal fallback if flat
  -> density-defined xi_shell and sigma-equivalent xi_flow if requested
  -> optional shell-gradient correction at clamped xi_shell table lookup
  -> flow-trans-rho0 * radius_profile^flow-trans-profile-power
  -> optional kappa2 compensation
  -> beta vector

这个 sampler 的物理意图是：方向各向异性主要来自 freeze-out density 的几何梯度，而不是显式强行叠加的解析二阶项。

9.4 flow-velocity-sampler = affine-effective

该 sampler 只允许配合 density-evolution = affine-gaussian。它先用第 4.2 节的闭合量恢复

\[ H_{\rm in}^{\rm eff},\qquad H_{\rm out}^{\rm eff}. \]

令

\[ \bar H=\frac{1}{2} (H_{\rm in}^{\rm eff}+H_{\rm out}^{\rm eff}), \]

\[ \bar R=\sqrt{\sigma_{{\rm in},f}\sigma_{{\rm out},f}}. \]

并在 freeze-out 主轴基中写

\[ x'=(\mathbf r_f-\bar{\mathbf r}_f)\cdot\mathbf e_{\rm in}, \]

\[ y'=(\mathbf r_f-\bar{\mathbf r}_f)\cdot\mathbf e_{\rm out}. \]

affine-effective-mode = additive-rho

该模式保留 density-normal 方向，并把 affine 几何修正加到 \(\rho_0\) baseline 上。

局域 affine 几何速度代理：

\[ u_{\rm geom} = \kappa_{\rm flow}\,\tilde r^\alpha \sqrt{ (H_{\rm in}^{\rm eff}x')^2 + (H_{\rm out}^{\rm eff}y')^2 }. \]

各向同性参考项：

\[ u_{\rm iso} = \kappa_{\rm flow}\,\tilde r^\alpha \bar H\,\tilde r\,\bar R. \]

速度代理先被 affine-u-max 截断：

\[ u_{\rm geom}^{\rm clip}=\min(u_{\rm geom},u_{\max}), \]

\[ u_{\rm iso}^{\rm clip}=\min(u_{\rm iso},u_{\max}). \]

几何修正写成流快度差：

\[ \Delta\rho_{\rm geom} = \operatorname{atanh}(u_{\rm geom}^{\rm clip}) - \operatorname{atanh}(u_{\rm iso}^{\rm clip}). \]

最终

\[ \rho_t = \max\left( 0,\, \rho_0\tilde r^\alpha+\Delta\rho_{\rm geom} \right), \]

\[ \beta_T = \min\left(u_{\max},\tanh\rho_t\right). \]

方向仍是 density-normal 方向。

affine-effective-mode = full-tensor

该模式直接把主轴膨胀率映射成横向速度张量：

\[ u_{\rm in}' = \kappa_{\rm flow}\tilde r^\alpha H_{\rm in}^{\rm eff}x', \]

\[ u_{\rm out}' = \kappa_{\rm flow}\tilde r^\alpha H_{\rm out}^{\rm eff}y'. \]

旋回 lab frame：

\[ \boldsymbol\beta_T = u_{\rm in}'\mathbf e_{\rm in} + u_{\rm out}'\mathbf e_{\rm out}. \]

若

\[ |\boldsymbol\beta_T|>u_{\max}, \]

则整体缩放到 \(u_{\max}\)。该模式中 flow-trans-rho0 不参与速度公式。

抽象操作

affine-gaussian initial/final semi-axes
  -> effective expansion rates H_in/out
  -> additive-rho: flow-trans-rho0 baseline + geometry rapidity correction
  -> full-tensor: principal-axis velocity tensor
  -> clipped beta vector

affine-kappa-aniso 当前只是 legacy/no-op 兼容参数。

  • src/FlowFieldModel.cpp：computeAffineEffectiveSharedTerms() 计算 \(H_{\rm in/out}^{\rm eff}\)、\(\bar H\)、\(\bar R\)。
  • src/FlowFieldModel.cpp：evaluateAffineEffectiveAdditiveRho() 实现 additive-rho 公式。
  • src/FlowFieldModel.cpp：evaluateAffineEffectiveFullTensor() 实现 full-tensor 公式。
  • src/FlowFieldModel.cpp：evaluateAffineEffectiveFlow() 根据 affineEffectiveMode 调度两个子模式。

10. Lorentz boost 到实验室系

数学/物理公式

给定局域静止系四动量

\[ p^{*\mu}=(E^*,\mathbf p^*) \]

和流体元速度

\[ \boldsymbol\beta=(\beta_x,\beta_y,\beta_z), \]

\[ \beta^2=\boldsymbol\beta\cdot\boldsymbol\beta, \qquad \gamma=\frac{1}{\sqrt{1-\beta^2}}. \]

\[ \mathbf p = \mathbf p^* + \frac{\gamma-1}{\beta^2} (\boldsymbol\beta\cdot\mathbf p^*)\boldsymbol\beta + \gamma E^*\boldsymbol\beta. \]

\[ E=\gamma(E^*+\boldsymbol\beta\cdot\mathbf p^*). \]

local thermal momentum + fluid velocity
  -> lab-frame particle momentum

这一层是最终动量各向异性进入 particles.px/py/pz/E 的地方。

12.3 事件级 events.v2

12.4 末态三阶 Q-vector 与初态平面投影

初态平面来自第 2 节的几何定义：\(\Psi_2\) 是 covariance participant-plane，\(\Psi_3\) 是 recentered \(n=3\) harmonic plane。事件接受粒子后，末态粒子同时累积二阶和三阶 Q-vector：

\[ Q_2=\sum_j w_j e^{2i\varphi_j},\qquad Q_3=\sum_j w_j e^{3i\varphi_j}. \]

除了

\[ v_n=\frac{|Q_n|}{\sum_j w_j}, \]

还保存相对初态几何平面的投影：

\[ v_{2|\Psi_2} = \frac{\operatorname{Re}(Q_2 e^{-2i\Psi_2})}{\sum_j w_j}, \qquad v_{3|\Psi_3} = \frac{\operatorname{Re}(Q_3 e^{-3i\Psi_3})}{\sum_j w_j}. \]

二阶 Q-vector 同时写出 lab-frame 分量：

\[ v_{2,x}^{\rm lab}=\frac{\operatorname{Re}Q_2}{\sum_j w_j},\qquad v_{2,y}^{\rm lab}=\frac{\operatorname{Im}Q_2}{\sum_j w_j}. \]

ROOT branch 名分别是 v2_lab_x 和 v2_lab_y，并满足

\[ v_2=\sqrt{(v_{2,x}^{\rm lab})^2+(v_{2,y}^{\rm lab})^2},\qquad v_{2|\Psi_2}=v_{2,x}^{\rm lab}\cos2\Psi_2+v_{2,y}^{\rm lab}\sin2\Psi_2. \]

这些几何平面投影在 ROOT 中写为 v2_wrt_psi2 和 v3_wrt_psi3。它们用于 response test，因为初态几何平面已知，不需要先做末态 event-plane reconstruction。lab-frame 分量只用于固定坐标下的符号和旋转诊断，主 response 解释仍优先使用 v2_wrt_psi2。 对应的四张 response/cross-talk TH2 使用完整存储范围 epsilon = 0..1、v_{n|\Psi_n} = -1..1，但默认显示窗口缩到 epsilon = 0..0.35、v_{n|\Psi_n} = -0.15..0.15，方便直接查看当前 response-test 的点云密集区。

若 eps2 与 eps3 在生成样本中相关，raw cross slope 会混入同阶响应项。notebooks/vn_epsn_multivariate_regression.ipynb 对同一个 events tree 做条件线性响应拟合：

\[ v_{2|\Psi_2}=c_2+k_{22}\epsilon_2+k_{23}\epsilon_3, \qquad v_{3|\Psi_3}=c_3+k_{32}\epsilon_2+k_{33}\epsilon_3. \]

其中 \(k_{23}\) 和 \(k_{32}\) 是固定另一个 eccentricity 后的条件交叉响应估计。notebook 同时报告 two-step 扣除量

\[ \hat k_{23}^{\rm two-step} = \mathrm{slope}(v_{2|\Psi_2}\sim\epsilon_3) - \mathrm{slope}(v_{2|\Psi_2}\sim\epsilon_2)\, \mathrm{slope}(\epsilon_2\sim\epsilon_3), \]

\[ \hat k_{32}^{\rm two-step} = \mathrm{slope}(v_{3|\Psi_3}\sim\epsilon_2) - \mathrm{slope}(v_{3|\Psi_3}\sim\epsilon_3)\, \mathrm{slope}(\epsilon_3\sim\epsilon_2), \]

作为和直觉扣除法的对照；最终解释优先使用二维回归系数。

13. 差分 v_n{2}(pT) 分析

数学/物理公式

该分析是可选 payload，配置 v2pt-bins 会开启 \(n=2\)，配置 v3pt-bins 会开启 \(n=3\)。生成时可 joint analysis，也可用 analyze_blastwave_vnpt 后处理。

每个事件中，参考粒子是该事件所有粒子。单位 track weight 固定为 1，当前故意忽略 particles.emission_weight。令

\[ Q_n=\sum_{\rm ref} e^{in\phi},\quad n\in\{2,3\}. \]

对一个 \(p_T\) bin，POI Q-vector 为

\[ P_n(p_T{\rm\ bin}) = \sum_{\rm POI\ in\ bin}e^{in\phi}. \]

积分参考 cumulant：

\[ c_n\{2\} = \frac{ \sum_{\rm events}\left(|Q_n|^2-M\right) }{ \sum_{\rm events}M(M-1) }, \]

其中 \(M\) 是事件粒子数。

差分 cumulant：

\[ d_n\{2\}(p_T) = \frac{ \sum_{\rm events} \left( {\rm Re}\left[P_n(p_T)Q_n^*\right]-m_{p_T} \right) }{ \sum_{\rm events}m_{p_T}(M-1) }, \]

其中 \(m_{p_T}\) 是该事件中落入该 \(p_T\) bin 的 POI 数。

最终

\[ v_n\{2\}(p_T) = \frac{d_n\{2\}(p_T)} {\sqrt{c_n\{2\}}}. \]

误差使用 leave-one-event-out jackknife。若共有 \(N\) 个 jackknife 样本，记第 \(a\) 个 leave-one-out 结果为 \(v_a\)，均值为 \(\bar v\)，则

\[ \sigma_v = \sqrt{ \frac{N-1}{N} \sum_{a=1}^{N}(v_a-\bar v)^2 }. \]

抽象操作

particles tree or live particles
  -> unit-weight Qn and Pn accumulators for enabled n=2/3
  -> integrated cn{2}
  -> differential dn{2}(pT)
  -> v_n{2}(pT) histogram + jackknife errors

它和 events.v2/events.v3 是不同对象：

  • events.v2/events.v3 是逐事件 Q-vector 模长摘要，默认可被 emission_weight 影响。
  • v2{2}(pT) / v3{2}(pT) 是跨事件二粒子 cumulant 差分分析，当前固定单位 track weight。

代码位置

  • include/blastwave/DifferentialFlowCumulant.h：harmonic-aware track 输入、结果结构和 DifferentialFlowCumulant 接口。
  • src/DifferentialFlowCumulant.cpp：addEvent() 实现 \(Q_n\)、\(P_n\)、self-correlation subtraction 和事件贡献缓存；finalize() 实现 \(c_n\{2\}\)、\(d_n\{2\}\)、v_n{2}(pT) 和 jackknife error。
  • include/blastwave/io/DifferentialFlowRootPayload.h 与 src/DifferentialFlowRootPayload.cpp：按 harmonic 读写 v2_2_pt_* / v3_2_pt_* ROOT payload。
  • apps/generate_blastwave/RootEventFileWriter.cpp：生成时 joint analysis 的 DifferentialFlowCumulant 累积和 payload 写出。
  • apps/analyze_blastwave_vnpt.cpp：独立后处理入口，从 ROOT 文件读 particles 和已存在的 v2_2_pt_edges / v3_2_pt_edges 后写出分析 payload。

15. 端到端物理抽象总结

从物理抽象上，当前生成器做的是：

  1. 用 Woods-Saxon + black-disk \(NN\) 碰撞判据得到逐事件 participant 初态。
  2. 用 participant covariance 得到 \(\epsilon_{2,0}\)、\(\Psi_{2,0}\)、主轴和广义半径度量，并用 recentered harmonic moment 得到 \(\epsilon_{3,0}\)、\(\Psi_{3,0}\)。
  3. 用 Gaussian point-cloud 构造连续横向密度。
  4. 根据 density-evolution 把初态密度变成发射阶段密度，或在 gradient-response 中把演化推到逐粒子响应。
  5. 根据 participant multiplicity 产生 emission sites。
  6. 对每个 site 抽 \(\eta_s\)、局域热动量和横向流速度。
  7. 用 Lorentz boost 得到 lab-frame 粒子四动量。
  8. 从粒子和发射点云回填事件级几何、源尺寸和末态 flow 摘要。
  9. 把事件、participant、粒子和可选 QA/analysis 对象写入 ROOT。

因此代码里的核心抽象不是“完整 hydrodynamic evolution”，而是：

\[ \text{participant geometry} \rightarrow \text{event medium} \rightarrow \text{emission sites} \rightarrow \text{local thermal momentum + flow boost} \rightarrow \text{ROOT observables}. \]
