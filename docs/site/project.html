Blast-Wave 项目说明

这是什么

这是一个用 C++17 和 ROOT 实现的小型 Blast-Wave 事例生成器。它的目标不是做完整重离子末态模拟，而是在一个足够小、足够清晰、可继续扩展的代码骨架里，完成以下事情：

  • 用 Monte Carlo Glauber 产生逐事例的 participant 几何涨落
  • 在 participant hotspot 上做简化的 blast-wave 发射
  • 通过 ROOT 输出事件树、粒子树和基础 QA 直方图
  • 提供一个独立 QA 程序检查输出是否合法、接口是否稳定

当前第一版只生成单一直接 pi+，适合做几何和流场方向的快速验证，不适合直接拿去和实验末态全带电粒子分布逐项对比。

当前能力边界

已经支持：

  • 固定冲量参数 b 的事件生成
  • 由 b 线性近似得到的事件级 centrality
  • Pb-Pb 风格 Woods-Saxon 核几何
  • participant eccentricity eps2 和 participant plane psi2
  • 基于末态粒子二阶 Q-vector 的事件级 v2，以及固定 lab 坐标下的 v2_lab_x/v2_lab_y 分量诊断
  • opt-in response-test-023 初态几何模板，用 0+2+3 人工点云做二阶/三阶响应测试
  • 初态三阶几何和末态三阶响应摘要：eps3/psi3、v3、v2_wrt_psi2、v3_wrt_psi3
  • 局域热动量抽样
  • 局域流场 + 正确 Lorentz boost
  • opt-in affine-effective 横向流速采样：从 affine-gaussian 的初末态半轴闭合恢复有效膨胀率
  • opt-in V2 梯度响应：用 participant density 梯度同时生成横向位移和横向流速
  • ROOT 输出与独立 QA
  • CMake 构建

还没有做：

  • 最小偏置 b 抽样
  • 共振衰变
  • 多粒子种类
  • hadronic rescattering
  • 完整 Cooper-Frye freeze-out 超曲面采样

目录结构

  • include/blastwave/BlastWaveGenerator.h 对外数据结构和生成器类声明。
  • src/BlastWaveGenerator.cpp 单事件生成编排。
  • include/blastwave/DensityFieldModel.h Gaussian point-cloud density 与解析 gradient 接口。
  • include/blastwave/EventMedium.h 事件级介质状态，区分 participant 几何、初始 density、V2 marker/dynamics density、发射阶段 density 和发射阶段几何。
  • include/blastwave/EmissionSampler.h 横向发射点抽样接口，支持 participant hotspot、density-field 和 V2 gradient-response backend。
  • include/blastwave/FlowFieldModel.h 基于事件介质的流体元速度抽样接口。
  • apps/generate_blastwave_events.cpp 命令行生成程序，负责写 ROOT 文件。
  • apps/qa_blastwave_output.cpp 独立 QA 程序，负责读取 ROOT 文件并做一致性检查。
  • CMakeLists.txt 主构建文件。
  • CMakePresets.json 默认 CMake preset，同时开启 compile_commands.json 输出。
  • config/ 受版本控制的示例配置文件。
  • scripts/ 本地运行辅助脚本，与静态配置分开存放。
  • reference/legacy-root-macros/ 历史参考宏，只用于理解物理意图，不应直接复制代码。
  • qa/ 本地测试输出样本。
  • res/ 正式运行结果。
  • docs/ 当前说明、公式说明、简明手记和历史计划归档。
  • project-state/ 仓库内工程协作台账。

如何编译

本项目默认使用当前机器上的 O2Physics/ROOT 环境来编译。

在项目根目录执行：

/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc 'cmake --preset default -S /Users/allenzhou/Research_software/Blast_wave'"
/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc 'cmake --build /Users/allenzhou/Research_software/Blast_wave/build'"

构建完成后会得到：

  • bin/generate_blastwave_events
  • bin/analyze_blastwave_vnpt
  • bin/qa_blastwave_output
  • bin/test_maxwell_juttner_sampler
  • bin/test_flow_field_model
  • bin/test_emission_sampler
  • bin/test_run_options
  • bin/test_differential_flow_cumulant
  • bin/test_physics_utils
  • bin/test_output_path_utils
  • build/compile_commands.json

其中 compile_commands.json 已经开启，便于编辑器、clangd 和静态分析工具使用。

ROOT-free core 回归测试可直接在 build 目录执行：

cd /Users/allenzhou/Research_software/Blast_wave/build && ctest --output-on-failure

如何运行

1. 生成事例

示例：

/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc '/Users/allenzhou/Research_software/Blast_wave/bin/generate_blastwave_events --nevents 5000 --b 8 --output /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_5000.root'"

也可以直接传配置文件：

/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc '/Users/allenzhou/Research_software/Blast_wave/bin/generate_blastwave_events /Users/allenzhou/Research_software/Blast_wave/config/test_b8.cfg --output /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_override.root'"

仓库里已经附带了一份示例配置：config/test_b8.cfg。

另外还有一份 opt-in affine-effective 示例：config/test_b8_affine_effective.cfg。

三阶响应测试可使用完整中文说明配置：config/test_023_dense.cfg。

/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc '/Users/allenzhou/Research_software/Blast_wave/bin/generate_blastwave_events /Users/allenzhou/Research_software/Blast_wave/config/test_023_dense.cfg --output /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_response_023.root'"

宽分布 response-test 可使用 config/test_023_fluctuating.cfg。若要隔离检查二阶/三阶模板随机化是否引入交叉相关，可使用 config/test_023_dense_eps2_only_fluct.cfg 和 config/test_023_dense_eps3_only_fluct.cfg；这两个控制配置只打开一个模板权重区间，其它逐事件随机区间都退化为固定值。独立源池诊断配置 config/test_023_dense_fluct_independent_pools.cfg 使用 initial-geometry-source-allocation = independent-pools，用于检查去掉固定总源数竞争后的残余相关；当前验证显示它仍未达到 |corr(eps2,eps3)| < 0.08 的解耦目标。

独立源池的流场对比配置包括 config/test_023_dense_fluct_independent_pools.cfg、config/test_023_dense_mix_fluct_independent_pools.cfg、config/test_023_dense_newrap_fluct_independent_pools.cfg 和 config/test_023_ellipse_fluct_independent_pools.cfg。这些配置使用同一组 fluctuating 0+2+3 初态随机区间，只改变流场/半径 profile 选择，便于在同一个几何相关背景下比较 raw cross slope 与条件交叉响应。

2. 做 QA 校验

示例：

/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc '/Users/allenzhou/Research_software/Blast_wave/bin/qa_blastwave_output --input /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_5000.root --output /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_5000_validation.root --expect-nevents 5000'"

如果一切正常，QA 程序会输出 validation_passed ...，并写出一个新的 QA ROOT 文件。

3. 独立做 v_n{2}(pT) 分析

如果生成时已经把 v2_2_pt_edges 或 v3_2_pt_edges 写入主结果文件，也可以后处理。程序会读取文件里存在的 harmonic bin metadata，并计算对应的 v2{2}(pT) / v3{2}(pT)：

/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc '/Users/allenzhou/Research_software/Blast_wave/bin/analyze_blastwave_vnpt --input /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_5000.root --output /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_5000_flowpt.root'"
/bin/zsh -lc "alienv setenv O2Physics/latest-master-o2 -c sh -lc '/Users/allenzhou/Research_software/Blast_wave/bin/analyze_blastwave_vnpt --input /Users/allenzhou/Research_software/Blast_wave/qa/test_b8_5000.root --inplace'"

默认情况下，不传 --output 且不使用 --inplace 时，程序会写同目录兄弟文件 <input-stem>_flowpt.root。

4. 做事件级 v_n-epsilon_n 线性回归

交互式分析可使用 notebooks/vn_epsn_regression.ipynb。若要同时查看 normal diagonal response，并从 raw v2-eps3 / v3-eps2 交叉斜率中扣除 eps2-eps3 的线性几何相关，可使用 notebooks/vn_epsn_multivariate_regression.ipynb；该 notebook 默认读取四个 *_fluct_independent_pools.root 文件，并拟合

v2_wrt_psi2 = c2 + k22 * eps2 + k23 * eps3
v3_wrt_psi3 = c3 + k32 * eps2 + k33 * eps3

其中 k22/k33 是固定另一个 eccentricity 后的条件对角响应估计，k23/k32 是条件交叉响应估计。notebook 会画 raw 与 partial-regression 两组图：对角项 v2~eps2、v3~eps3，以及交叉项 v2~eps3、v3~eps2。推荐先创建 root_notebook conda 环境：

conda env create -f notebooks/environment-notebook_with_root.yml
conda activate root_notebook
python -m ipykernel install --user --name root_notebook --display-name "root_notebook"

notebook 的 INPUT_FILES 支持多文件比较，每项写成 (ROOT 文件路径, 图例标签)。默认会尝试读取 qa/test_b8_response_023.root、qa/test_b8_response_023_mix.root 和 qa/test_b8_023_newrap.root 中当前存在且 schema 合格的文件；也可以在第一段路径配置里改成任意当前 schema 的 Blast_wave ROOT 结果文件列表。主分析量是：

  • events.eps2 对 events.v2_wrt_psi2，用于同阶 v_n/epsilon_n response
  • events.eps3 对 events.v3_wrt_psi3，用于同阶 v_n/epsilon_n response
  • events.eps3 对 events.v2_wrt_psi2、events.eps2 对 events.v3_wrt_psi3，用于交叉阶 v_n/epsilon_m mixing 检查

这里使用相对 participant plane 的带符号投影 v2_wrt_psi2 / v3_wrt_psi3，而不是非负模长 v2 / v3，因为前者更直接对应线性响应斜率 kappa_n。回归直接使用 events tree 的原始 event rows；图中的 binned mean 只作为视觉辅助，不参与拟合。环境文件现在只要求 uproot 与常规 Python 分析栈，不再安装完整 PyROOT；如需检查 ROOT 对象结构，优先用 uproot 打开文件和列出对象。

如果当前 ROOT 文件包含 events.v2_lab_x 和 events.v2_lab_y，两个 regression notebook 会额外运行 lab-frame V2 诊断：v2_lab_x 是固定 lab-x 方向的 mean(cos(2*phi))，最适合检查 x/y 朝向和符号；v2_lab_y 是 mean(sin(2*phi))，用于检查整体旋转或非对齐。它们不替代 v2_wrt_psi2 作为主 response 量。

以下参数仍然可以继续直接通过 CLI 传入。

基础参数：

  • --nevents <int> 生成多少个事件。
  • --b <fm> 冲量参数，单位 fm。
  • --temperature <GeV> 局域静止系热动量抽样的温度尺度。
  • --thermal-sampler <maxwell-juttner|gamma> 热动量大小抽样模式；默认 maxwell-juttner，gamma 保留为兼容模式。
  • --mj-pmax <GeV> Maxwell-Juttner 查表的最大动量截断。
  • --mj-grid-points <int> Maxwell-Juttner 查表网格点数。
  • --tau0 <fm/c> 固定冻结 proper time。
  • --smear <fm> participant hotspot 的横向高斯展宽。
  • --sigma-nn <fm^2> 核子-核子非弹截面参数，默认 7.0 fm^2。
  • --seed <uint64> 随机数种子。
  • --output <path> 输出 ROOT 文件路径。
  • --v2pt-bins <comma-separated edges> 开启差分二粒子 cumulant v2{2}(pT)，分 bin 约定为 [edge_i, edge_{i+1})，最后一 bin 右端闭合。
  • --v3pt-bins <comma-separated edges> 开启差分二粒子 cumulant v3{2}(pT)，分 bin 约定同 v2pt-bins。
  • --flowpt-output-mode <same-file|separate-file> 控制启用的 v2{2}(pT) / v3{2}(pT) payload 是写回主结果文件，还是一起写到独立分析文件；默认 same-file。
  • --flowpt-output <path> 仅在 flowpt-output-mode = separate-file 时有效；若省略，则默认写成主输出旁边的 <output-stem>_flowpt.root。
  • --progress 强制显示进度条。进度条会每秒更新一个活动符号，并按已完成事件的平均耗时给出粗略 ETA。
  • --no-progress 强制关闭进度条。
  • --initial-geometry <glauber|response-test-023> 选择初态几何来源。默认 glauber 保持 Monte Carlo Glauber 流程；response-test-023 生成可控 0+2+3 人工横向点云，并把 participants.nucleus_id 写为 -1。
  • --initial-geometry-source-allocation <ratio-total|independent-pools> 选择 response-test-023 的源点分配模式。默认 ratio-total 保持旧行为：initial-geometry-source-count 是总源点数，按 1:A2:A3 分配到三类分量。independent-pools 把 initial-geometry-source-count 解释为圆形背景池 N0，并使用 N2=round(N0*A2)、N3=round(N0*A3)；若 A3>0 且取整后少于 3，则三角池补到 3 个源点。因此该模式下总 Npart 变为 N0+N2+N3，不再等于 source-count。
  • --initial-geometry-fluctuate 仅对 response-test-023 开启逐事件模板参数随机化；默认关闭。Glauber 下启用会直接报错。
  • --no-initial-geometry-fluctuate 显式关闭逐事件模板参数随机化；关闭时所有 *-min/max 范围即使被解析也不参与生成，固定 023 继续作为 closure baseline。
  • --initial-geometry-source-count <int> response-test-023 的横向源点数参数。ratio-total 下它是总源点数；independent-pools 下它是圆形背景池 N0。
  • --initial-geometry-source-count-min/max <int> 逐事件随机化时 source-count 参数的闭区间，必须成对配置且为正整数；其物理含义随 initial-geometry-source-allocation 改变。
  • --initial-geometry-r0 <fm> n=0 圆形高斯背景宽度。
  • --initial-geometry-a2 <value> n=2 模板权重；它只控制生成点云，不等同于 eps2。
  • --initial-geometry-a2-min/max <value> 逐事件随机化时 A2 模板权重的区间；这是模板权重范围，不是物理 eps2 范围。
  • --initial-geometry-r2x <fm> n=2 椭圆高斯局域 x 主轴宽度。
  • --initial-geometry-r2x-min/max <fm> 逐事件随机化时 r2x 的正值区间，单位 fm。
  • --initial-geometry-r2y <fm> n=2 椭圆高斯局域 y 主轴宽度；与 r2x 和 phi2 联合定义椭圆模板。
  • --initial-geometry-r2y-min/max <fm> 逐事件随机化时 r2y 的正值区间，单位 fm。
  • --initial-geometry-phi2 <rad> n=2 椭圆模板方向角。
  • --initial-geometry-a3 <value> n=3 三角 hotspot 模板权重；它只控制生成点云，不等同于 eps3。
  • --initial-geometry-a3-min/max <value> 逐事件随机化时 A3 模板权重的区间；这是模板权重范围，不是物理 eps3 范围。
  • --initial-geometry-r3 <fm> 三个 hotspot 中心到原点的半径。
  • --initial-geometry-r3-min/max <fm> 逐事件随机化时 r3 的正值区间，单位 fm。
  • --initial-geometry-sigma3 <fm> 每个三角 hotspot 的高斯宽度。
  • --initial-geometry-sigma3-min/max <fm> 逐事件随机化时 sigma3 的正值区间，单位 fm。

调参与 QA 相关参数：

  • --flow-trans-rho0 <value> 椭圆法向流场的各向同性快度基线；在 density-defined flow-trans-radius 下是 covariance-equivalent xi_flow = 1 处的横向流快度尺度，不是 density 边界处的最大流强。在 affine-effective-mode = additive-rho 下也作为 baseline 平均流强参与闭合。
  • --kappa2 <value> 二阶流响应系数；事件级二阶快度调制振幅为 a2 = kappa2 * eps2_initial。对于 affine-gaussian + density-normal，只有在开启 density-normal-kappa-compensation 时才参与显式补偿。
  • --flow-trans-profile-power <value> 横向流强半径 profile 的幂指数；默认 covariance 半径使用 rTilde，density-defined 半径用 R_density(phi) 决定角向几何，再用 xi_flow = q * r / R_density(phi) 进入主流强公式。
  • --flow-trans-magnitude-mode <radius-profile|shell-gradient-corrected> density-normal 流强模式。默认 radius-profile 保持 rhoRaw = flowTransRho0 * pow(xiFlow, flowTransProfilePower)；shell-gradient-corrected 只允许与 flow-velocity-sampler = density-normal 且 flow-trans-radius = density-percentile:<p> 或 density-level:<fraction> 联合使用。
  • --flow-trans-gradient-strength <value> shell-gradient-corrected 的累计梯度修正强度 \(\lambda_A\)。默认 0.0，在 corrected mode 下合法并退化为主径向项；若显式设置但未选择 shell-gradient-corrected 会报错。
  • --flow-trans-gradient-density-floor-fraction <value> 计算累计梯度代理时的密度 floor 比例 \(f_{\rm floor}\)，分母使用 max(density, f_floor * emissionDensityScale)；必须 finite 且大于 0，并且只在 shell-gradient-corrected 下可显式设置。
  • --flow-trans-gradient-max-factor-delta <value> 限制乘法修正因子偏离量 \(\delta\)，最终因子为 1 + delta，要求 0 <= value < 1。它不是快度加法项上限；显式设置时也要求 shell-gradient-corrected。
  • --flow-trans-direction-gradient-fraction <0..1> density-normal 专用梯度响应份额。0 表示纯几何膨胀方向，1 表示纯局域发射密度梯度方向；0<f<1 时，局域梯度方向必须至少保留 1-f 的全局外向投影，若热点导致梯度过度向内或横向则投影到这个外向 cone 的边界。它不引入手调各向异性系数，也不强行规定椭圆或三角形状；若在其他 sampler 下显式配置会报错。
  • --flow-trans-radius <covariance|density-percentile:p|density-level:fraction> density-normal 专用横向半径定义。covariance 使用协方差椭圆半径；density-percentile:p 用发射密度沿角向射线累计到分位数 p 的参考半径；density-level:fraction 用 fraction * emissionDensityScale 的密度等值线参考半径，要求 0 < fraction < 1；若在其他 sampler 下显式配置会报错。density-defined 的 R_density(phi) 定义局域密度几何形状，主流强半径会换算到 covariance/sigma-equivalent 尺度。
  • --flow-trans-radius-resolution <balanced|precise|fast> density-normal 专用的 density-defined 半径 profile 数值分辨率。只影响 density-percentile / density-level 的事件级边界表构建，不改变 R_density(phi)、xi_shell = r / R_density(phi) 或 q 的定义；density-defined 模式下 xi_shell 和 xi_flow 都可大于 1。balanced 为默认 240 x 256，precise 复现旧 360 x 512，fast 为 120 x 128 预扫选项。若在其他 sampler 下显式配置会报错；covariance 半径模式下该项无实际效果。

算法是怎么做的

1. 几何阶段

先对两个核分别做 Woods-Saxon 抽样，得到 3D 核子位置。然后把两个核沿 x 方向平移 ±b/2。接着只在横向平面上判断核子是否发生碰撞：如果两核子的横向距离满足

dx^2 + dy^2 <= sigmaNN / pi

就把它们标记为 participant。最后把两边所有 participant 合并成该事例的参与核子集合。

如果设置 initial-geometry = response-test-023，这一阶段不会抽 Woods-Saxon 核子，而是生成一个人工横向点云：

  • n=0：以原点为中心的圆形高斯背景，宽度 initial-geometry-r0
  • n=2：按 initial-geometry-r2x/r2y/phi2 生成的椭圆高斯分量
  • n=3：三个间隔 2*pi/3 的高斯 hotspot，参数为 initial-geometry-r3/sigma3/phi3

三类源点的默认分配模式是 initial-geometry-source-allocation = ratio-total，即按 1:A2:A3 在固定总源数内近似分配，生成后整体 recenter，避免人工引入一阶 dipole。模板参数 A2/A3 只控制点云混合权重，最终响应图应使用输出中实际计算得到的 eps2/eps3。可选 independent-pools 模式把 source-count 解释为圆形背景池 N0，再分别生成 N2=round(N0*A2) 和 N3=round(N0*A3) 的二阶/三阶池；该模式移除了固定总源数竞争，但总 Npart 与共同二阶/三阶矩分母仍会随 A2/A3 变化。

默认 initial-geometry-fluctuate = false 时，response-test-023 使用上述固定模板参数，适合作为 closure baseline。若设置 initial-geometry-fluctuate = true，生成器会在每个事件开始时从完整配置的 source-count/a2/a3/r2x/r2y/r3/sigma3 区间均匀抽样一组 event-local 模板参数；r0、phi2、phi3 第一版仍为固定单值。该随机化只允许和 response-test-023 联合使用，不改变 Glauber 生成逻辑。ROOT events.geo_a2/geo_a3/geo_r2x/geo_r2y/geo_r3/geo_sigma3 写入的是该事件实际使用的模板参数；物理/分析目标仍应看生成后的 events.eps2/eps3，不是配置范围本身。

注意：A2/A3 的随机数独立不等价于测得的 eps2/eps3 独立。ratio-total 在固定总源数下按 1:A2:A3 竞争分配源点；提高 A3 会改变圆形背景和椭圆分量的相对占比以及总二阶矩分母，因此单独随机化 A3 也可能带出 events.eps2 的反相关。independent-pools 只移除了固定总源数竞争，当前 5000-event 诊断仍未通过 |corr(eps2,eps3)| < 0.08 的解耦验收；若需要严格隔离二阶和三阶初态自由度，应继续引入后抽样分层匹配或目标 eccentricity 反解，而不是只切换源池模式。

2. eps2 和 psi2

先计算 participant 在横向平面的质心，再计算中心化后的二阶矩：

  • sigmaX2
  • sigmaY2
  • sigmaXY

然后构造二阶偏心率向量：

  • eccentricityX = sigmaY2 - sigmaX2
  • eccentricityY = 2 * sigmaXY

最终得到：

  • eps2 = sqrt(ex^2 + ey^2) / (sigmaX2 + sigmaY2)
  • psi2 = 0.5 * atan2(ey, ex)

物理含义是：

  • eps2 表示横向几何有多“扁”
  • psi2 表示这个椭圆主轴朝哪个方向

3. 事件介质和发射密度

代码先把 participant 点云转换成 EventMedium。默认 V1a 模式使用带公开默认值的 Glauber-density response：

  • initialDensity 由 participant Gaussian point-cloud density 给出
  • 在 participant plane 中对短轴方向乘默认 lambda_in = 1.20，对长轴方向乘默认 lambda_out = 1.05
  • Gaussian kernel 做同样的仿射变换后再与默认 sigma_evo = 0.5 fm 平滑卷积；这三个量都可通过 config/CLI 调整
  • emissionDensity 表示 freeze-out 发射密度 s_f
  • emissionGeometry 从 s_f 的二阶矩恢复，并写出 eps2_f、psi2_f 和 chi2 = eps2_f / eps2

如果设置 density-evolution = none，介质退回 identity：initialDensity == emissionDensity，participantGeometry == emissionGeometry。

如果设置 density-evolution = gradient-response，介质进入 V2 opt-in 路径：

  • s0 仍由 participant Gaussian point-cloud density 给出
  • s_em 的有效宽度是 sqrt(flowDensitySigma^2 + gradientSigmaEm^2)，用于抽发射 marker 初态 r0
  • s_dyn 的有效宽度是 sqrt(flowDensitySigma^2 + gradientSigmaDyn^2)，用于计算 -grad(s_dyn)/(s_dyn + floor)
  • gradientSigmaDyn 必须大于 gradientSigmaEm，保证动力学梯度密度比发射 marker 密度更平滑
  • V2 必须同时设置 flow-velocity-sampler = gradient-response

每个 participant 的发射粒子数仍用 Gamma-Poisson 混合模型抽样，也就是常说的 NBD-like 涨落模型。这样可以保留逐热点、逐事例的粒子数涨落。

4. 发射点坐标

横向发射点通过内部 EmissionSite 接口传递：

  • EmissionSite::initialPosition 是 V2 marker 初态坐标，写入 particles.x0/y0；非 V2 模式下设为 participant anchor
  • EmissionSite::position 是实际横向发射坐标，写入 particles.x/y
  • EmissionSite::sourceAnchor 是未 smear 的 participant 坐标，继续写入 particles.source_x/source_y
  • EmissionSite::betaTX/betaTY 是 V2 梯度响应生成的横向速度
  • EmissionSite::emissionWeight 是可选 Cooper-Frye 风格分析权重，写入 particles.emission_weight

默认 V1a 模式从 emissionDensity 抽样横向发射坐标；density-evolution = none 时保留原先 participant hotspot + smearSigma 横向展宽。纵向部分采用固定 tau0，再结合 eta_s 得到：

V2 模式中，每个候选粒子先从对应 participant 的 s_em Gaussian component 抽 r0，再在 r0 查询 s_dyn 梯度，生成 bounded 位移 dr 和横向速度 beta_T。末态横向发射点为 rf = r0 + dr。

  • t = tau0 * cosh(eta_s)
  • z = tau0 * sinh(eta_s)

5. 纵向分布

当前实现已经不是“纯高斯 eta_s”，而是：

  • 中心区是平顶核心
  • 两侧是高斯尾部

对应参数是：

  • etaPlateauHalfWidth
  • sigmaEta

这样做的原因是，纯高斯源在中快度会产生过尖的 eta 分布；平顶核心更接近当前项目想要的近中快度平台行为。

需要注意，这仍然只是一个现象学近似，不是完整的实验 dN/deta 描述。

6. 局域热动量抽样

代码先在流体局域静止系里抽样粒子的热动量。当前默认模式改为 Maxwell-Juttner (MJ) 查表反演：

  • 在 [0, mjPMax] 上建立均匀动量网格
  • 用 w(p) = p^2 exp(-sqrt(p^2 + m^2) / T) 计算权重并积分成归一化 CDF
  • 运行时对均匀随机数做 lower_bound + 线性插值反演，得到动量大小
  • 方向仍然各向同性
  • 能量仍由壳条件 E^2 = p^2 + m^2 给出

同时保留 gamma 兼容模式，继续使用原先的 Gamma 分布近似。若 temperature <= 0，则直接返回零三动量和 E = m，不建立 MJ 表。

7. 流场

当前横向流场由两个正交开关控制：

  • density-evolution 决定使用 identity 介质、V1a s_0 -> s_f 响应层，还是 opt-in V2 梯度响应层
  • flow-velocity-sampler 决定横向流速来源来自 covariance-ellipse、density-normal、affine-effective，还是 V2 gradient-response

在 density-evolution = none 下，covariance-ellipse 保留原先公式：

  • 用 participant 质心和二阶矩构造 sigmaX2、sigmaY2、sigmaXY
  • 解析对角化得到 lambdaMajor、lambdaMinor、radiusMajor、radiusMinor
  • 继续用历史定义写回 eps2 和 psi2

对每个发射点：

  • 先减质心并投影到主轴系，得到 x'、y'
  • 定义 rTilde = sqrt(x'^2 / radiusMajor^2 + y'^2 / radiusMinor^2)
  • 定义椭圆法向角 phiB = atan2(y' / radiusMinor^2, x' / radiusMajor^2)
  • 定义 rhoRaw = pow(rTilde, flowTransProfilePower) * (flowTransRho0 + kappa2 * eps2_initial * cos(2 * phiB))
  • 用 betaT = tanh(max(0, rhoRaw)) 得到横向流大小，再裁到 0.95

在默认 V1a 模式下：

  • 若 flow-velocity-sampler = covariance-ellipse，流场用 freeze-out 几何确定 rTilde 和法向方向，二阶响应振幅和平面来自初态 participant eccentricity vector：
    • rhoRaw = flowTransRho0 * pow(rTilde, flowTransProfilePower) * exp(2 * kappa2 * eps2_initial * cos(2 * (phiB - psi2_initial)))
  • 若 flow-velocity-sampler = density-normal，默认方向来自共享 emissionDensity 的解析 density gradient；在平坦或退化区域回退到 emissionGeometry 的协方差法向：
    • 默认 density-normal-kappa-compensation = false 且不显式配置新方向/半径键时，rhoRaw = flowTransRho0 * pow(rTilde, flowTransProfilePower)
    • 若显式开启 density-normal-kappa-compensation，则恢复同样的 V1a 指数补偿：
      • rhoRaw = flowTransRho0 * pow(rTilde, flowTransProfilePower) * exp(2 * kappa2 * eps2_initial * cos(2 * (phiB - psi2_initial)))
    • 若显式配置 flow-trans-direction-gradient-fraction 或 flow-trans-radius，方向仍由 density-gradient 给出自然形状，但中间值 0<f<1 会施加全局膨胀 cone：梯度方向相对几何径向的投影不得小于 1-f，否则保留梯度的切向符号并投影到 cone 边界。flow-trans-direction-gradient-fraction = 0 是纯几何膨胀方向，1 是纯局域梯度方向。半径按 covariance、density-percentile:<p> 或 density-level:<fraction> 定义。density-defined 半径只给出角向几何边界 R_density(phi)；主流强使用 xi_flow = q * xi_shell，其中 xi_shell = r / R_density(phi)，density-percentile:p 的 q = sqrt(-2*log(1-p))，density-level:fraction 的 q = sqrt(-2*log(fraction))。默认 radius-profile 流强保持 rhoRaw = flowTransRho0 * pow(xiFlow, flowTransProfilePower)；f 只约束方向，不削弱总 baseline，也不引入手调各向异性系数。若启用 shell-gradient-corrected，梯度修正写作 delta = f * flowTransGradientStrength * (tildeA - shellMean)。最终横向速度仍由 betaT = tanh(max(0, rhoRaw)) 并保留 0.95 上限保护。flow-trans-radius-resolution 只选择 density-defined 参考半径 profile 的数值网格（balanced = 240 x 256、precise = 360 x 512、fast = 120 x 128），不改变 R_density(phi) 或 sigma-equivalent 标定。
    • 若进一步选择 flow-trans-magnitude-mode = shell-gradient-corrected，则必须使用 density-percentile 或 density-level 半径。代码沿每条角向射线累计 \(\max(0,-\nabla s_f\cdot\hat e_\phi/\max(s_f,f_{\rm floor}s_{\rm scale}))\)，做事件级 RMS 归一化，再扣除相同 lookup 壳层的角向平均，得到 delta = flowTransDirectionGradientFraction * flowTransGradientStrength * (tildeA - shellMean)。delta 被截断到 [-flowTransGradientMaxFactorDelta,+flowTransGradientMaxFactorDelta]；基础 rhoRaw 使用 xi_flow，但 correction table 只在 0 <= xi_shell <= 1 上定义，所以 xi_shell > 1 时复用最外层表格值。最终 rhoRaw = flowTransRho0 * pow(xiFlow, flowTransProfilePower) * (1 + delta)；修正表退化时回退到 radius-profile。
  • 若 flow-velocity-sampler = affine-effective，流场不再复用旧的 kappa2 调制或 affine-kappa-aniso 分裂公式，而是把 affine-gaussian 的几何闭合转成局域有效膨胀率；默认 additive-rho 子模式会继续把 flow-trans-rho0 作为 baseline 平均流强：
    • affine-effective 只允许与 density-evolution = affine-gaussian 配对；闭合无效时直接返回零横向流，不回退其他 sampler。
    • 共享闭合量：
      • H_in_eff = lambdaIn / affineDeltaTauRef
      • H_out_eff = lambdaOut / affineDeltaTauRef
      • H_bar = 0.5 * (H_in_eff + H_out_eff)
      • R_bar = sqrt(sigmaInFinal * sigmaOutFinal)
    • 默认子模式 affine-effective-mode = additive-rho：
      • 方向固定复用 density-normal：优先密度梯度法向，平坦/退化区回退协方差法向。
      • u_geom = affineKappaFlow * pow(rTilde, flowTransProfilePower) * sqrt((H_in_eff * x')^2 + (H_out_eff * y')^2)
      • u_geom_iso = affineKappaFlow * pow(rTilde, flowTransProfilePower) * H_bar * rTilde * R_bar
      • 分别裁剪后取 rho_geom = atanh(u_geom)、rho_geom_iso = atanh(u_geom_iso)。
      • baseline 保留：rho_base = flowTransRho0 * pow(rTilde, flowTransProfilePower)。
      • 最终快度：rho_total = max(0, rho_base + rho_geom - rho_geom_iso)，再映射 betaT = min(affine-u-max, tanh(rho_total))。
    • 可选子模式 affine-effective-mode = full-tensor：
      • u_x' = affineKappaFlow * pow(rTilde, flowTransProfilePower) * H_in_eff * x'
      • u_y' = affineKappaFlow * pow(rTilde, flowTransProfilePower) * H_out_eff * y'
      • 旋回实验室系后若 betaT > affine-u-max 则按比例整体裁剪。
    • affine-kappa-aniso 在两个子模式中都不再进入公式（legacy/no-op），仅保留解析兼容与 finite 校验。

需要注意，affine-effective 只改横向流速闭合，不改发射权重语义。本轮没有调整任何 shell_weight 结构，也没有改写 EmissionSite::emissionWeight 的生产和使用方式。

横向方向沿所选 sampler 的法向，随后再与 eta_s 的 Bjorken-like 纵向流组合。

若选择 gradient-response，横向流速不再重新由椭圆或 density-normal 计算，而是直接使用 EmissionSite 中保存的 betaTX/betaTY。这条路径只允许和 density-evolution = gradient-response 同时开启，速度大小由 gradient-vmax * tanh(gradient-velocity-kappa * |grad ln s_dyn|) 给出。

7a. V2 半径与权重诊断

  • r2_0 和 r2_f 是 accepted 粒子 marker 初态点云与末态发射点云的中心化横向二阶矩。
  • r2_ratio = r2_f / r2_0，当 r2_0 <= 0 时写 0。
  • emission_weight 默认是 1；当 cooper-frye-weight = mt-cosh 时写入 m_T cosh(y - eta_s)。
  • events.v2 在存在非单位权重时使用加权 Q-vector；默认 cooper-frye-weight = none 时仍退化为原先的未加权定义。

8. 正确 boost 到实验室系

这是该项目和旧参考思路最重要的区别之一。当前实现不是简单地给 px、py 人工加一个偏移量，而是：

  1. 先在局域静止系抽样热动量
  2. 再按局域流速做完整 Lorentz boost

这比直接“加速度偏移”更符合 blast-wave 的物理过程。

几个容易误解的点

eps2 到底是什么

它是二阶 participant eccentricity，也就是初态参与核子横向分布的椭圆偏心率。它不是末态流系数 v2，而是一个“初态几何各向异性”的量。

这里的 v2 怎么定义

当前输出里的事件级 v2 采用历史参考宏一致的逐事件二阶 Q-vector 定义：

  • Q2x = sum_i cos(2 * phi_i)
  • Q2y = sum_i sin(2 * phi_i)
  • v2 = sqrt(Q2x^2 + Q2y^2) / Nch

其中 phi_i 使用写入 particles 树后的末态横向动量方位角，Nch 就是该事件写出的粒子数。若 cooper-frye-weight = mt-cosh，这里会使用粒子树里的 emission_weight 做加权 Q-vector；默认 none 时所有权重为 1。它表示“这个事件整体有多强的二阶方位各向异性”，不是 eps2，也不是相对 psi2 的 mean(cos(2 * (phi - psi2)))。

当前 ROOT schema 还写出 v2_lab_x = Q2x / sum_i w_i 和 v2_lab_y = Q2y / sum_i w_i。这两个量保留固定 lab 坐标下的符号：若长轴/短轴相对 x/y 翻转，v2_lab_x 的均值会比非负模长 v2 更直接暴露方向差异。相对初态二阶平面的投影满足 v2_wrt_psi2 = v2_lab_x*cos(2*psi2) + v2_lab_y*sin(2*psi2)。

v_n{2}(pT) 怎么定义

当前还额外支持可选差分二粒子 cumulant，n=2 由 v2pt-bins 开启，n=3 由 v3pt-bins 开启：

  • reference particles 是事件内全部粒子
  • POI 是一个给定 pT bin 内的全部粒子
  • 固定使用单位 track weight，不读取 particles.emission_weight
  • 不加 eta gap，也不做 subevent split
  • 每个 bin 的最终结果是
    • v_n{2}(bin) = d_n{2}(bin) / sqrt(c_n{2})
  • 统计误差用 leave-one-event-out jackknife 计算

这个 observable 故意和当前 events.v2/events.v3 分开：

  • events.v2/events.v3 仍是逐事件 Q-vector 模长
  • v2{2}(pT) / v3{2}(pT) 是跨事件累计后的二粒子 cumulant
  • cooper-frye-weight 只影响前者，不影响后者

为什么 px-py 不一定肉眼上就像横椭圆

因为原始 inclusive px-py 图会被大量低 pT 的近各向同性热粒子主导，所以即便存在正确方向的二阶各向异性，2D 密度图看起来也可能不明显。更敏感的量通常是：

  • phi 分布
  • v2
  • 对较高 pT 做切后再看动量各向异性

为什么 eta 不能直接和实验 dN_ch/deta 一一对应

因为当前模型只生成单一直接 pi+，没有：

  • 其他粒子种类
  • 共振衰变 feed-down
  • 探测器接受度和效率

所以这里的 eta 更适合做模型内部趋势检查，而不是直接拿来当实验全带电分布。

当前实现上的注意事项

  • ROOT 树里的浮点分支现在使用 Double_t，不是 Float_t 原因是之前 Float_t 在大 |eta_s| 尾部会让壳条件偏差变大，不利于 QA。
  • 生成器核心尽量保持 ROOT-free 如果后续要扩展算法，优先改 include/ 和 src/，不要把 ROOT 类型带进核心物理层。
  • 输出文件接口现在已经比较稳定 如果修改树名、分支名或直方图名，需要同步更新 QA 程序和所有下游分析。

如果后续继续扩展，优先建议做什么

按性价比排序，比较值得优先做的是：

  1. 增加更直接的各向异性观测量，例如 phi - psi2 和 v2(pT)
  2. 增加多粒子种类与共振衰变
  3. 把当前简化热抽样升级为更标准的源函数或 Cooper-Frye 采样
  4. 支持最小偏置 b 抽样和中心度切分

一句话总结

这个项目当前最适合被理解为：一个结构清晰、接口稳定、便于继续演化的 v1 blast-wave 生成器骨架，用来研究 Glauber 几何如何通过局域流场映射到末态单粒子相空间分布。
