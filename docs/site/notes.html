简介

这份手记只保留当前主链和最容易误解的语义。 详细运行说明、参数解释和完整 ROOT 输出列表以 docs/项目说明.md 为准。

当前主链

  1. 默认用 Monte Carlo Glauber 生成逐事例 participant 几何；若设置 initial-geometry = response-test-023，则改用人工 0+2+3 横向点云做响应测试。
  2. 按 density-evolution 构造事件介质：
    • none：participant 直接作为旧对照路径的发射锚点
    • affine-gaussian：默认路径，用 s0 -> sf 的仿射膨胀和平滑得到 freeze-out density
    • gradient-response：V2 opt-in 路径，用 s_em 抽 marker、用 s_dyn 给位移和横向速度
  3. 按当前介质抽横向发射位置，再配上 eta_s 和固定 tau0 得到时空坐标。
  4. 抽局域静止系热动量，再叠加流体元速度并 boost 到实验室系。
  5. 写出 events、participants、particles 和基础 QA 对象。
  6. 如果配置了 v2pt-bins 或 v3pt-bins，再写对应差分 v_n{2}(pT) 元数据，或额外生成完整分析载荷。

当前默认路径

  • 默认介质演化：density-evolution = affine-gaussian
  • 默认横向流方向：flow-velocity-sampler = covariance-ellipse
  • 默认热动量模式：thermal-sampler = maxwell-juttner
  • 默认 V1a 二阶响应旋钮：kappa2
  • affine-effective 是额外的 opt-in 闭合流场，不替换默认主线；默认子模式是 affine-effective-mode = additive-rho
  • V2 gradient-response 仍然是 opt-in，不替换默认主线
  • response-test-023 也是 opt-in，不替换默认 Glauber 初态

最容易混淆的点

  • eps2/psi2 是初态 participant geometry。
  • eps3/psi3 是 recentered 初态三阶几何；eps2/psi2 仍保留 covariance 语义。
  • initial-geometry-a2/a3 只是人工模板混合权重，不等同于 eps2/eps3。
  • eps2_f/psi2_f/chi2 是 freeze-out 几何诊断量，不替换初态语义。
  • source_x/source_y、x0/y0、x/y 分别是：
    • participant 锚点
    • marker 初态位置
    • 最终发射位置
  • events.v2 是事件级末态汇总量。
  • events.v3 是事件级末态三阶 Q-vector 模长；v2_wrt_psi2 和 v3_wrt_psi3 是相对初态几何平面的投影。
  • v2{2}(pT) / v3{2}(pT) 是单独的差分分析结果，不替换 events.v2/events.v3，而且当前固定使用单位 track weight。
  • smear 是旧 hotspot 发射路径的横向展宽；flow-density-sigma 是 density 沉积宽度，两者不是一回事。
  • 横向平均流强和半径 profile 的公开名现在是 flow-trans-rho0 / flow-trans-profile-power；旧 rho0 / flow-power 不再兼容。density-defined 半径下 flow-trans-rho0 是 covariance-equivalent xi_flow = 1 处的快度尺度，不是全局最大流强，也不是密度边界本身的流强。
  • flow-trans-direction-gradient-fraction、flow-trans-radius 和显式 flow-trans-radius-resolution 只属于 density-normal sampler；resolution 只改变 density-percentile/level 边界 profile 的数值网格。density-defined 的 R(phi) 只给角向 shell 几何，主流强使用 xi_flow = q * xi_shell，其中 xi_shell = r / R(phi)。
  • flow-trans-magnitude-mode = shell-gradient-corrected 是 density-normal + density-percentile/level 专用的 opt-in 流强修正；默认 radius-profile 使用 sigma-equivalent xi_flow 主径向项，显式 flow-trans-gradient-* 参数不能在默认模式下静默生效；xi_shell > 1 时 correction lookup 复用最外层表格值。
  • density-normal-kappa-compensation 默认关闭，只对 affine-gaussian + density-normal 有意义。
  • affine-effective 只允许配 affine-gaussian；默认 additive-rho 用密度梯度法向、保留 flow-trans-rho0 baseline，并叠加 affine 几何修正。
  • affine-effective-mode = full-tensor 是同一 sampler 下的 opt-in 主轴张量速度场；affine-kappa-aniso 现在只是 legacy/no-op 兼容解析参数。
  • affine-effective 输出会额外写首个有效事件的演化前/后 density map：affine_effective_density_initial_x-y 与 affine_effective_density_final_x-y，默认用 LEGO1 查看。
  • debug-initial-geometry 会写首个有效事件的 initial_geometry_density_x-y，主要用于查看 response-test-023 初态模板。
  • gradient-response 必须同时设置：
    • density-evolution = gradient-response
    • flow-velocity-sampler = gradient-response
  • flowpt-output-mode = separate-file 时，主结果文件只带启用 harmonic 的 *_2_pt_edges 也算合法，不代表输出缺失。
  • 当前 centrality 仍是固定 b 的简单映射，不应当作完备实验中心度模型来理解。

运行提醒

  • config/test_b8.cfg 是当前规范示例配置。
  • qa/ 目录里保留了多代历史 ROOT 文件，不应默认把它们当成当前 schema 样本。
  • 这台机器上的沙箱内 alienv ROOT smoke 结果不具权威性；若出现 PCM / module 噪声，应改用沙箱外命令复核。
