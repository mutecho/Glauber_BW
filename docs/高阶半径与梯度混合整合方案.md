# 高阶半径与梯度混合整合方案

## 方案定位

本文是在 `docs/高阶半径&梯度混合.md` 基础上的整理版方案。目标是升级横向流场的方向与流强定义，使三角扰动、多 hotspot 或更高阶初态结构能进入横向流，而不是继续让流强只依赖二阶 covariance 椭圆半径。

本文仍是实现前设计文档，不是当前代码行为说明。真正改代码前必须先完成配置键名整理，避免在已有 `flow-velocity-sampler`、`density-evolution`、`gradient-response`、`affine-effective` 等配置面上继续叠加含义混乱的新键。

---

## Part 0：配置键名整理与命名契约收敛

### 0.1 前置目标

新增横向流方向、归一化半径和梯度修正前，先整理当前所有配置键名。该步骤是功能实现的前置任务：

1. 先完成配置键名 audit；
2. 再给出新旧键名映射表；
3. 再更新示例配置和中文配置说明；
4. 最后才开始实现新的横向流方向、半径和梯度修正逻辑。

如果跳过此步骤，后续新增的 `flow-trans-*` 配置会和当前 `flow-velocity-sampler`、旧 `flow-power`、`flow-density-sigma`、`gradient-*` 等键混在一起，导致运行配置难以理解和维护。

2026-05-06 落地决策：旧公开名 `rho0` 与 `flow-power` 不再兼容，分别直接迁移为 `flow-trans-rho0` 与 `flow-trans-profile-power`；`kappa2` 保留为二阶响应系数；本轮新增的 `flow-trans-direction-gradient-fraction` 与 `flow-trans-radius` 只允许 `density-normal` sampler 使用。

### 0.2 命名原则

配置键名应满足：

1. 层级清楚；
2. 职责单一；
3. 简洁；
4. 不和已有键名产生含义重叠；
5. 不增加不必要的模式选择；
6. 不设计大量只在特定模式下静默无效的选项；
7. 如果某个参数只对某个算法有意义，应优先把它绑定到该算法选择值中，或者在 validation 中明确拒绝无效组合。

### 0.3 建议配置层级

建议将当前和新增配置按职责分层：

| 层级 | 键名范围 | 职责 |
|---|---|---|
| run control | `nevents`, `seed`, `output`, `progress` | 运行控制 |
| initial geometry | `initial-geometry-*` | 初态点云或 response-test 模板 |
| medium evolution | `density-evolution`, `affine-*`, `gradient-response-*` | 密度演化和介质构造 |
| transverse flow | `flow-trans-*` | 横向流方向、横向流强、横向流修正 |
| thermal sampling | `thermal-*`, `mj-*`, `temperature` | 局域热动量 |
| longitudinal / multiplicity | `tau0`, `sigma-eta`, `eta-plateau`, `nbd-*` | 纵向发射和 multiplicity |
| analysis output | `v2pt-*`, `v3pt-*`, `flowpt-*` | 差分 flow 后处理输出 |
| debug output | `debug-*` | 可选诊断载荷 |

### 0.4 `flow-velocity-sampler` 的边界

`flow-velocity-sampler` 当前是高层横向流 backend 选择，例如：

```text
flow-velocity-sampler = covariance-ellipse
flow-velocity-sampler = density-normal
flow-velocity-sampler = gradient-response
flow-velocity-sampler = affine-effective
```

新增配置不要再使用含糊的 `flow-*` 前缀表示横向流内部细节。新的方向、半径和梯度修正统一使用：

```text
flow-trans-*
```

其中 `trans` 表示 transverse flow，避免和：

- `flow-velocity-sampler`
- `flowpt-output-mode`
- `v2pt-bins`
- `gradient-response`

混淆。

实现前应对下列键做完整 audit，并给出最终迁移表：

| 当前键 | 初步处理方向 | 理由 |

### 0.6 新增横向流配置命名规则

新增 transverse-flow 内部配置统一使用 `flow-trans-*`。

建议第一轮候选配置面：

```text
flow-trans-direction-gradient-fraction = 1.0
flow-trans-radius = covariance
flow-trans-rho0 = 0.9
flow-trans-profile-power = 1.0
flow-trans-gradient-strength = 0.0
flow-trans-gradient-density-floor-fraction = 1.0e-4
flow-trans-gradient-max-delta-rho = 0.2
```

说明：

- `flow-trans-direction-gradient-fraction = 0.0` 表示纯几何径向方向；
- `flow-trans-direction-gradient-fraction = 1.0` 表示纯 density-gradient 外向方向；
- 中间值表示二者连续混合；
- `flow-trans-gradient-strength = 0.0` 表示关闭累计梯度修正；
- 不新增 `flow-trans-direction = mixed` 这类额外 enum，除非后续确实需要多个无法用连续参数表达的方向算法；
- 不新增 `flow-trans-gradient-correction = off|...`，除非后续确实实现了两个以上梯度修正算法。

### 0.7 避免无效参数的半径模式写法

为了避免：

```text
flow-trans-radius = covariance
flow-trans-radius-boundary = 0.95
```

这种 `radius-boundary` 在 `covariance` 下无意义的组合，推荐把带参数的半径模式写成单一值：

```text
flow-trans-radius = covariance
flow-trans-radius = density-percentile:0.95
flow-trans-radius = density-level:1.0e-3
```

这样：

- `covariance` 不需要额外参数；
- `density-percentile:0.95` 的参数就是累计密度百分位；
- `density-level:1.0e-3` 的参数就是密度等值面 cutoff fraction；
- validation 可以直接校验模式和值域；
- 不存在长期“设置了但无效”的 boundary 键。

若后续解析复杂度过高，可以退回两键方案：

```text
flow-trans-radius = density-percentile
flow-trans-radius-boundary = 0.95
```

但两键方案必须在 validation 中拒绝 `covariance + boundary` 这类无效组合，不能静默忽略。

## Part I：横向流方向升级

### 1.1 物理目标

横向流方向应同时支持整体几何径向膨胀和局域压力梯度膨胀。

几何径向方向：

\[
\hat{\mathbf e}_r
=
\frac{\mathbf r-\mathbf c}{|\mathbf r-\mathbf c|}
\]

其中 \(\mathbf c\) 是事件中心，优先使用发射阶段几何中心。

密度梯度外向方向：

\[
\hat{\mathbf n}_s
=
-\frac{\nabla s_f(\mathbf r)}
{|\nabla s_f(\mathbf r)|}
\]

该方向来自 smeared freeze-out density，可响应 hotspot、三角扰动和复杂局部结构。

### 1.2 混合方向定义

使用一个连续权重控制两类方向：

\[
w_g = \texttt{flow-trans-direction-gradient-fraction}
\]

\[
\hat{\mathbf n}_{\rm mix}
=
\frac{(1-w_g)\hat{\mathbf e}_r+w_g\hat{\mathbf n}_s}
{|(1-w_g)\hat{\mathbf e}_r+w_g\hat{\mathbf n}_s|}
\]

约定：

- \(w_g=0\)：纯几何径向方向；
- \(w_g=1\)：纯密度梯度方向；
- \(0<w_g<1\)：混合方向。

### 1.3 数值保护

方向计算必须处理：

1. 中心附近 \(|\mathbf r-\mathbf c|\approx 0\)；
2. 局域密度梯度过小；
3. 低密度尾部；
4. 两个方向近似反向导致混合向量过小；
5. 非有限值。

建议 fallback 顺序：

1. 若混合方向有效，使用混合方向；
2. 否则使用有效的密度梯度方向；
3. 否则使用有效的几何径向方向；
4. 否则使用当前 covariance normal fallback；
5. 最后返回零横向流。

## Part II：横向流强方案

### 2.1 总体结构

横向流强拆成两个层次：

\[
\rho_t
=
\rho_{\rm base}
+
\Delta\rho_{\rm grad}
\]

其中：

- \(\rho_{\rm base}\)：由归一化外向坐标决定的主径向增长；
- \(\Delta\rho_{\rm grad}\)：可选累计梯度修正，只负责同壳层方向差异。

第一阶段只实现 \(\rho_{\rm base}\)。第二阶段再加入 \(\Delta\rho_{\rm grad}\)。

### 2.2 归一化外向坐标

对发射点 \(\mathbf r\)，定义中心到该点的射线方向：

\[
\hat{\mathbf e}_\phi
=
\frac{\mathbf r-\mathbf c}{|\mathbf r-\mathbf c|}
\]

在同一方向上定义有效边界半径 \(R_{\rm eff}(\phi)\)，然后：

\[
\xi(\mathbf r)
=
\frac{|\mathbf r-\mathbf c|}
{R_{\rm eff}(\phi)}
\]

默认用于 flow profile 的值为：

\[
\xi_{\rm used}=\min(\xi,1)
\]

默认先截断到 1，避免极低密度尾部或边界外点产生异常 boost。是否允许 \(\xi>1\) 应作为后续系统误差扫描，而不是第一轮公开默认行为。

### 2.3 并行边界方案 A：累计密度百分位

配置候选：

```text
flow-trans-radius = density-percentile:0.95
```

对每个方位 \(\phi\)，沿中心射线积分二维极坐标密度权重：

\[
I(R,\phi)
=
\int_0^R
t\,s_f(\mathbf c+t\hat{\mathbf e}_\phi)\,dt
\]

定义总权重：

\[
I_{\rm total}(\phi)
=
\int_0^{R_{\max}}
t\,s_f(\mathbf c+t\hat{\mathbf e}_\phi)\,dt
\]

百分位边界满足：

\[
\frac{I(R_p,\phi)}
{I_{\rm total}(\phi)}
=p
\]

其中 \(p=0.95\) 是初始建议值。

优点：

- 抗低密度尾巴；
- 对不同事件的整体 density scale 不敏感；
- 更适合作为稳健默认高阶半径方案。

风险：

- \(p\) 过高会被尾部影响；
- \(p\) 过低会截断真实外层结构；
- 需要定义 \(R_{\max}\) 和积分网格。

### 2.4 并行边界方案 B：密度等值面

配置候选：

```text
flow-trans-radius = density-level:1.0e-3
```

先定义事件级 density scale：

\[
s_{\rm scale}
=
\max_{\rm support}
s_f(\mathbf r_i)
\]

实现上应在 `EventMedium` 中显式保存 `emissionDensityScale`，而不是复用 `markerDensityScale` 或 `dynamicsDensityScale` 的隐含等价关系。

等值面边界满足：

\[
s_f(\mathbf c+R_{\rm level}(\phi)\hat{\mathbf e}_\phi)
=
f_{\rm cut}\,s_{\rm scale}
\]

其中初始建议：

\[
f_{\rm cut}=10^{-3}
\]

若一条射线上有多个 crossing，取最外层 crossing：

\[
R_{\rm level}(\phi)=
\max R
\quad\text{such that}\quad
s_f(\mathbf c+R\hat{\mathbf e}_\phi)
\ge
f_{\rm cut}s_{\rm scale}
\]

优点：

- 物理图像直接；
- debug map 上容易理解；
- 对三角 hotspot 的等值面形状响应明确。

风险：

- 对 smoothing scale 和 cutoff fraction 敏感；
- 多 hotspot 下可能出现多个分离区；
- 低 cutoff 会把 Gaussian 尾部拉得过远。

### 2.5 旧半径方案：covariance

配置候选：

```text
flow-trans-radius = covariance
```

该模式保持当前 covariance-based generalized radius：

\[
\tilde r^2
=
(\mathbf r-\mathbf c)^T
\Sigma_f^{-1}
(\mathbf r-\mathbf c)
\]

它用于：

- 旧结果对照；
- 回归测试；
- 圆形或椭圆 baseline；
- 验证新增高阶半径不会改变旧配置行为。

### 2.6 主径向流强

主径向流强使用：

\[
\rho_{\rm base}
=
\rho_0\,\xi_{\rm used}^{\alpha}
\]

对应候选配置：

```text
flow-trans-rho0 = 0.9
flow-trans-profile-power = 1.0
```

其中：

- \(\rho_0\)：横向流快度 baseline；
- \(\alpha\)：径向 profile 幂指数；
- `covariance` 半径模式下 \(\xi_{\rm used}\) 对应旧 `rTilde`；
- `density-percentile` 或 `density-level` 下 \(\xi_{\rm used}\) 是新的归一化外向坐标。

### 2.7 累计梯度修正

累计梯度修正不在第一阶段实现。它是第二阶段增强项。

目标：

在相似 \(\xi\) 的壳层上，不同方向可能由于压力梯度历史不同而有不同流强。

推荐代理：

\[
G(\mathbf r)
=
\int_0^{r}
\max\left(
0,
-
\frac{\nabla s_f(\mathbf c+t\hat{\mathbf e}_\phi)
\cdot
\hat{\mathbf e}_\phi}
{\max(s_f(\mathbf c+t\hat{\mathbf e}_\phi),s_{\rm floor})}
\right)dt
\]

为了避免重复表达“越靠外流越强”，使用同壳层平均扣除：

\[
\Delta\rho_{\rm grad}
=
\kappa_{\rm grad}
\left[
G(\mathbf r)
-
\langle G\rangle_{\xi}
\right]
\]

候选配置：

```text
flow-trans-gradient-strength = 0.0
flow-trans-gradient-density-floor-fraction = 1.0e-4
flow-trans-gradient-max-delta-rho = 0.2
```

约定：

- `flow-trans-gradient-strength = 0.0` 表示关闭；
- 第一轮不新增 correction-mode enum；
- 如果未来实现多种修正算法，再考虑引入模式选择。

### 2.8 最终限制

最终横向流快度：

\[
\rho_t
=
\max(0,\rho_{\rm base}+\Delta\rho_{\rm grad})
\]

最终横向速度：

\[
\beta_T=\tanh\rho_t
\]

必须施加：

1. 非负保护；
2. 有限值保护；
3. 最大 \(\beta_T\) 限制；
4. 最大 \(\Delta\rho_{\rm grad}\) 限制；
5. 异常边界 fallback。

## Part III：方向与流强解耦

方向和流强不得写死为单一模式。

推荐计算链：

```text
emission density
  -> transverse direction
  -> transverse radius coordinate
  -> transverse rho_t
  -> beta_T * direction
  -> Bjorken longitudinal composition
  -> Lorentz boost
```

应能独立测试：

| 方向 | 半径/流强 | 目的 |
|---|---|---|
| 当前 density-gradient | 当前 covariance 半径 | 旧行为对照 |
| 几何径向 | 当前 covariance 半径 | 检查方向替换影响 |
| 混合方向 | 当前 covariance 半径 | 检查方向平滑性 |
| 当前 density-gradient | `density-percentile` 半径 | 检查高阶边界半径 |
| 当前 density-gradient | `density-level` 半径 | 检查等值面半径 |
| 混合方向 | 高阶半径 | 检查完整方向/半径组合 |
| 混合方向 | 高阶半径 + 弱梯度修正 | 检查第二阶段增强 |

### Step 1：建立横向方向混合

```text

```

并验证：

- `0.0` 为纯几何径向；
- `1.0` 为当前 density-gradient 方向；

- 旧配置结果不变。

### Step 2：建立 transverse radius profile cache

为每个事件预计算角向边界：

```cpp
struct FlowTransRadiusProfile {
  bool valid;
  double centerX;
  double centerY;
  std::vector<double> boundaryRadii;
};
```

实现时优先内部固定合理默认：

- angle bins：180 或 360；
- radial bins：128 或 256；
- 二分 refine：可选；
- public 配置暂不暴露 bin 数，除非验证显示确实需要。

### Step 3：实现 `density-percentile` 半径

实现：

```text
flow-trans-radius = density-percentile:0.95
```

验证圆形、椭圆、三角、多 hotspot profile。

### Step 4：实现 `density-level` 半径

实现：

```text
flow-trans-radius = density-level:1.0e-3
```

并显式加入 `emissionDensityScale`，避免复用其他 density scale 的隐式假设。

### Step 5：接入主径向流强

使用：

\[
\rho_{\rm base}
=
\rho_0\xi^\alpha
\]

先不启用累计梯度修正。

### Step 6：第二阶段加入累计梯度修正

只有在高阶半径第一阶段验证稳定后，再加入：

```text
flow-trans-gradient-strength
flow-trans-gradient-density-floor-fraction
flow-trans-gradient-max-delta-rho
```

从弱修正开始扫描。

## Part V：验证要求

### 5.1 ROOT-free 单元测试

至少覆盖：

1. 圆形 Gaussian profile：`density-percentile` 和 `density-level` 边界近似与角度无关；
2. 椭圆 Gaussian profile：边界随主轴方向变化；
3. 三角 hotspot profile：边界有三阶角向结构；
4. 多 hotspot profile：等值面边界取最外层 crossing；
5. 中心点：\(\xi=0\)，方向 fallback 稳定；
6. 边界外点：默认 \(\xi_{\rm used}=1\)；
7. 无效 density 或无效 geometry：回退旧稳定路径或零流。

### 5.2 配置解析和 validation 测试

必须覆盖：

1. `flow-trans-radius = covariance` 不需要额外参数；
2. `flow-trans-radius = density-percentile:0.95` 解析成功；
3. `flow-trans-radius = density-level:1.0e-3` 解析成功；
4. 百分位不在 `(0,1)` 时拒绝；
5. 等值面 cutoff 非正时拒绝；
6. `flow-trans-direction-gradient-fraction` 不在 `[0,1]` 时拒绝；
7. 旧键迁移策略按文档执行；
8. 无效组合不静默忽略。

### 5.3 物理输出验证

至少比较：

1. 横向方向场；
2. 横向半径场；
3. 横向流强场；
4. \(p_T\) 谱；
5. \(\phi\) 分布；
6. \(v_2\)；
7. \(v_3\)；
8. \(v_{3|\Psi_3}\) 对 \(\epsilon_3\) 的 response；
9. 如已有 femto 分析，则比较 HBT 或 source observables。

重点检查：

- 新半径是否比 covariance 半径更保留三阶边界结构；
- 等值面方案是否对 cutoff 过敏；
- 百分位方案是否更稳健；
- 方向混合是否降低纯梯度方向噪声；
- 梯度修正是否导致 \(p_T\) 谱过硬；
- 旧配置结果是否不变。
