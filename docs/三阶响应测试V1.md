# 0+2+3 初态几何模板方案：用于 Glauber–BlastWave 三角响应测试

## 目标

在现有 Glauber–BlastWave 流程中加入一套人为可控的初态几何模板，用于检查模型链路对二阶、三阶几何信息的末态保留程度。

本方案采用：

\[
n=0 + n=2 + n=3
\]

即：

- \(n=0\)：圆形背景，提供平均系统大小与总发射源；
- \(n=2\)：椭圆几何扰动，作为二阶响应校准；
- \(n=3\)：三角 hotspot 几何扰动，作为三角流响应测试核心。

暂不加入 \(n=1\) dipole geometry，避免和整体质心平移、recenter、有限粒子数涨落混淆。

---

## 设计原则

该方案不是为了精确复现实验初态，而是用于 response / closure test：

1. 初态几何应当可控；
2. 每组事件应能计算实际的 \(\varepsilon_2,\varepsilon_3\)；
3. 末态应能检查 \(v_2,v_3\) 是否随初态几何变化；
4. 尽量减少对现有 Glauber–BlastWave 主流程的侵入；
5. 模板参数只用于生成几何，最终分析以实际计算得到的 \(\varepsilon_n\) 为准。

---

## 总体流程

```text
生成基础圆形背景
        ↓
叠加二阶椭圆结构
        ↓
叠加三角 hotspot 结构
        ↓
归一化 / 抽样 emission points
        ↓
进入现有 BlastWave / emission / boost 流程
        ↓
输出末态粒子
        ↓
计算初态 ε2, ε3 与末态 v2, v3
        ↓
检查 εn → vn 的保留关系
~~~

------

## **1. 初态密度构造**

构造二维横向密度：

# **[** **\rho(x,y)**

\rho_0(x,y)
 +
 A_2 \rho_2(x,y)
 +
 A_3 \rho_3(x,y)
 ]

其中：

- (\rho_0(x,y))：圆形背景；
- (\rho_2(x,y))：二阶椭圆扰动；
- (\rho_3(x,y))：三角 hotspot 扰动；
- (A_2)：二阶结构权重；
- (A_3)：三阶结构权重。

这里的 (A_2,A_3) 只是模板强度，不应直接等同于 (\varepsilon_2,\varepsilon_3)。

------

## **2. 圆形背景 (n=0)**

圆形背景用于提供平均系统尺度和基本发射源。

可以采用高斯背景或 Woods–Saxon-like 背景。若现有代码已经有 Glauber-smeared density，也可以直接将其作为背景；若需要纯 closure test，则建议先用解析圆形背景。

示例形式：

# **[** **\rho_0(x,y)**

C_0
 \exp\left[
 -\frac{x^2+y^2}{2R_0^2}
 \right]
 ]

其中：

- (R_0)：圆形背景横向尺度；
- (C_0)：归一化常数或相对权重。

物理含义：

- 代表没有角向各向异性的平均 fireball；
- 对应 (n=0)，不是一阶几何；
- 用于建立 baseline。

------

## **3. 二阶椭圆结构 (n=2)**

二阶结构用于检查已有流程是否能保留椭圆几何信息。

可以采用两种实现之一。

### **方案 2A：椭圆高斯背景**

构造一个椭圆密度：

# **[** **\rho_2(x,y)**

\exp\left[
 -\frac{x_{\Phi_2}^2}{2R_{2,x}^2}
 -\frac{y_{\Phi_2}^2}{2R_{2,y}^2}
 \right]
 ]

其中：

[
 x_{\Phi_2}=x\cos\Phi_2+y\sin\Phi_2
 ]

[
 y_{\Phi_2}=-x\sin\Phi_2+y\cos\Phi_2
 ]

- (\Phi_2)：二阶几何方向；
- (R_{2,x},R_{2,y})：椭圆主轴尺度。

该方法更平滑，适合作为稳定 baseline。

### **方案 2B：双 hotspot 结构**

也可以用两个相对的高斯热点：

# **[** **\rho_2(x,y)**

G(x,y;R_2\cos\Phi_2,R_2\sin\Phi_2,\sigma_2)
 +
 G(x,y;-R_2\cos\Phi_2,-R_2\sin\Phi_2,\sigma_2)
 ]

其中：

# **[** **G(x,y;x_c,y_c,\sigma)**

\exp\left[
 -\frac{(x-x_c)^2+(y-y_c)^2}{2\sigma^2}
 \right]
 ]

- (R_2)：双热点距离中心的半径；
- (\sigma_2)：热点宽度；
- (\Phi_2)：双热点方向。

该方法和三角 hotspot 的形式更统一，但比椭圆高斯更容易引入高阶成分。

建议优先实现一种即可，不需要同时实现两个版本。

------

## **4. 三角 hotspot 结构 (n=3)**

三角结构是本方案核心。使用三个等间隔高斯热点：

# **[** **\rho_3(x,y)**

\sum_{k=0}^{2}
 G
 \left(
 x,y;
 R_3\cos\left(\Phi_3+\frac{2\pi k}{3}\right),
 R_3\sin\left(\Phi_3+\frac{2\pi k}{3}\right),
 \sigma_3
 \right)
 ]

其中：

- (R_3)：三角 hotspot 距离中心的半径；
- (\sigma_3)：每个热点的宽度；
- (\Phi_3)：三角结构方向；
- (k=0,1,2)：三个 hotspot。

物理含义：

- 这是一个结构性三角初态；
- 主要贡献 (\varepsilon_3)；
- 同时可能引入 (n=6,9,\ldots) 等高阶成分；
- 若和非圆背景叠加，也可能产生少量阶间串扰。

因此最终必须重新计算实际的 (\varepsilon_3)，而不是直接使用 (A_3) 作为三角性大小。

------

## **5. 密度归一化与抽样**

构造总密度后：

# **[** **\rho(x,y)**

\rho_0(x,y)
 +
 A_2 \rho_2(x,y)
 +
 A_3 \rho_3(x,y)
 ]

需要进行归一化，使其可以作为发射点抽样权重。

可选做法：

### **做法 A：固定总 multiplicity**

将 (\rho(x,y)) 归一化为概率密度：

# **[** **P(x,y)**

\frac{\rho(x,y)}
 {\int dxdy \rho(x,y)}
 ]

然后按固定 multiplicity 抽样 emission points。

### **做法 B：固定总权重，保留 NBD / Poisson multiplicity**

若现有代码中 multiplicity 已经由 NBD / Poisson 控制，则只需要将 (\rho(x,y)) 作为空间抽样分布，不直接改变总 multiplicity。

建议优先使用现有 multiplicity 机制，避免几何测试和 multiplicity 测试混在一起。

------

## **6. 事件级几何量计算**

每个事件生成后，应在进入末态发射前计算初态几何量。

先计算密度加权中心：

# **[** **\bar{x}**

\frac{\sum_i w_i x_i}{\sum_i w_i}
 ]

# **[** **\bar{y}**

\frac{\sum_i w_i y_i}{\sum_i w_i}
 ]

然后使用 recentered 坐标：

[
 x_i’ = x_i-\bar{x}
 ]

[
 y_i’ = y_i-\bar{y}
 ]

再计算：

[
 r_i=\sqrt{x_i’^2+y_i’^2}
 ]

[
 \phi_i=\operatorname{atan2}(y_i’,x_i’)
 ]

对 (n=2,3)，定义：

# **[** **\mathcal{E}_n**

# **\varepsilon_n e^{in\Psi_n}**

- ​	![Attachment.tiff](../../../../../Attachment.tiff)	

\frac{
 \sum_i w_i r_i^n e^{in\phi_i}
 }{
 \sum_i w_i r_i^n
 }
 ]

其中：

- (w_i)：第 (i) 个 density cell / marker / emission source 的权重；
- (\varepsilon_n = |\mathcal{E}_n|)；
- (\Psi_n = \arg(\mathcal{E}_n)/n)。

需要保存：

```text
epsilon2_init
psi2_init
epsilon3_init
psi3_init
x_center
y_center
R_rms
A2
A3
R3
sigma3
```

如果代码中没有显式 density grid，而是直接有 emission points，则可将每个点权重设为 1 或使用已有 emission weight。

------

## **7. 进入现有 BlastWave 流程**

生成 emission points 后，继续使用现有流程：

```text
emission point (x, y, z, t)
        ↓
确定局域流速 / 横向 boost
        ↓
LRF 中热抽样动量
        ↓
boost 回实验室系
        ↓
输出末态粒子
```

本方案不强制修改流场模型。

重点是：同一套流场和发射规则下，比较不同初态几何模板对末态 (v_2,v_3) 的影响。

如果当前阶段想避免“流场定义过复杂”带来的混淆，可以先固定流场模型，只扫描初态几何参数。

------

## **8. 末态响应量计算**

对每个事件末态粒子计算复流矢量：

# **[** **V_n**

\frac{1}{N}
 \sum_{j=1}^{N}
 e^{in\varphi_j}
 ]

其中：

- (N)：用于分析的末态粒子数；
- (\varphi_j)：第 (j) 个粒子的方位角；
- (n=2,3)。

保存：

[
 v_n^{\rm abs}=|V_n|
 ]

以及相对于初态几何平面的投影：

# **[** **v_n{\Psi_n^{\rm init}}**

\operatorname{Re}
 \left[
 V_n e^{-in\Psi_n^{\rm init}}
 \right]
 ]

推荐优先使用：

```text
v2_wrt_init_plane
v3_wrt_init_plane
abs_V2
abs_V3
```

用于 closure test。

原因是初态几何平面已知，用它做投影可以直接检查几何响应，不需要先引入 event-plane reconstruction 的额外不确定性。

------

## **9. 推荐测试矩阵**

### **Stage 0：圆形 baseline**

```text
A2 = 0
A3 = 0
```

目标：

- 检查无注入几何时的 residual (v_2,v_3)；
- 估计有限 multiplicity 和流程本身带来的背景偏差。

预期：

```text
epsilon2_init ≈ 0
epsilon3_init ≈ 0
v2 ≈ 0
v3 ≈ 0
```

实际结果不会严格为零，但应接近统计涨落水平。

------

### **Stage 1：只开二阶结构**

```text
A2 > 0
A3 = 0
```

目标：

- 验证 (\varepsilon_2 \rightarrow v_2) 链路；
- 检查二阶结构是否错误地产生显著 (v_3)。

输出检查：

```text
v2_wrt_init_plane vs epsilon2_init
v3_wrt_init_plane vs epsilon3_init
```

------

### **Stage 2：只开三角结构**

```text
A2 = 0
A3 > 0
```

目标：

- 验证 (\varepsilon_3 \rightarrow v_3) 链路；
- 这是本方案核心。

建议扫描：

```text
A3 = 0.02, 0.05, 0.10, 0.15
```

或等价的一组相对权重。

输出检查：

```text
v3_wrt_init_plane vs epsilon3_init
abs_V3 vs epsilon3_init
```

预期：

- (v_3) 应随 (\varepsilon_3) 单调增强；
- 若没有增强，说明流程中三角几何没有有效传递到末态；
- 若 (v_2) 同时显著增强，需要检查阶间串扰或流场定义。

------

### **Stage 3：二阶 + 三阶同时开启**

```text
A2 > 0
A3 > 0
```

目标：

- 检查在椭圆背景存在时三角响应是否仍然可见；
- 检查 (n=2) 与 (n=3) 是否发生明显串扰。

输出检查：

```text
v2_wrt_init_plane vs epsilon2_init
v3_wrt_init_plane vs epsilon3_init
correlation(epsilon2_init, v3)
correlation(epsilon3_init, v2)
```

如果三阶响应只在无二阶背景时存在，而加入二阶后消失，需要检查流场定义或 emission 权重是否过度椭圆化。

------

每轮测试建议至少生成以下图：

### **初态诊断图**

epsilon2 distribution
epsilon3 distribution
psi2 distribution
psi3 distribution
epsilon3 vs A3

```

### **末态响应图**

```

### **串扰检查图**

```text

```

## **11. 成功判据**

该方案的最小成功标准：

1. 圆形 baseline 中 (v_2,v_3) 接近统计涨落水平；
2. 只开二阶结构时，(v_2) 对 (\varepsilon_2) 有单调响应；
3. 只开三角结构时，(v_3) 对 (\varepsilon_3) 有单调响应；
4. 二阶 + 三阶同时开启时，(v_3) 仍能随 (\varepsilon_3) 增强；
5. 阶间串扰存在但不应压倒主响应。

如果满足这些条件，可以认为当前 Glauber–BlastWave 流程至少在 toy-level 上保留了初态三角几何信息。

------

## **12. 注意事项**

### **1. 不要把模板参数直接当成偏心率**

例如：

```text
A3 != epsilon3
```

必须在生成 density 后重新计算实际：

```text
epsilon3_init
psi3_init
```

后续所有响应图都应以实际 (\varepsilon_3) 为横轴。

------

### **2. 三角 hotspot 会引入高阶结构**

三角 hotspot 不只产生 (n=3)，也可能产生 (n=6,9,\ldots)。

这不是错误，但需要理解为：

```text
这是结构性三角模板，不是纯 Fourier n=3 mode。
```

若需要更干净的线性响应测试，后续可以再引入平滑的 Fourier density modulation。

------

### **3. 优先固定流场，先不要同时调太多机制**

初始测试阶段建议固定：

```text
freeze-out time
temperature
radial flow strength
multiplicity model
longitudinal profile
```

只扫描 (A_2,A_3) 或三角 hotspot 几何参数。

否则无法判断末态变化来自初态几何，还是来自其他模型参数。

------

### **4. 初态几何平面优先于末态 event plane**

closure test 阶段优先使用：

```text
v3_wrt_init_plane
```

不要一开始就只看：

```text
v3{2}
```

因为 (v_3{2}) 混入有限 multiplicity、自相关、非流和 event-by-event fluctuation，不如已知初态平面直接。

------

## **结论**

本方案将初态构造为：

[
 \boxed{
 \rho(x,y)=\rho_0(x,y)+A_2\rho_2(x,y)+A_3\rho_3(x,y)
 }
 ]

它对应：

[
 \boxed{
 n=0+n=2+n=3
 }
 ]

而不是：

[
 n=1+n=2+n=3
 ]

该方案适合作为 Glauber–BlastWave 三角响应测试的第一套可控几何模板。重点不在于模板本身是否完全真实，而在于能否建立清晰的检查链路：

[
 \boxed{
 \varepsilon_3^{\rm init}
 \rightarrow
 v_3^{\rm final}
 }
 ]

如果这条链路在纯三角 hotspot 测试中无法成立，则需要优先检查流场定义、emission 权重、boost 方向和末态分析方法，而不是直接进入更复杂的真实 Glauber fluctuation 测试。
