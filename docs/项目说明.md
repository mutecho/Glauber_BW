# Blast-Wave 项目说明

## 这是什么

这是一个用 C++17 和 ROOT 实现的小型 Blast-Wave 事例生成器。它的目标不是做完整重离子末态模拟，而是在一个足够小、足够清晰、可继续扩展的代码骨架里，完成以下事情：

- 用 Monte Carlo Glauber 产生逐事例的 participant 几何涨落
- 在 participant hotspot 上做简化的 blast-wave 发射
- 通过 ROOT 输出事件树、粒子树和基础 QA 直方图
- 提供一个独立 QA 程序检查输出是否合法、接口是否稳定

当前第一版只生成单一直接 `pi+`，适合做几何和流场方向的快速验证，不适合直接拿去和实验末态全带电粒子分布逐项对比。

## 当前能力边界

已经支持：

- 固定冲量参数 `b` 的事件生成
- 由 `b` 线性近似得到的事件级 `centrality`
- Pb-Pb 风格 Woods-Saxon 核几何
- participant eccentricity `eps2` 和 participant plane `psi2`
- 基于末态粒子二阶 Q-vector 的事件级 `v2`
- 局域热动量抽样
- 局域流场 + 正确 Lorentz boost
- ROOT 输出与独立 QA
- CMake 构建

还没有做：

- 最小偏置 `b` 抽样
- 共振衰变
- 多粒子种类
- hadronic rescattering
- 完整 Cooper-Frye freeze-out 超曲面采样

  - `progress`

  - `flow-power`
  - `debug-flow-ellipse`
  - `sigma-eta`

  - `nbd-mu`

- 配置文件里的相对 `output` 路径会相对配置文件所在目录解析

最小示例：

```text
# config/test_b8.cfg
nevents = 5000
b = 8
temperature = 0.2

# mj-pmax = 8.0
# mj-grid-points = 4096

```

以下参数仍然可以继续直接通过 CLI 传入。

基础参数：

- `--nevents <int>`
  生成多少个事件。
- `--b <fm>`
  冲量参数，单位 `fm`。
- `--temperature <GeV>`
  局域静止系热动量抽样的温度尺度。
- `--thermal-sampler <maxwell-juttner|gamma>`
  热动量大小抽样模式；默认 `maxwell-juttner`，`gamma` 保留为兼容模式。
- `--mj-pmax <GeV>`
  Maxwell-Juttner 查表的最大动量截断。
- `--mj-grid-points <int>`
  Maxwell-Juttner 查表网格点数。
- `--tau0 <fm/c>`
  固定冻结 proper time。
- `--smear <fm>`
  participant hotspot 的横向高斯展宽。
- `--sigma-nn <fm^2>`
  核子-核子非弹截面参数，默认 `7.0 fm^2`。
- `--seed <uint64>`
  随机数种子。
- `--output <path>`
  输出 ROOT 文件路径。
- `--progress`
  强制显示进度条。
- `--no-progress`
  强制关闭进度条。

调参与 QA 相关参数：

- `--rho0 <value>`
  椭圆法向流场的各向同性快度基线。
- `--rho2 <value>`
  与事件 `eps2` 相乘的二阶各向异性快度振幅。
- `--flow-power <value>`
  归一化椭圆半径 `rTilde` 的幂指数。
- `--debug-flow-ellipse`
  额外写出 `flow_ellipse_debug` 树和 `flow_ellipse_participant_norm_x-y` 汇总直方图。
- `--no-debug-flow-ellipse`
  显式关闭上述 debug 输出。
- `--sigma-eta <value>`
  纵向 `eta_s` 高斯尾宽度。
- `--eta-plateau <value>`
  纵向 `eta_s` 平顶核心半宽。
- `--nbd-mu <value>`
  每个 hotspot 的平均发射强度。
- `--nbd-k <value>`
  NBD 形状参数，控制涨落强弱。

旧参数 `--vmax`、`--kappa2`、`--r-ref` 已废弃并会直接报错：

- `vmax -> rho0 = atanh(vmax)`
- `kappa2 -> 重新调参的 rho2`
- `r-ref -> 已由事件椭圆半轴内生吸收`

如果配置文件没有设置 `progress`，并且 CLI 也没有传 `--progress` 或
`--no-progress`，程序只会在 `stderr` 连接到交互终端时自动显示进度条。

QA 程序 `qa_blastwave_output` 的参数较少：

- `--input <file.root>`
- `--output <qa.root>`
- `--expect-nevents <int>`

若开启 `debug-flow-ellipse`，输出文件还会额外包含：

- `flow_ellipse_debug`
  逐事件树，记录质心、协方差、本征值、半轴、主轴、`eps2`、`psi2` 和 `valid`
- `flow_ellipse_participant_norm_x-y`
  所有 `valid` 事件的 participant 在主轴系并按半轴归一化后的汇总 `TH2`

QA 程序会始终要求默认主契约对象存在；对上述 debug 对象采用“存在则验证，不存在则忽略”的策略。

## 算法是怎么做的

### 1. 几何阶段

先对两个核分别做 Woods-Saxon 抽样，得到 3D 核子位置。然后把两个核沿 `x` 方向平移 `±b/2`。接着只在横向平面上判断核子是否发生碰撞：如果两核子的横向距离满足

`dx^2 + dy^2 <= sigmaNN / pi`

就把它们标记为 participant。最后把两边所有 participant 合并成该事例的参与核子集合。

### 2. `eps2` 和 `psi2`

先计算 participant 在横向平面的质心，再计算中心化后的二阶矩：

- `sigmaX2`
- `sigmaY2`
- `sigmaXY`

然后构造二阶偏心率向量：

- `eccentricityX = sigmaY2 - sigmaX2`
- `eccentricityY = 2 * sigmaXY`

最终得到：

- `eps2 = sqrt(ex^2 + ey^2) / (sigmaX2 + sigmaY2)`
- `psi2 = 0.5 * atan2(ey, ex)`

物理含义是：

- `eps2` 表示横向几何有多“扁”
- `psi2` 表示这个椭圆主轴朝哪个方向

### 3. 每个 participant 发多少粒子

每个 participant 被视为一个发射 hotspot。其发射粒子数不是固定值，而是用一个 Gamma-Poisson 混合模型来抽样，也就是常说的 NBD-like 涨落模型。这样可以保留逐热点、逐事例的粒子数涨落。

### 4. 发射点坐标

每个粒子并不从 participant 的精确位置发出，而是先对横向位置做一个宽度为 `smearSigma` 的高斯展宽。纵向部分采用固定 `tau0`，再结合 `eta_s` 得到：

- `t = tau0 * cosh(eta_s)`
- `z = tau0 * sinh(eta_s)`

### 5. 纵向分布

当前实现已经不是“纯高斯 `eta_s`”，而是：

- 中心区是平顶核心
- 两侧是高斯尾部

对应参数是：

- `etaPlateauHalfWidth`
- `sigmaEta`

这样做的原因是，纯高斯源在中快度会产生过尖的 `eta` 分布；平顶核心更接近当前项目想要的近中快度平台行为。

需要注意，这仍然只是一个现象学近似，不是完整的实验 `dN/deta` 描述。

### 6. 局域热动量抽样

代码先在流体局域静止系里抽样粒子的热动量。当前默认模式改为 Maxwell-Juttner (MJ) 查表反演：

- 在 `[0, mjPMax]` 上建立均匀动量网格
- 用 `w(p) = p^2 exp(-sqrt(p^2 + m^2) / T)` 计算权重并积分成归一化 CDF
- 运行时对均匀随机数做 `lower_bound` + 线性插值反演，得到动量大小
- 方向仍然各向同性
- 能量仍由壳条件 `E^2 = p^2 + m^2` 给出

同时保留 `gamma` 兼容模式，继续使用原先的 Gamma 分布近似。若 `temperature <= 0`，则直接返回零三动量和 `E = m`，不建立 MJ 表。

### 7. 流场

当前默认横向流场不再使用“实验室原点径向流 + 标量调制”，而是先从 participant 协方差恢复事件椭圆：

- 用 participant 质心和二阶矩构造 `sigmaX2`、`sigmaY2`、`sigmaXY`
- 解析对角化得到 `lambdaMajor`、`lambdaMinor`、`radiusMajor`、`radiusMinor`
- 继续用历史定义写回 `eps2` 和 `psi2`

对每个发射点：

- 先减质心并投影到主轴系，得到 `x'`、`y'`
- 定义 `rTilde = sqrt(x'^2 / radiusMajor^2 + y'^2 / radiusMinor^2)`
- 定义椭圆法向角 `phiB = atan2(y' / radiusMinor^2, x' / radiusMajor^2)`
- 定义 `rhoRaw = pow(rTilde, flowPower) * (rho0 + rho2 * eps2 * cos(2 * phiB))`
- 用 `betaT = tanh(max(0, rhoRaw))` 得到横向流大小，再裁到 `0.95`

横向方向沿椭圆法向，随后再与 `eta_s` 的 Bjorken-like 纵向流组合。

### 8. 正确 boost 到实验室系

这是该项目和旧参考思路最重要的区别之一。当前实现不是简单地给 `px`、`py` 人工加一个偏移量，而是：

1. 先在局域静止系抽样热动量
2. 再按局域流速做完整 Lorentz boost

这比直接“加速度偏移”更符合 blast-wave 的物理过程。

## QA 程序会检查什么

`qa_blastwave_output` 会独立打开 ROOT 文件并检查：

- `events`、`participants` 和 `particles` 三棵树是否存在
- 必需直方图是否存在
- 事件号是否连续
- `Npart` 是否等于该事件在 `participants` 树中的实际条目数
- `Nch` 是否等于该事件在粒子树中的实际条目数
- `events.v2` 是否等于由该事件全部粒子反算出的二阶 Q-vector 模长除以 `Nch`
- `centrality` 是否保持在 `0-100` 且与当前 `b` 的线性映射一致
- 是否出现 `NaN` 或 `Inf`
- 所有粒子是否满足壳条件，误差不超过 `1e-4 GeV^2`
- `v2` 直方图条目数和均值是否与 `events.v2` 一致
- 若 `flow_ellipse_debug` 存在：
  - 条目数是否等于 `events`
  - `lambda_major >= lambda_minor >= 0`
  - `R_major^2`、`R_minor^2` 是否分别等于本征值
  - 主轴和短轴是否归一且正交
  - `valid = 1` 时半轴是否严格为正
- 若 `flow_ellipse_participant_norm_x-y` 存在：
  - 是否确实是 `TH2`
  - 是否至少接收到所有 `valid` 事件的 participant 填充

同时它会把重新读出的 QA 直方图写到新文件中，例如：

- `qa_x-y`
- `qa_px-py`
- `qa_psi2`
- `qa_v2`
- `qa_pT`
- `qa_eta`
- `qa_phi`

## 几个容易误解的点

### `eps2` 到底是什么

它是二阶 participant eccentricity，也就是初态参与核子横向分布的椭圆偏心率。它不是末态流系数 `v2`，而是一个“初态几何各向异性”的量。

### 这里的 `v2` 怎么定义

当前输出里的事件级 `v2` 采用历史参考宏一致的逐事件二阶 Q-vector 定义：

- `Q2x = sum_i cos(2 * phi_i)`
- `Q2y = sum_i sin(2 * phi_i)`
- `v2 = sqrt(Q2x^2 + Q2y^2) / Nch`

其中 `phi_i` 使用写入 `particles` 树后的末态横向动量方位角，`Nch` 就是该事件写出的粒子数。它表示“这个事件整体有多强的二阶方位各向异性”，不是 `eps2`，也不是相对 `psi2` 的 `mean(cos(2 * (phi - psi2)))`。

### 为什么 `px-py` 不一定肉眼上就像横椭圆

因为原始 inclusive `px-py` 图会被大量低 `pT` 的近各向同性热粒子主导，所以即便存在正确方向的二阶各向异性，2D 密度图看起来也可能不明显。更敏感的量通常是：

- `phi` 分布
- `v2`
- 对较高 `pT` 做切后再看动量各向异性

### 为什么 `eta` 不能直接和实验 `dN_ch/deta` 一一对应

因为当前模型只生成单一直接 `pi+`，没有：

- 其他粒子种类
- 共振衰变 feed-down
- 探测器接受度和效率

所以这里的 `eta` 更适合做模型内部趋势检查，而不是直接拿来当实验全带电分布。

## 当前实现上的注意事项

- ROOT 树里的浮点分支现在使用 `Double_t`，不是 `Float_t`
  原因是之前 `Float_t` 在大 `|eta_s|` 尾部会让壳条件偏差变大，不利于 QA。
- 生成器核心尽量保持 ROOT-free
  如果后续要扩展算法，优先改 `include/` 和 `src/`，不要把 ROOT 类型带进核心物理层。
- 输出文件接口现在已经比较稳定
  如果修改树名、分支名或直方图名，需要同步更新 QA 程序和所有下游分析。

## 如果后续继续扩展，优先建议做什么

按性价比排序，比较值得优先做的是：

1. 增加更直接的各向异性观测量，例如 `phi - psi2` 和 `v2(pT)`
2. 增加多粒子种类与共振衰变
3. 把当前简化热抽样升级为更标准的源函数或 Cooper-Frye 采样
4. 支持最小偏置 `b` 抽样和中心度切分

## 一句话总结

这个项目当前最适合被理解为：一个结构清晰、接口稳定、便于继续演化的 v1 blast-wave 生成器骨架，用来研究 Glauber 几何如何通过局域流场映射到末态单粒子相空间分布。
