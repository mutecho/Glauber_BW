# Blast-Wave 项目说明

## 这是什么

这是一个用 C++17 和 ROOT 实现的小型 Blast-Wave 事例生成器。它的目标不是做完整重离子末态模拟，而是在一个足够小、足够清晰、可继续扩展的代码骨架里，完成以下事情：

- 用 Monte Carlo Glauber 产生逐事例的 participant 几何涨落
- 在 participant hotspot 上做简化的 blast-wave 发射
- 通过 ROOT 输出事件树、粒子树和基础 QA 直方图
- 提供一个独立 QA 程序检查输出是否合法、接口是否稳定

当前第一版只生成单一直接 `pi+`，适合做几何和流场方向的快速验证，不适合直接拿去和实验末态全带电粒子分布逐项对比。

## 当前能力边界

已经支持：

- 固定冲量参数 `b` 的事件生成
- 由 `b` 线性近似得到的事件级 `centrality`
- Pb-Pb 风格 Woods-Saxon 核几何
- participant eccentricity `eps2` 和 participant plane `psi2`
- 基于末态粒子二阶 Q-vector 的事件级 `v2`
- 局域热动量抽样
- 局域流场 + 正确 Lorentz boost
- opt-in V2 梯度响应：用 participant density 梯度同时生成横向位移和横向流速
- ROOT 输出与独立 QA
- CMake 构建

还没有做：

- 最小偏置 `b` 抽样
- 共振衰变
- 多粒子种类
- hadronic rescattering
- 完整 Cooper-Frye freeze-out 超曲面采样

## 目录结构

- `include/blastwave/BlastWaveGenerator.h`
  对外数据结构和生成器类声明。
- `src/BlastWaveGenerator.cpp`
  单事件生成编排。
- `include/blastwave/DensityFieldModel.h`
  Gaussian point-cloud density 与解析 gradient 接口。
- `include/blastwave/EventMedium.h`
  事件级介质状态，区分 participant 几何、初始 density、V2 marker/dynamics density、发射阶段 density 和发射阶段几何。
- `include/blastwave/EmissionSampler.h`
  横向发射点抽样接口，支持 participant hotspot、density-field 和 V2 gradient-response backend。
- `include/blastwave/FlowFieldModel.h`
  基于事件介质的流体元速度抽样接口。
- `apps/generate_blastwave_events.cpp`
  命令行生成程序，负责写 ROOT 文件。
- `apps/qa_blastwave_output.cpp`
  独立 QA 程序，负责读取 ROOT 文件并做一致性检查。
- `CMakeLists.txt`
  主构建文件。
- `CMakePresets.json`
  默认 CMake preset，同时开启 `compile_commands.json` 输出。
- `config/`
  受版本控制的示例配置文件。
- `scripts/`
  本地运行辅助脚本，与静态配置分开存放。
- `reference/legacy-root-macros/`
  历史参考宏，只用于理解物理意图，不应直接复制代码。
- `qa/`
  本地测试输出样本。
- `res/`
  正式运行结果。
- `docs/`
  设计、handoff 和说明文档。
- `project-state/`
  仓库内工程协作台账。

- `--mj-pmax <GeV>`

- `--mj-grid-points <int>`
  Maxwell-Juttner 查表网格点数。

  固定冻结 proper time。
- `--smear <fm>`
  participant hotspot 的横向高斯展宽。
- `--sigma-nn <fm^2>`
  核子-核子非弹截面参数，默认 `7.0 fm^2`。
- `--seed <uint64>`

- `--output <path>`
  输出 ROOT 文件路径。
- `--progress`
  强制显示进度条。
- `--no-progress`
  强制关闭进度条。

调参与 QA 相关参数：

- `--rho0 <value>`
  椭圆法向流场的各向同性快度基线。
- `--kappa2 <value>`
  二阶流响应系数；事件级二阶快度调制振幅为 `a2 = kappa2 * eps2_initial`。对于 `affine-gaussian + density-normal`，只有在开启 `density-normal-kappa-compensation` 时才参与显式补偿。
- `--flow-power <value>`
  归一化椭圆半径 `rTilde` 的幂指数。
- `--flow-velocity-sampler <covariance-ellipse|density-normal|gradient-response>`
  选择横向流速来源：协方差椭圆法向、发射密度梯度法向或 V2 梯度响应保存的 site velocity。
- `--density-evolution <affine-gaussian|none|gradient-response>`
  选择介质演化层。默认 `affine-gaussian` 启用 V1a 响应；`none` 保留原先 identity 介质和 participant-hotspot 发射作为对照；`gradient-response` 是 opt-in V2 模式，必须与 `flow-velocity-sampler = gradient-response` 成对使用。
- `--flow-density-sigma <fm>`
  初态 Gaussian point-cloud density 的沉积宽度；V1a 中也作为 `s_0` 的 `sigma_dep`。
- `--affine-lambda-in <value>`
  `affine-gaussian` 模式下 participant plane 短轴方向的默认膨胀系数；默认 `1.20`。
- `--affine-lambda-out <value>`
  `affine-gaussian` 模式下 participant plane 长轴方向的默认膨胀系数；默认 `1.05`。
- `--affine-sigma-evo <fm>`
  `affine-gaussian` 模式下额外 Gaussian evolution smoothing 的宽度；默认 `0.5 fm`，允许设为 `0` 关闭这层平滑。
- `--density-normal-kappa-compensation`
  仅在 `density-evolution = affine-gaussian` 且 `flow-velocity-sampler = density-normal` 时有效；开启后恢复 `kappa2` 的显式 V1a 指数补偿。
- `--no-density-normal-kappa-compensation`
  显式关闭上述补偿；这是默认行为。
- `--debug-flow-ellipse`
  额外写出 `flow_ellipse_debug` 树和 `flow_ellipse_participant_norm_x-y` 汇总直方图。
- `--no-debug-flow-ellipse`
  显式关闭上述 debug 输出。
- `--gradient-sigma-em <fm>`
  V2 发射 marker 密度 `s_em` 的额外平滑宽度。
- `--gradient-sigma-dyn <fm>`
  V2 动力学梯度密度 `s_dyn` 的额外平滑宽度，必须大于 `gradient-sigma-em`。
- `--gradient-density-floor-fraction <value>`
  计算 `-grad(s_dyn)/(s_dyn + floor)` 时使用的事件级 floor 比例。
- `--gradient-density-cutoff-fraction <value>`
  接受 V2 `r0` marker 样本时使用的 `s_em` cutoff 比例。
- `--gradient-displacement-max <fm>`
  V2 确定位移的最大长度。
- `--gradient-displacement-kappa <fm>`
  V2 位移响应长度尺度，使 `kappa * |grad ln s_dyn|` 无量纲。
- `--gradient-diffusion-sigma <fm>`
  V2 位移的可选二维 Gaussian diffusion 宽度。
- `--gradient-vmax <value>`
  V2 横向速度上限，要求 `0 <= gradient-vmax < 1`。
- `--gradient-velocity-kappa <fm>`
  V2 横向速度响应长度尺度。
- `--cooper-frye-weight <none|mt-cosh>`
  可选分析权重，写入粒子树并用于事件级 `v2` 的 Q-vector；默认 `none`。
- `--debug-gradient-response`
  写出首个有 participant 和粒子的 V2 density debug TH2。
- `--no-debug-gradient-response`
  显式关闭 V2 density debug 输出。
- `--sigma-eta <value>`
  纵向 `eta_s` 高斯尾宽度。
- `--eta-plateau <value>`
  纵向 `eta_s` 平顶核心半宽。
- `--nbd-mu <value>`
  每个 hotspot 的平均发射强度。
- `--nbd-k <value>`
  NBD 形状参数，控制涨落强弱。

旧参数 `--vmax`、`--rho2`、`--r-ref` 已废弃并会直接报错：

- `vmax -> rho0 = atanh(vmax)`
- `rho2 -> kappa2`
- `r-ref -> 已由事件椭圆半轴内生吸收`

如果配置文件没有设置 `progress`，并且 CLI 也没有传 `--progress` 或
`--no-progress`，程序只会在 `stderr` 连接到交互终端时自动显示进度条。

QA 程序 `qa_blastwave_output` 的参数较少：

- `--input <file.root>`
- `--output <qa.root>`
- `--expect-nevents <int>`

- `density_normal_event_density_x-y`

## 算法是怎么做的

### 1. 几何阶段

先对两个核分别做 Woods-Saxon 抽样，得到 3D 核子位置。然后把两个核沿 `x` 方向平移 `±b/2`。接着只在横向平面上判断核子是否发生碰撞：如果两核子的横向距离满足

`dx^2 + dy^2 <= sigmaNN / pi`

就把它们标记为 participant。最后把两边所有 participant 合并成该事例的参与核子集合。

### 2. `eps2` 和 `psi2`

先计算 participant 在横向平面的质心，再计算中心化后的二阶矩：

- `sigmaX2`
- `sigmaY2`
- `sigmaXY`

然后构造二阶偏心率向量：

- `eccentricityX = sigmaY2 - sigmaX2`
- `eccentricityY = 2 * sigmaXY`

最终得到：

- `eps2 = sqrt(ex^2 + ey^2) / (sigmaX2 + sigmaY2)`
- `psi2 = 0.5 * atan2(ey, ex)`

物理含义是：

- `eps2` 表示横向几何有多“扁”
- `psi2` 表示这个椭圆主轴朝哪个方向

### 3. 事件介质和发射密度

代码先把 participant 点云转换成 `EventMedium`。默认 V1a 模式使用带公开默认值的 Glauber-density response：

- `initialDensity` 由 participant Gaussian point-cloud density 给出
- 在 participant plane 中对短轴方向乘默认 `lambda_in = 1.20`，对长轴方向乘默认 `lambda_out = 1.05`
- Gaussian kernel 做同样的仿射变换后再与默认 `sigma_evo = 0.5 fm` 平滑卷积；这三个量都可通过 config/CLI 调整
- `emissionDensity` 表示 freeze-out 发射密度 `s_f`
- `emissionGeometry` 从 `s_f` 的二阶矩恢复，并写出 `eps2_f`、`psi2_f` 和 `chi2 = eps2_f / eps2`

如果设置 `density-evolution = none`，介质退回 identity：`initialDensity == emissionDensity`，`participantGeometry == emissionGeometry`。

如果设置 `density-evolution = gradient-response`，介质进入 V2 opt-in 路径：

- `s0` 仍由 participant Gaussian point-cloud density 给出
- `s_em` 的有效宽度是 `sqrt(flowDensitySigma^2 + gradientSigmaEm^2)`，用于抽发射 marker 初态 `r0`
- `s_dyn` 的有效宽度是 `sqrt(flowDensitySigma^2 + gradientSigmaDyn^2)`，用于计算 `-grad(s_dyn)/(s_dyn + floor)`
- `gradientSigmaDyn` 必须大于 `gradientSigmaEm`，保证动力学梯度密度比发射 marker 密度更平滑
- V2 必须同时设置 `flow-velocity-sampler = gradient-response`

每个 participant 的发射粒子数仍用 Gamma-Poisson 混合模型抽样，也就是常说的 NBD-like 涨落模型。这样可以保留逐热点、逐事例的粒子数涨落。

### 4. 发射点坐标

横向发射点通过内部 `EmissionSite` 接口传递：

- `EmissionSite::initialPosition` 是 V2 marker 初态坐标，写入 `particles.x0/y0`；非 V2 模式下设为 participant anchor
- `EmissionSite::position` 是实际横向发射坐标，写入 `particles.x/y`
- `EmissionSite::sourceAnchor` 是未 smear 的 participant 坐标，继续写入 `particles.source_x/source_y`
- `EmissionSite::betaTX/betaTY` 是 V2 梯度响应生成的横向速度
- `EmissionSite::emissionWeight` 是可选 Cooper-Frye 风格分析权重，写入 `particles.emission_weight`

默认 V1a 模式从 `emissionDensity` 抽样横向发射坐标；`density-evolution = none` 时保留原先 participant hotspot + `smearSigma` 横向展宽。纵向部分采用固定 `tau0`，再结合 `eta_s` 得到：

V2 模式中，每个候选粒子先从对应 participant 的 `s_em` Gaussian component 抽 `r0`，再在 `r0` 查询 `s_dyn` 梯度，生成 bounded 位移 `dr` 和横向速度 `beta_T`。末态横向发射点为 `rf = r0 + dr`。

- `t = tau0 * cosh(eta_s)`
- `z = tau0 * sinh(eta_s)`

### 5. 纵向分布

当前实现已经不是“纯高斯 `eta_s`”，而是：

- 中心区是平顶核心
- 两侧是高斯尾部

对应参数是：

- `etaPlateauHalfWidth`
- `sigmaEta`

这样做的原因是，纯高斯源在中快度会产生过尖的 `eta` 分布；平顶核心更接近当前项目想要的近中快度平台行为。

需要注意，这仍然只是一个现象学近似，不是完整的实验 `dN/deta` 描述。

### 6. 局域热动量抽样

代码先在流体局域静止系里抽样粒子的热动量。当前默认模式改为 Maxwell-Juttner (MJ) 查表反演：

- 在 `[0, mjPMax]` 上建立均匀动量网格
- 用 `w(p) = p^2 exp(-sqrt(p^2 + m^2) / T)` 计算权重并积分成归一化 CDF
- 运行时对均匀随机数做 `lower_bound` + 线性插值反演，得到动量大小
- 方向仍然各向同性
- 能量仍由壳条件 `E^2 = p^2 + m^2` 给出

同时保留 `gamma` 兼容模式，继续使用原先的 Gamma 分布近似。若 `temperature <= 0`，则直接返回零三动量和 `E = m`，不建立 MJ 表。

### 7. 流场

当前横向流场由两个正交开关控制：

- `density-evolution` 决定使用 identity 介质、V1a `s_0 -> s_f` 响应层，还是 opt-in V2 梯度响应层
- `flow-velocity-sampler` 决定横向流速来源来自 `covariance-ellipse`、`density-normal`，还是 V2 `gradient-response`

在 `density-evolution = none` 下，`covariance-ellipse` 保留原先公式：

- 用 participant 质心和二阶矩构造 `sigmaX2`、`sigmaY2`、`sigmaXY`
- 解析对角化得到 `lambdaMajor`、`lambdaMinor`、`radiusMajor`、`radiusMinor`
- 继续用历史定义写回 `eps2` 和 `psi2`

对每个发射点：

- 先减质心并投影到主轴系，得到 `x'`、`y'`
- 定义 `rTilde = sqrt(x'^2 / radiusMajor^2 + y'^2 / radiusMinor^2)`
- 定义椭圆法向角 `phiB = atan2(y' / radiusMinor^2, x' / radiusMajor^2)`
- 定义 `rhoRaw = pow(rTilde, flowPower) * (rho0 + kappa2 * eps2_initial * cos(2 * phiB))`
- 用 `betaT = tanh(max(0, rhoRaw))` 得到横向流大小，再裁到 `0.95`

在默认 V1a 模式下：

- 若 `flow-velocity-sampler = covariance-ellipse`，流场用 freeze-out 几何确定 `rTilde` 和法向方向，二阶响应振幅和平面来自初态 participant eccentricity vector：
  - `rhoRaw = rho0 * pow(rTilde, flowPower) * exp(2 * kappa2 * eps2_initial * cos(2 * (phiB - psi2_initial)))`
- 若 `flow-velocity-sampler = density-normal`，方向来自共享 `emissionDensity` 的解析 density gradient；在平坦或退化区域回退到 `emissionGeometry` 的协方差法向：
  - 默认 `density-normal-kappa-compensation = false` 时，`rhoRaw = rho0 * pow(rTilde, flowPower)`
  - 若显式开启 `density-normal-kappa-compensation`，则恢复同样的 V1a 指数补偿：
    - `rhoRaw = rho0 * pow(rTilde, flowPower) * exp(2 * kappa2 * eps2_initial * cos(2 * (phiB - psi2_initial)))`

横向方向沿所选 sampler 的法向，随后再与 `eta_s` 的 Bjorken-like 纵向流组合。

若选择 `gradient-response`，横向流速不再重新由椭圆或 density-normal 计算，而是直接使用 `EmissionSite` 中保存的 `betaTX/betaTY`。这条路径只允许和 `density-evolution = gradient-response` 同时开启，速度大小由 `gradient-vmax * tanh(gradient-velocity-kappa * |grad ln s_dyn|)` 给出。

### 7a. V2 半径与权重诊断

- `r2_0` 和 `r2_f` 是 accepted 粒子 marker 初态点云与末态发射点云的中心化横向二阶矩。
- `r2_ratio = r2_f / r2_0`，当 `r2_0 <= 0` 时写 `0`。
- `emission_weight` 默认是 `1`；当 `cooper-frye-weight = mt-cosh` 时写入 `m_T cosh(y - eta_s)`。
- `events.v2` 在存在非单位权重时使用加权 Q-vector；默认 `cooper-frye-weight = none` 时仍退化为原先的未加权定义。

### 8. 正确 boost 到实验室系

这是该项目和旧参考思路最重要的区别之一。当前实现不是简单地给 `px`、`py` 人工加一个偏移量，而是：

1. 先在局域静止系抽样热动量
2. 再按局域流速做完整 Lorentz boost

这比直接“加速度偏移”更符合 blast-wave 的物理过程。

## QA 程序会检查什么

`qa_blastwave_output` 会独立打开 ROOT 文件并检查：

- `events`、`participants` 和 `particles` 三棵树是否存在
- 必需直方图是否存在
- 事件号是否连续
- `Npart` 是否等于该事件在 `participants` 树中的实际条目数
- `Nch` 是否等于该事件在粒子树中的实际条目数
- `events.v2` 是否等于由该事件全部粒子反算出的二阶 Q-vector 模长除以 `Nch`
- `events.eps2_f`、`events.psi2_f`、`events.chi2` 是否有限且 `chi2` 与 `eps2_f / eps2` 一致
- `events.r2_0`、`events.r2_f`、`events.r2_ratio` 是否等于粒子树中 `x0/y0` 和 `x/y` 反算出的中心化二阶矩
- `particles.emission_weight` 是否有限且非负
- `centrality` 是否保持在 `0-100` 且与当前 `b` 的线性映射一致
- 是否出现 `NaN` 或 `Inf`
- 所有粒子是否满足壳条件，误差不超过 `1e-4 GeV^2`
- `v2` 直方图条目数和均值是否与 `events.v2` 一致
- 若 `flow_ellipse_debug` 存在：
  - 条目数是否等于 `events`
  - `lambda_major >= lambda_minor >= 0`
  - `R_major^2`、`R_minor^2` 是否分别等于本征值
  - 主轴和短轴是否归一且正交
  - `valid = 1` 时半轴是否严格为正
- 若 `flow_ellipse_participant_norm_x-y` 存在：
  - 是否确实是 `TH2`
  - 是否至少接收到所有 `valid` 事件的 participant 填充
- 若 `density_normal_event_density_x-y` 存在：
  - 是否确实是 `TH2`
  - bin content 是否有限且非负
  - 默认 draw option 是否为 `LEGO` / `SURF` 这类三维样式
  - 是否至少有一个正 density bin
- 若任一 V2 gradient debug histogram 存在：
  - `gradient_s0_x-y`、`gradient_s_em_x-y`、`gradient_s_dyn_x-y`、`gradient_s_f_x-y` 是否全部存在
  - bin content 是否有限且非负
  - 每个 histogram 是否至少有一个正 bin

同时它会把重新读出的 QA 直方图写到新文件中，例如：

- `qa_x-y`
- `qa_px-py`
- `qa_psi2`
- `qa_v2`
- `qa_pT`
- `qa_eta`
- `qa_phi`

## 几个容易误解的点

### `eps2` 到底是什么

它是二阶 participant eccentricity，也就是初态参与核子横向分布的椭圆偏心率。它不是末态流系数 `v2`，而是一个“初态几何各向异性”的量。

### 这里的 `v2` 怎么定义

当前输出里的事件级 `v2` 采用历史参考宏一致的逐事件二阶 Q-vector 定义：

- `Q2x = sum_i cos(2 * phi_i)`
- `Q2y = sum_i sin(2 * phi_i)`
- `v2 = sqrt(Q2x^2 + Q2y^2) / Nch`

其中 `phi_i` 使用写入 `particles` 树后的末态横向动量方位角，`Nch` 就是该事件写出的粒子数。若 `cooper-frye-weight = mt-cosh`，这里会使用粒子树里的 `emission_weight` 做加权 Q-vector；默认 `none` 时所有权重为 `1`。它表示“这个事件整体有多强的二阶方位各向异性”，不是 `eps2`，也不是相对 `psi2` 的 `mean(cos(2 * (phi - psi2)))`。

### 为什么 `px-py` 不一定肉眼上就像横椭圆

因为原始 inclusive `px-py` 图会被大量低 `pT` 的近各向同性热粒子主导，所以即便存在正确方向的二阶各向异性，2D 密度图看起来也可能不明显。更敏感的量通常是：

- `phi` 分布
- `v2`
- 对较高 `pT` 做切后再看动量各向异性

### 为什么 `eta` 不能直接和实验 `dN_ch/deta` 一一对应

因为当前模型只生成单一直接 `pi+`，没有：

- 其他粒子种类
- 共振衰变 feed-down
- 探测器接受度和效率

所以这里的 `eta` 更适合做模型内部趋势检查，而不是直接拿来当实验全带电分布。

## 当前实现上的注意事项

- ROOT 树里的浮点分支现在使用 `Double_t`，不是 `Float_t`
  原因是之前 `Float_t` 在大 `|eta_s|` 尾部会让壳条件偏差变大，不利于 QA。
- 生成器核心尽量保持 ROOT-free
  如果后续要扩展算法，优先改 `include/` 和 `src/`，不要把 ROOT 类型带进核心物理层。
- 输出文件接口现在已经比较稳定
  如果修改树名、分支名或直方图名，需要同步更新 QA 程序和所有下游分析。

## 如果后续继续扩展，优先建议做什么

按性价比排序，比较值得优先做的是：

1. 增加更直接的各向异性观测量，例如 `phi - psi2` 和 `v2(pT)`
2. 增加多粒子种类与共振衰变
3. 把当前简化热抽样升级为更标准的源函数或 Cooper-Frye 采样
4. 支持最小偏置 `b` 抽样和中心度切分

## 一句话总结

这个项目当前最适合被理解为：一个结构清晰、接口稳定、便于继续演化的 v1 blast-wave 生成器骨架，用来研究 Glauber 几何如何通过局域流场映射到末态单粒子相空间分布。
