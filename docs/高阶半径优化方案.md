# `flow-trans-radius` density-defined 半径计算优化落地计划

## Summary
本轮只实现“方案 1+2”：降低 `density-percentile` / `density-level` 事件级 boundary profile 的数值开销，并新增 resolution preset。物理定义不变：仍然用 `R(phi)` 定义 `xi = r / R(phi)`，再计算 `rhoRaw = flowTransRho0 * pow(xi, flowTransProfilePower)`；`covariance` 半径模式不受影响。

当前瓶颈在 `src/FlowFieldModel.cpp`：旧实现固定 `360` 个角向点、每条射线 `512` 个径向点，每个径向点调用会遍历全部 support points 的 `evaluateDensityField()`。新默认 `balanced = 240 x 256`，把 boundary profile 构建的 density query 数从 `184320/event` 降到 `61440/event`，约 3 倍减少；旧数值保留为 `precise = 360 x 512`。

## Public Contract
新增公开配置/CLI 项：

```text
flow-trans-radius-resolution = balanced | precise | fast
--flow-trans-radius-resolution <balanced|precise|fast>
```

默认值和含义：

```text
balanced: angular=240, radial=256   # 新默认，推荐正式运行
precise:  angular=360, radial=512   # 旧数值行为，用于精度基线/回退
fast:     angular=120, radial=128   # 大统计预扫
```

契约细节：

- 新配置只影响 `flow-trans-radius = density-percentile:<p>` 和 `density-level:<fraction>` 的 profile 构建。
- `flow-trans-radius = covariance` 不构建 density boundary profile，因此该配置无实际效果。
- 若用户显式设置 `flow-trans-radius-resolution`，但 `flow-velocity-sampler != density-normal`，validation 必须报错；默认值本身不能让默认 `covariance-ellipse` 配置报错。
- 不添加兼容别名，不支持裸数值 `angular/radial` 两个散乱配置项。

## Implementation Changes
核心类型和配置传递：

- 在 `include/blastwave/FlowFieldModel.h` 增加 `enum class FlowTransRadiusResolution { Balanced, Precise, Fast };`。
- 在 `BlastWaveConfig` 增加：
  - `FlowTransRadiusResolution flowTransRadiusResolution = FlowTransRadiusResolution::Balanced;`
  - `bool hasFlowTransRadiusResolution = false;`
- 在 `FlowFieldParameters` 末尾追加对应字段，避免破坏现有 aggregate initializer；`BlastWaveGeneratorSampling.cpp` 要把 config 传给 flow 参数。
- 在 `EventMedium::FlowTransRadiusProfile` 增加 `resolution`、`radialSamples`，并让 cache key 同时检查 mode、fraction、center、resolution、angularSamples、radialSamples、boundaryRadii size。

数值路径：

- 在 `DensityFieldModel` 增加 `evaluateDensityValue(const DensityField&, double x, double y)`，只算 density，不算 gradient；保留 `evaluateDensityField()` 给 density-normal 方向使用。
- `computeDensityPercentileBoundaryRadius()` 和 `computeDensityLevelBoundaryRadius()` 改用 `evaluateDensityValue()`。
- 移除 percentile 每条射线里的 `radii` vector；`radius = radialStep * i` 可直接恢复。
- percentile cumulative buffer 在 `getFlowTransRadiusProfile()` 中按 `radialSamples + 1` 分配一次，并传入每条射线复用，避免 per-ray allocation。
- `computeDensityDefinedXiUsed()` 不再由调用方传入 `radiusUpperBound`；改为 `getFlowTransRadiusProfile()` 在 cache miss 时计算 upper bound。cache hit 时不能重复扫 support points。

解析、validation、help：

- 在 `RunOptions.cpp` 增加 `parseFlowTransRadiusResolution()`，接受 `balanced|precise|fast`。
- 在 option dispatch 中处理 `flow-trans-radius-resolution`，同时适用于 config 和 CLI。
- 更新 `--help` 的 config key 列表和 QA-facing tuning knobs。
- 在 `BlastWaveGeneratorValidation.cpp` 把 `hasFlowTransRadiusResolution` 纳入 density-normal-only 显式配置检查，错误信息列出三个 flow-trans density-normal-only 项。

## Acceptance Criteria
- `density-percentile` / `density-level` 数学语义不变，`covariance` 行为不变。
- `balanced` 为默认 resolution；`precise` 可复现旧 `360 x 512` 数值路径。
- 不再在 boundary profile 中计算 gradient。
- `radiusUpperBound` 不再在每个 emission site 上重复计算。
- profile cache 不会跨 resolution 误复用。
- `ctest` 全过，ROOT generate + QA smoke 全过。
