Glauber–BlastWave 横向流场升级任务：方向混合与折中流强方案

1. 任务背景

当前 Glauber–BlastWave 框架已经能够从 Glauber participants 生成 smeared transverse density，并在此基础上构造横向流场。现有横向流方向主要采用密度梯度外向方向。现在需要扩展两个部分：

1. 横向流方向
    从纯梯度外向推广为几何外向与梯度外向的可调混合。
2. 横向流强大小
    新增一种折中方案：用 density contour 或累计密度百分位定义复杂几何中的归一化外向坐标，同时用累计压力梯度代理修正不同方向上的流强。

这两个部分应尽量解耦，使方向定义和流强定义可以分别替换、测试和组合。

⸻

Part I：横向流方向升级

2. 物理目标

横向流方向应同时兼顾两种图像：

几何外向图像

几何外向方向表示从事件中心指向当前发射点的方向。它对应整体径向膨胀，数值稳定，适合圆形、椭圆形或整体平滑的源。

优点：

* 稳定；
* 平滑；
* 与传统 blast-wave 的径向流图像兼容；
* 不容易受局部 density 噪声影响。

缺点：

* 对 hotspot、三角扰动、局部凹凸结构响应较弱；
* 可能过度强调整体几何，弱化局域压力梯度。

梯度外向图像

梯度外向方向表示从高密度区指向低密度区的方向。它可以看作局域压力梯度方向的代理。

优点：

* 更接近“压力梯度推动膨胀”的物理图像；
* 对复杂局部结构更敏感；
* 能够自然响应非椭圆结构。

缺点：

* 对 smoothing scale、网格分辨率和局域噪声敏感；
* 在低密度边缘或 saddle 区域可能方向不稳定；
* 多 hotspot 情况下可能出现局部方向剧烈变化。

混合方向图像

新增方向方案应允许在二者之间连续插值。

概念上：

flow direction =
normalized combination of:
  geometric outward direction
  gradient outward direction

行为要求：

* 可以退化为纯几何外向；
* 可以退化为纯梯度外向；
* 可以使用中间混合状态；
* 如果旧实现是纯梯度方向，则应能通过某种配置恢复旧行为；
* 混合方向应比纯梯度方向更平滑，但仍保留部分局域结构响应。

⸻

3. 方向部分计算流程

对每个发射点或流体元：

1. 读取该点位置；
2. 根据事件中心获得几何外向方向；
3. 根据 smeared density 的局域变化获得梯度外向方向；
4. 按用户配置或模型参数混合两类方向；
5. 对结果归一化；
6. 将最终方向传给现有 transverse boost 流程。

不要求新增固定函数名或固定字段名。agent 应优先复用当前项目中已有的 density、gradient、flow-field、boost 相关结构。

⸻

4. 方向部分行为约束

方向计算应满足以下约束：

1. 单位方向约束
    最终用于 boost 的方向应是单位方向，或者在使用前保证归一化。
2. 旧行为兼容
    应能恢复当前纯梯度方向模式，便于与旧结果对照。
3. 纯几何退化
    应能得到纯几何外向方向，用于检查几何径向膨胀的极限情况。
4. 连续插值
    从纯几何到纯梯度的过渡应是连续的，不应出现模式切换导致的突变。
5. 数值稳定
    中心附近、低梯度区域、边缘低密度区域都不应产生 NaN、inf 或零方向。
6. 局部异常保护
    如果几何方向或梯度方向在某点不可靠，应有回退策略，例如回退到另一类方向或使用现有默认处理。

⸻

Part II：新增折中版横向流强方案

5. 物理目标

原始 density contour 方法的优点是可以在复杂几何中定义“同一层”的空间结构。但如果直接使用：

相同密度百分位 -> 相同流强

这个规则并不充分。因为密度百分位只是几何排序变量，不是动力学累计变量。

更合理的近似应同时满足：

1. 外层点通常具有更大流强
    即使不做完整流体演化，也应保留传统 blast-wave 中“越靠外，横向流越强”的结构。
2. 不同方向可以有不同流强修正
    因为不同方向的压力梯度历史不同，尤其在椭圆、三角、hotspot 或复杂初态中，不能只用统一径向函数描述全部方向。

因此新增方案的核心是：

主径向增长
+
累计梯度修正

而不是单纯由 density percentile 直接给定流强。

⸻

6. 新流强方案的整体流程

新增流强方案应按以下逻辑计算：

1. 从 smeared density 出发；
2. 在每个方位上定义一个有效外边界；
3. 用该有效边界把发射点位置转化为归一化外向坐标；
4. 用归一化外向坐标给出主径向流强；
5. 沿从中心到发射点的路径估计累计梯度代理；
6. 将累计梯度代理转化为方向性修正；
7. 合成最终横向流强；
8. 对最终流强施加物理和数值限制；
9. 将流强与方向组合，进入现有 boost 流程。

⸻

7. 有效边界的定义原则

新方案需要为每个方向定义一个有效外边界。该边界不是严格几何硬边界，而是用于归一化径向坐标的参考尺度。

推荐使用累计密度权重百分位，而不是固定 density cutoff。

理由：

* 固定 cutoff 容易受低密度尾巴影响；
* 固定 cutoff 对不同事件、不同 multiplicity、不同 smearing 参数可能不稳定；
* 累计密度权重百分位更适合定义“主要发射区域的有效范围”。

行为要求：

* 有效边界应随方位角变化，从而保留椭圆、三角或更复杂形状；
* 有效边界不应被极少数低密度尾巴拉得过远；
* 有效边界不应过早截断主要发射区域；
* 边界计算应可缓存或预计算，避免对每个粒子重复高成本计算。

⸻

8. 归一化外向坐标

对每个发射点，根据其方向上的有效边界，定义一个归一化外向坐标。

概念上：

normalized outward coordinate =
distance from center / effective boundary distance in the same direction

该坐标表示：

当前点在该方向上有多靠外

行为要求：

* 中心附近接近 0；
* 有效边界附近接近 1；
* 从中心到边界应平滑增长；
* 边界外点允许 \(\xi>1\)，并通过最终速度上限稳定处理；
* 圆对称 profile 中应退化为普通径向坐标；
* 椭圆或复杂 profile 中应保留方向依赖的有效半径。

⸻

9. 主径向流强

主径向流强由归一化外向坐标决定。

概念上：

main radial flow increases with normalized outward coordinate

它负责表达：

越靠外 -> 横向流越强

行为要求：

* 外层点流强通常应大于内层点；
* 中心附近流强应较小；
* 边界附近流强接近该事件或该模式允许的主要流强尺度；
* 应允许控制径向增长形状，例如线性增长、更边缘增强或更平缓增长；
* 不应强制“相同 density percentile 的点必然具有完全相同流强”。

⸻

10. 累计梯度修正

10.1 修正目的

累计梯度修正用于表达：

在相似外向坐标处，不同方向可能由于压力梯度结构不同而有不同流强

它不应取代主径向流强，而应作为方向性修正。

10.2 梯度代理

推荐使用归一化梯度或对数密度梯度一类的代理，而不是直接使用密度梯度大小。

原因：

* 直接密度梯度容易受整体密度规模支配；
* 归一化梯度更接近压力梯度加速度的结构；
* 对数密度梯度能更自然表达相对变化。

但具体采用哪种数值实现，由 agent 根据已有 density field、gradient 计算方式和数值稳定性决定。

10.3 路径累计

对每个发射点，沿中心到该点的路径累计梯度代理。

概念上：

accumulated gradient =
integral or discrete sum of outward pressure-gradient proxy
from center to the point

不要求解析积分。可以使用：

* 离散路径采样；
* 网格插值；
* 预计算累计表；
* 其他与现有框架更容易耦合的近似方法。

行为要求：

* 累计量应反映路径上的压力梯度结构；
* 对 density floor 做保护，避免低密度区域发散；
* 对梯度场做必要平滑，避免噪声主导；
* 计算成本应可控。

⸻

11. 避免径向增长的重复计算

累计梯度通常也会随半径增大。如果直接把累计梯度乘到主径向流强上，可能重复表达：

越靠外 -> 流越强

从而导致边缘流强过强。

因此建议采用“同壳层平均扣除”的思想。

概念上：

gradient correction =
local accumulated gradient
-
average accumulated gradient among points with similar normalized outward coordinate

这样：

* 主径向项负责整体半径依赖；
* 梯度修正只负责同一外向层上的方向差异；
* 整体流强尺度不会被梯度修正大幅改变。

行为要求：

* 梯度修正关闭时，应退化为纯归一化外向坐标流强；
* 梯度修正开启后，主要改变不同方向上的相对流强；
* 梯度修正不应显著改变事件整体平均流强，除非明确配置允许；
* 梯度修正强度应可控。

⸻

12. 最终流强限制

最终横向流强必须满足物理和数值约束：

1. 不允许负流强；
2. 转换为速度后必须小于光速；
3. 不允许 NaN 或 inf；
4. 不允许少数异常点产生极端大 boost；
5. 应支持最大流强限制；
6. 默认参数应保守，避免过度硬化 p_T 谱；
7. 如果修正项导致不合理结果，应截断或回退到稳定值。

⸻

Part III：方向与流强的组合原则

13. 解耦原则

方向定义和流强定义应尽量相互独立。

推荐逻辑：

density field
 -> flow direction
 -> flow magnitude
 -> transverse boost

而不是把某一种方向和某一种流强写死为单一模式。

这样可以独立测试：

* 旧方向 + 旧流强；
* 混合方向 + 旧流强；
* 旧方向 + 新流强；
* 混合方向 + 新流强；
* 纯几何方向 + 新流强；
* 纯梯度方向 + 新流强。

⸻

14. 与旧框架兼容

新增功能应作为可选模式加入。

必须保持：

* 旧配置仍可运行；
* 旧模式结果不因新功能改变；
* 当前纯梯度方向可复现；
* 当前已有流强模式可复现；
* 当前粒子抽样和 boost 流程不应被无关重写。

⸻

Part IV：建议实现步骤

Step 1：识别现有流场流程

先确认当前框架中：

1. density field 如何生成；
2. 发射点如何选取；
3. 梯度方向在哪里计算；
4. 流强在哪里计算；
5. transverse boost 在哪里执行；
6. 配置系统如何控制不同模式。

目标是最小侵入接入新逻辑。

⸻

Step 2：加入方向混合

在现有梯度方向基础上加入几何方向，并允许两者混合。

实现重点：

* 保留旧纯梯度方向；
* 新增纯几何方向；
* 新增混合方向；
* 加入必要的数值保护；
* 确保输出方向可直接用于现有 boost。

⸻

Step 3：加入有效边界计算

为新流强方案建立方向相关的有效边界。

实现重点：

* 支持非圆形和复杂 profile；
* 优先使用累计密度权重百分位；
* 允许缓存或预计算；
* 对异常边界做保护；
* 不影响旧流强模式。

⸻

Step 4：计算归一化外向坐标

用有效边界将发射点位置映射为归一化外向坐标。

实现重点：

* 坐标随外向距离单调增长；
* 圆对称情况下退化为普通径向半径；
* 复杂几何下保留方向依赖；
* 对边界外点保留无上限基础 \(\xi\)，最终速度仍受物理上限保护。

⸻

Step 5：计算累计梯度代理

基于 density field 计算归一化梯度代理，并沿路径累计。

实现重点：

* 使用平滑后的 density；
* 避免低密度发散；
* 控制计算成本；
* 结果做事件内归一化；
* 不强制使用某个具体积分实现。

⸻

Step 6：构造方向性修正

通过同壳层平均扣除，得到方向性梯度修正。

实现重点：

* 避免重复计算径向增长；
* 修正项主要影响方向差异；
* 修正强度可控；
* 修正关闭时退化为纯主径向流强。

⸻

Step 7：合成最终流强并接入 boost

将主径向流强和梯度修正合成最终横向流强，然后与方向组合进入现有 boost。

实现重点：

* 非负限制；
* 最大值限制；
* 速度小于光速；
* 不重写无关抽样流程；
* 保持旧模式不变。

⸻

Part V：验证要求

15. 数值验证

至少测试以下 profile：

1. 圆形平滑源；
2. 椭圆源；
3. 带三角扰动的源；
4. 多 hotspot 源；
5. 有低密度尾巴或稀疏边缘的源。

检查：

* 无 NaN；
* 无 inf；
* 无异常零方向；
* 方向归一化正常；
* 流强非负；
* boost 速度小于光速；
* 边缘点没有异常大流强；
* 中心附近行为稳定。

⸻

16. 退化行为验证

方向部分

必须验证：

* 纯梯度设置能复现旧方向；
* 纯几何设置给出径向外向方向；
* 混合设置在二者之间平滑过渡。

流强部分

必须验证：

* 新流强方案关闭梯度修正时，退化为基于归一化外向坐标的主径向流强；
* 新流强方案关闭时，完全使用旧流强；
* 旧配置结果不因新代码路径改变。

⸻

17. 物理输出验证

至少比较：

1. 横向方向场；
2. 横向流强场；
3. 单粒子 \phi 分布；
4. p_T 谱；
5. v_2；
6. v_3；
7. event-by-event v_n；
8. femto source 或 HBT 相关输出，如果已有。

重点观察：

* 混合方向是否降低纯梯度方向的噪声；
* 混合方向是否仍保留复杂几何响应；
* 新流强是否体现外层更强流；
* 梯度修正是否带来合理方向性差异；
* p_T 谱是否被过度硬化；
* v_2、v_3 是否随初态几何变化；
* femto source 是否出现非物理收缩、畸变或振荡。

⸻

Part VI：建议测试组合

不要求一次性跑大量参数扫描，但至少覆盖以下几类：

类别	方向	流强	目的
旧基线	旧方向	旧流强	旧结果对照
纯几何方向	几何外向	旧流强	检查方向替换影响
混合方向	几何 + 梯度	旧流强	检查方向混合效果
新流强主径向项	任一稳定方向	新流强但关闭梯度修正	检查归一化外向坐标
新流强弱修正	混合方向	新流强 + 弱梯度修正	检查稳定性
新流强中等修正	混合方向	新流强 + 中等梯度修正	检查物理响应

默认先从弱梯度修正开始，不建议一开始使用强修正。

⸻

Part VII：交付物要求

agent 完成后应提供：

1. 改动摘要

说明：

* 横向流方向新增了什么行为；
* 横向流强新增了什么行为；
* 如何恢复旧行为；
* 新旧模式如何切换；
* 哪些部分保持不变。

2. 示例配置

提供至少三类配置示例：

1. 旧行为；
2. 混合方向 + 旧流强；
3. 混合方向 + 新折中流强。

字段命名应遵循项目已有风格，不需要照搬本文表述。

3. 验证输出

至少提供：

* 方向场图；
* 流强场图；
* p_T 谱对比；
* \phi 分布对比；
* v_2、v_3 对比；
* 如已有 femto 分析，则提供 source 或 HBT 相关对比。

4. 风险说明

需要记录：

* 梯度方向对 smoothing scale 敏感；
* 累计梯度代理不是完整流体演化；
* 有效边界定义会影响归一化外向坐标；
* 边界百分位过高可能受低密度尾巴影响；
* 边界百分位过低可能截断外层结构；
* 梯度修正过强可能使 p_T 谱过硬；
* 壳层平均扣除方式会影响修正平滑性；
* 多 hotspot profile 中局部方向可能不连续，需要通过混合权重和平滑控制。

⸻

Part VIII：最终验收标准

任务完成后应满足：

1. 横向流方向可以在几何外向、梯度外向、混合方向之间切换；
2. 当前纯梯度方向行为可以恢复；
3. 新增流强方案可以独立启用或关闭；
4. 新流强方案关闭梯度修正后，退化为归一化外向坐标主导的径向流强；
5. 方向和流强可以自由组合；
6. 旧配置结果不应被新功能改变；
7. 基础 profile 测试无数值异常；
8. p_T、\phi、v_2、v_3、femto source 不应出现明显非物理行为；
9. 文档中应明确说明：该方案是无完整流体演化条件下的有效参数化，不是严格的流体动力学解。
