## Add `v2{2}(pT)` Analysis for Blast-Wave Outputs

### Summary
- Add a new differential-flow analysis feature that computes `v2{2}(pT)` from this project’s blast-wave result files using explicitly configured `pT` bin edges.
- Support both execution modes:
  - joint mode: compute during `generate_blastwave_events`
  - standalone mode: compute later from a result ROOT file with `analyze_blastwave_v2pt`
- Support both output placements:
  - write analysis payload into the main result ROOT file
  - write analysis payload into a separate ROOT file
- Do not support pre-feature historical ROOT files. New files produced after this change must carry the `pT` bin metadata needed for standalone analysis.

### Fixed Decisions
- Observable definition is differential two-particle cumulant:
  - reference particles: all particles in the event
  - POI: particles in one configured `pT` bin
  - self-correlations: removed
  - no `eta` gap
  - no subevent split
  - no PID or charge filtering beyond whatever particles are already in the file
- Track weighting is fixed to unit weight:
  - ignore `particles.emission_weight`
  - this is intentionally closer to a standard track-based experimental baseline than to the current weighted `events.v2`
- Use `pt = hypot(px, py)` and `phi = computeAzimuth(px, py)`.
- `pT` bin convention is `[edge_i, edge_{i+1})`, with the final bin closed on the right.
- Empty or undefined bins are allowed:
  - if a bin has zero contributing POI-reference pairs globally, write bin content `0` and bin error `0`
  - do not fail the whole analysis for empty tail bins
- Global reference cumulant must be positive:
  - if final `c2{2} <= 0`, treat analysis as failed and do not write partial `v2_2_pt` objects

### Algorithm Contract
- Per event, let `M` be the event particle multiplicity and `m_bin` the POI count in one `pT` bin.
- Build event Q-vectors with unit weights:
  - `Q2 = Σ_ref exp(2iφ)`
  - `P2(bin) = Σ_poi in bin exp(2iφ)`
- Accumulate the integrated reference cumulant with pair-count weighting:
  - `c2{2} = Σ(|Q2|^2 - M) / Σ(M(M-1))`
  - only events with `M >= 2` contribute
- Accumulate the differential cumulant for each `pT` bin:
  - `d2{2}(bin) = Σ(Re[P2(bin) Q2*] - m_bin) / Σ(m_bin (M-1))`
  - only events with `M >= 2` and `m_bin > 0` contribute to that bin
- Final observable:
  - `v2{2}(bin) = d2{2}(bin) / sqrt(c2{2})`
- Statistical uncertainty:
  - use leave-one-event-out jackknife
  - jackknife sample = all events that contributed to the global `c2{2}` denominator
  - for each leave-one-out sample, recompute `c2{2}`, `d2{2}(bin)`, and `v2{2}(bin)` from accumulated event contributions
  - error formula: `sqrt((N-1)/N * Σ(v_i - mean(v))^2)`

### Public Interface Changes
- Keep analysis options out of `blastwave::BlastWaveConfig`. They belong in the app layer, alongside `outputPath` and `progressMode`.
- Extend `blastwave::app::RunOptions` with:
  - `std::vector<double> v2PtBinEdges`
  - `enum class V2PtOutputMode { SameFile, SeparateFile }`
  - `std::string v2PtOutputPath`
- New config/CLI keys for generation:
  - `v2pt-bins = 0.0,0.2,0.4,0.6,...`
  - `v2pt-output-mode = same-file|separate-file`
  - `v2pt-output = <path>`
- Parsing/validation rules:
  - `v2pt-bins` is optional; if absent, no `v2{2}(pT)` work is done
  - if present, require at least 2 edges
  - all edges finite, non-negative, and strictly increasing
  - allow optional spaces around commas
  - `v2pt-output-mode` defaults to `same-file`
  - `v2pt-output` is valid only for `separate-file`
  - relative `v2pt-output` in config files resolves relative to the config file directory, same as `output`
- Add standalone executable:
  - `analyze_blastwave_v2pt --input <result.root> [--output <analysis.root> | --inplace]`
  - `--output` and `--inplace` are mutually exclusive
  - default standalone output path: sibling file `<input-stem>_v2pt.root`
  - standalone analysis reads `pT` bin edges from the input ROOT metadata; if missing, fail with a clear error

### ROOT Output Contract
- Add object-name constants in [include/blastwave/io/RootOutputSchema.h](/Users/allenzhou/Research_software/Blast_wave/include/blastwave/io/RootOutputSchema.h):
  - `v2_2_pt_edges`
  - `v2_2_pt`
  - `v2_2_pt_canvas`
- Object types:
  - `v2_2_pt_edges`: `TVectorD`
  - `v2_2_pt`: `TH1D` with variable bin edges
  - `v2_2_pt_canvas`: `TCanvas`
- Histogram title:
  - `Differential v_{2}{2};p_{T} [GeV];v_{2}{2}`
- Writing rules:
  - if `v2pt-bins` is configured, always write `v2_2_pt_edges` into the main result file
  - if `same-file`, also write `v2_2_pt` and `v2_2_pt_canvas` into the main result file
  - if `separate-file`, write `v2_2_pt_edges`, `v2_2_pt`, and `v2_2_pt_canvas` into the separate analysis file; keep `v2_2_pt_edges` in the main result file too
- Overwrite policy:
  - same object names are rewritten on re-analysis
  - never leave a partially written payload if the analysis fails

### QA and Validation Behavior
- Extend `qa_blastwave_output` with optional validation of the new payload:
  - if none of `v2_2_pt_edges`, `v2_2_pt`, `v2_2_pt_canvas` exists, ignore the feature
  - if any exists, require all three
  - require `v2_2_pt_edges` to be finite, non-negative, and strictly increasing
  - require `v2_2_pt` bin edges to match `v2_2_pt_edges`
  - require all `v2_2_pt` bin contents finite and all bin errors finite and non-negative
  - recompute `v2{2}(pT)` from the `particles` tree with the shared core module and compare to the file payload using the repo’s existing `nearlyEqual`-style tolerance
- Do not add a new `qa_v2_2_pt` output object in this iteration. Validation is enough.
